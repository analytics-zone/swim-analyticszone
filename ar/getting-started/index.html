البدء مع Swim Analytics

دليلك الشامل لتتبع أداء السباحة، اختبارات CSS، وتحليل حمل التدريب

مرحباً بك في السباحة القائمة على البيانات

يحول Swim Analytics تدريباتك في السباحة إلى رؤى قابلة للتنفيذ باستخدام مقاييس سرعة السباحة الحرجة (CSS)، ودرجة ضغط التدريب (sTSS)، ومخطط إدارة الأداء (PMC). سيأخذك هذا الدليل من الإعداد الأولي إلى تحليل حمل التدريب المتقدم في 4 خطوات بسيطة.

البدء السريع (5 دقائق)

1

تحميل وتثبيت

حمّل Swim Analytics من App Store وامنح إذناً للوصول إلى Apple Health. يقوم التطبيق بمزامنة تدريبات السباحة تلقائياً: لا حاجة لتسجيل يدوي.

تحميل التطبيق →
2

إجراء اختبار CSS

أكمل محاكمة زمنية 400م و200م لتحديد سرعة السباحة الحرجة الخاصة بك. هذا هو الأساس لجميع المقاييس: بدون CSS، لا يمكن حساب sTSS أو مناطق التدريب.

بروتوكول اختبار CSS ↓
3

إدخال نتائج CSS

أدخل أوقاتك لـ400م و200م في التطبيق. يحسب Swim Analytics CSS، مناطق السرعة ويخصص جميع المقاييس حسب فسيولوجيتك. حدّث كل 6-8 أسابيع مع تحسن لياقتك.

4

بدء تسجيل التدريبات

اسبح مع Apple Watch وتطبيق Health. يستورد Swim Analytics التدريبات تلقائياً، يحسب sTSS، يحدث CTL/ATL/TSB ويتتبع التقدم. لا حاجة لإدخال بيانات يدوي.

بروتوكول اختبار CSS الكامل

📋 ما تحتاجه

  • الوصول إلى المسبح: مسبح 25م أو 50م (25 ياردة مقبول)
  • التوقيت: ساعة إيقاف، ساعة حائط أو Apple Watch
  • وقت الإحماء: 15-20 دقيقة قبل الاختبار
  • الاستشفاء: 5-10 دقائق بين الاختبارات
  • الجهد: سرعة قصوى مستدامة (ليست سرعة كاملة)

⏱️ ظروف يوم الاختبار

  • مرتاح: بدون تدريب شاق 24-48 ساعة قبل
  • مرطّب: مرطّب جيداً، تغذية عادية
  • درجة حرارة المسبح: 26-28°م مثالي (تجنب البارد/الحار جداً)
  • وقت اليوم: عندما تتدرب عادة بشكل أفضل
  • المعدات: نفس التدريبات (نظارات، قبعة، بدلة سباحة)

اختبار CSS خطوة بخطوة

الإحماء

15-20 دقيقة

400-800م سباحة سهلة، تمارين تقنية وتدريجية. تضمن 2-3×50 بسرعة متزايدة (60%، 75%، 85% جهد). استرح 2-3 دقائق قبل الاختبار.

الاختبار 1

400م جهد أقصى

البداية من الجدار (بدون غطس). اسبح 400م بأسرع سرعة يمكنك الحفاظ عليها للمسافة الكاملة. هذا ليس سرعة كاملة: وزّع جهدك. سجل الوقت بصيغة دد:ثث (مثال: 6:08).

نصيحة السرعة: حاول الحفاظ على أجزاء 100م موحدة. يجب أن تكون 200م الثانية ≤ 200م الأولى (سلبي مثالي).
الاستشفاء

5-10 دقائق

مرحلة حرجة: سباحة سهلة أو راحة كاملة. انتظر حتى ينخفض معدل نبضات القلب عن 120 نبضة/دقيقة والتنفس يتعافى بالكامل. استشفاء غير كافٍ = CSS غير دقيق.

الاختبار 2

200م جهد أقصى

البداية من الجدار (بدون غطس). جهد مستدام أقصى لمسافة 200م. يجب أن يشعر هذا بأنه أصعب لكل 100م من 400م. سجل الوقت بصيغة دد:ثث (مثال: 2:30).

التحقق من الصلاحية: يجب أن تكون سرعة 200م/100م أسرع بـ3-6 ثواني من سرعة 400م/100م. إذا لم يكن كذلك، فالاستشفاء كان غير كافٍ أو السرعة غير مناسبة.
التبريد

10-15 دقيقة

300-500م سباحة سهلة، تمديد. سجل أوقاتك فوراً: لا تعتمد على الذاكرة.

⚠️ أخطاء شائعة في اختبار CSS

  • الانطلاق سريعاً جداً في 400م: يؤدي إلى الانهيار، CSS غير دقيق. استخدم سرعة موحدة.
  • استشفاء غير كافٍ بين الاختبارات: التعب يبطئ 200م، مما يجعل CSS سريعاً اصطناعياً → مناطق إفراط في التدريب.
  • استخدام بدايات الغطس: يضيف ميزة 0.5-1.5ث، يُحرّف الحسابات. ابدأ دائماً من الجدار.
  • الاختبار عندما تكون متعباً: حمل تدريب ثقيل 24-48 ساعة قبل = نتائج منخفضة. اختبر عندما تكون منتعشاً.
  • عدم التسجيل فوراً: الذاكرة غير موثوقة. دوّن الأوقات قبل التبريد.

إدخال نتائج CSS في Swim Analytics

الخطوة 1: فتح إعدادات CSS

في تطبيق Swim Analytics، اذهب إلى الإعدادات ← سرعة السباحة الحرجة. اضغط "إجراء اختبار CSS" أو "تحديث CSS".

الخطوة 2: إدخال الأوقات

أدخل وقت 400م الخاص بك (مثال: 6:08) ووقت 200م (مثال: 2:30). استخدم الصيغة الدقيقة المعروضة. اضغط "حساب".

الخطوة 3: مراجعة النتائج

يعرض التطبيق:

  • سرعة CSS: 0.917 م/ث
  • إيقاع CSS: 1:49/100م
  • مناطق التدريب: 7 مناطق مخصصة (المنطقة 1-7)
  • خط أساس sTSS: مُفعّل الآن لجميع التدريبات

الخطوة 4: حفظ ومزامنة

اضغط "حفظ CSS". يقوم التطبيق فوراً بـ:

  • إعادة حساب مناطق التدريب
  • تحديث sTSS بأثر رجعي لآخر 90 يوماً
  • ضبط حسابات CTL/ATL/TSB
  • تفعيل تحليل التدريبات حسب المناطق

💡 نصيحة محترف: اختبارات CSS التاريخية

إذا كنت تعرف CSS الخاص بك من اختبارات سابقة، يمكنك إدخال تلك الأوقات مباشرة. ومع ذلك، للحصول على أفضل النتائج، قم بإجراء اختبار جديد كل 6-8 أسابيع. يجب أن يتحسن CSS الخاص بك (يصبح أسرع) مع تقدم التدريب.

فهم المقاييس الخاصة بك

سرعة السباحة الحرجة (CSS)

ما هو: إيقاع عتبتك الهوائية: أسرع سرعة يمكنك الحفاظ عليها لحوالي 30 دقيقة دون إنهاك.

ماذا يعني: CSS = 1:49/100م يعني أنه يمكنك الحفاظ على إيقاع 1:49 في جهود العتبة المستدامة.

كيفية استخدامه: أساس لجميع مناطق التدريب وحساب sTSS. حدّث كل 6-8 أسابيع.

تعلم CSS →

مناطق التدريب

ما هي: 7 نطاقات شدة بناءً على CSS الخاص بك، من الاستشفاء (المنطقة 1) إلى السرعة (المنطقة 7).

ماذا تعني: كل منطقة تستهدف تكيفات فسيولوجية محددة (قاعدة هوائية، عتبة، VO₂max).

كيفية استخدامها: اتبع وصفات المنطقة للتدريب المنظم. يعرض التطبيق الوقت في المنطقة لكل تدريب.

مناطق التدريب →

درجة ضغط التدريب في السباحة (sTSS)

ما هو: ضغط التدريب المُحدّد كمياً يجمع بين الشدة والمدة. 1 ساعة بإيقاع CSS = 100 sTSS.

ماذا يعني: sTSS 50 = استشفاء سهل، sTSS 100 = معتدل، sTSS 200+ = جلسة صعبة جداً.

كيفية استخدامه: تتبع sTSS اليومي/الأسبوعي لإدارة حمل التدريب. اهدف إلى زيادة 5-10 sTSS في الأسبوع كحد أقصى.

دليل sTSS →

CTL / ATL / TSB

ما هي:

  • CTL: حمل التدريب المزمن (اللياقة) - متوسط 42 يوماً من sTSS
  • ATL: حمل التدريب الحاد (التعب) - متوسط 7 أيام من sTSS
  • TSB: توازن ضغط التدريب (الشكل) = CTL - ATL

كيفية استخدامها: TSB إيجابي = منتعش/في تخفيف، TSB سلبي = متعب. تنافس عندما TSB = +5 إلى +25.

📊 أهداف أسبوعك الأول

بعد إدخال CSS وإكمال 3-5 تدريبات:

  • التحقق من قيم sTSS: تأكد من تطابقها مع إدراك الجهد (سهل ~50، معتدل ~100، صعب ~150+)
  • مراجعة التوزيع حسب المناطق: هل تقضي 60-70% في المنطقة 2 (القاعدة الهوائية)؟
  • تحديد CTL أساسي: متوسط sTSS لأسبوعك الأول يصبح خط الأساس للياقة الأولي
  • تحديد الأنماط: أي التدريبات تولد أعلى sTSS؟ هل تتعافى بشكل مناسب؟

رحلة المستخدم النموذجية (أول 8 أسابيع)

الأسبوع 1-2: تحديد الخط الأساسي

  • إجراء اختبار CSS وإدخال النتائج
  • إكمال 3-5 تدريبات عادية
  • ملاحظة قيم sTSS والتوزيع حسب المناطق
  • تحديد CTL الأولي (مستوى اللياقة)
  • الهدف: فهم المقاييس، بدون تغييرات بعد

الأسبوع 3-4: تطبيق المناطق

  • استخدام مناطق CSS في تخطيط التدريبات
  • السباحة بقصد في المنطقة 2 للمجموعات الهوائية
  • تتبع إجماليات sTSS الأسبوعية (البحث عن الاتساق)
  • مراقبة TSB (يجب أن يكون سلبياً قليلاً = تدريب)
  • الهدف: التدريب حسب المناطق، ليس بالشعور

الأسبوع 5-6: التحميل التدريجي

  • زيادة sTSS الأسبوعي 5-10% من الخط الأساسي
  • إضافة 1 جلسة عتبة (المنطقة 4) في الأسبوع
  • يجب أن يرتفع CTL تدريجياً (اللياقة تتحسن)
  • قد يبلغ ATL ذروته في أسابيع صعبة (طبيعي)
  • الهدف: تقدم مُتحكَم به في اللياقة

الأسبوع 7-8: إعادة التقييم والضبط

  • إجراء اختبار CSS ثانٍ (يجب أن يكون أسرع)
  • تحديث المناطق في التطبيق (الإيقاع يتحسن)
  • مقارنة CTL الأسبوع 1 مقابل الأسبوع 8 (يجب أن يكون +10-20)
  • مراجعة التقدم: هل تنخفض الأوقات؟ هل يشعر أسهل؟
  • الهدف: التحقق من فعالية التدريب

✅ مؤشرات النجاح

بعد 8 أسابيع من التدريب المنظم مع Swim Analytics، يجب أن ترى:

  • تحسن CSS: 1-3% أسرع في إيقاع CSS (مثال: 1:49 → 1:47)
  • زيادة CTL: +15-25 نقطة (مثال: 30 → 50 CTL)
  • sTSS متسق: إجماليات أسبوعية ضمن تباين 10-15%
  • تقسيم أفضل: أجزاء أكثر انتظاماً، معايرة أفضل للجهد
  • استشفاء محسّن: TSB يدور بشكل متوقع (-10 إلى +5)

استكشاف الأخطاء والأسئلة الشائعة

يبدو sTSS الخاص بي مرتفعاً/منخفضاً جداً لجهد التدريب

السبب: CSS قديم أو غير دقيق.

الحل: أعد اختبار CSS. إذا اختبرت عندما كنت متعباً أو قسّمت بشكل سيء، فإن CSS سيكون غير صحيح. اختبار CSS مناسب حرج لجميع المقاييس اللاحقة.

يعرض التطبيق "لم يتم تكوين CSS"

السبب: اختبار CSS غير مكتمل أو غير محفوظ.

الحل: اذهب إلى الإعدادات → سرعة السباحة الحرجة → إجراء الاختبار. أدخل كلا وقتي 400م و200م، ثم اضغط حفظ.

التدريبات لا تتزامن من Apple Watch

السبب: أذونات تطبيق Health غير ممنوحة أو التدريب غير مصنف كـ"سباحة".

الحل: تحقق من الإعدادات → الخصوصية → الصحة → Swim Analytics → السماح بقراءة التدريبات. تأكد من أن نوع التدريب في Apple Watch هو "السباحة في المسبح" أو "السباحة في المياه المفتوحة".

CTL لا يزداد رغم التدريب المتسق

السبب: إجماليات sTSS منخفضة جداً أو تكرار غير متسق.

الحل: CTL هو متوسط مرجح أسياً لـ42 يوماً. يرتفع ببطء. زد sTSS الأسبوعي 5-10%، وحافظ على 4+ تدريبات/أسبوع لنمو CTL متسق.

كم مرة يجب أن أعيد اختبار CSS؟

التوصية: كل 6-8 أسابيع خلال مراحل القاعدة/البناء. أعد الاختبار بعد المرض، الإصابة، استراحة طويلة، أو عندما تشعر المناطق باستمرار سهلة جداً/صعبة جداً.

هل يمكنني استخدام Swim Analytics لأنماط أخرى؟

نعم، مع قيود: يتم اختبار CSS عادةً في النمط الحر. لتدريبات IM/ظهر/صدر، يتم حساب sTSS بناءً على CSS النمط الحر. فكر في إجراء اختبارات CSS محددة حسب النمط لمزيد من الدقة.
