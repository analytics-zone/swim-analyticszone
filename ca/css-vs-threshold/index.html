Velocitat crítica de natació (CSS) vs. ritme llindar: quin és important?

Durant anys, "Threshold Pace" (T-Pace) va ser l'estàndard d'or. El 2026, el canvi a la velocitat crítica de natació (CSS) està redefinint com els atletes ritmen les seves sessions i prediuen la fatiga del dia de la cursa.

🎯 La resposta ràpida

T-Pace (ritme llindar) és una estimació general del ritme que podeu mantenir durant una hora, que s'utilitza sovint per establir zones d'entrenament àmplies. És una mètrica "pràctica" però menys granular.

CSS (Velocitat crítica de natació) és el llindar "funcional" derivat matemàticament de dues distàncies diferents (per exemple, 400 m i 200 m). És el ritme teòric que pots mantenir sense colpejar el "mur". Si ets nedador o triatleta de resistència, el teu CSS és el número més important del teu programa d'entrenament.

El canvi de 2026: de l'esforç a la fisiologia

Gestionar la fatiga a la piscina és notòriament difícil en comparació amb anar en bicicleta o córrer. Sense comptador de potència, el ritme és el nostre únic indicador de la intensitat. Tot i que molts nedadors encara confien en els ritmes de "contrarellotge" de la vella escola de 1500 m, l'entrenament modern s'ha traslladat al Model CSS de dues distàncies.

La principal diferència és Precisió versus esforç percebut. Un T-Pace basat en un únic esforç llarg sovint captura la vostra força mental més que el vostre llindar metabòlic. El CSS, calculat a partir de dos punts diferents de la vostra "corba ritme-duració", revela el límit aeròbic real del vostre motor.

Comparació mètrica

Ritme llindar (ritme T)

La mètrica "simple".

El T-Pace sol derivar-se d'un sol esforç màxim de 1500 m o 1000 m. Representa la teva "línia vermella" per a una natació contínua.

  • Avantatges: Simple d'entendre. Una prova, un número. Ideal per a jocs llargs i continus.
  • Contres: Molt sensible als errors de ritme. No té en compte la rapidesa amb què s'esvaeix sota la fatiga. Tracta tots els nedadors amb el mateix ritme de 1.500 m com si tinguessin el mateix perfil fisiològic.

Velocitat crítica de natació (CSS)

La mètrica "funcional".

CSS és una derivació matemàtica basada en dues distàncies (normalment un esforç màxim de 400 m i un 200 m).

  • Avantatges: Elimina el biaix de ritme. Prediu millor el vostre comportament a intervals. Proporciona un "sostre aeròbic" objectiu.
  • Contres: Requereix dues proves separades. Matemàtiques una mica més complexes (calculades automàticament a Swim Analytics). Requereix que les proves es facin amb un descans constant.

Per què importa el Delta

El "Delta" és la diferència entre el teu ritme de 400 m i el teu ritme de 200 m. Dos nedadors poden tenir un ritme T d'1:40/100 m, però els seus perfils CSS poden semblar així:

Nedador A ("El dièsel")

Petit delta entre 200m i 400m. Aquest nedador té una eficiència aeròbica molt alta però una velocitat d'esprint més baixa. El seu CSS està molt a prop del seu T-Pace.

Nedador B ("El velocista")

Gran delta. Gran velocitat de 200 m, però baixa significativament als 400 m. El seu CSS serà més baix del que suggereix el seu T-Pace. Si intenten entrenar a un "T-Pace" genèric, cuinaran constantment el seu sistema anaeròbic.

📈 La fórmula CSS

CSS = (D1 - D2) / (T1 - T2)

On D1/D2 són les dues distàncies i T1/T2 són els temps en segons. Això revela el "pendent" del vostre ritme sostenible sense el soroll del vostre augment anaeròbic inicial.

Com realitzar la prova CSS (pas a pas)

Per calcular el vostre CSS, heu de realitzar dues proves màximes en dies diferents (o amb una recuperació completa entremig) en un grup estàndard:

  1. Escalfament: Realitzeu un escalfament exhaustiu i estandarditzat (normalment de 400 a 600 m de braçades i moviments mixts) per preparar el vostre cos.
  2. Prova 1 (400 m esforç màxim): Neda 400 m el més ràpid possible. Registra l'hora exacta. Estigui el teu ritme per no esgotar-te en els primers 100 m.
  3. Recuperació activa: Nedar entre 100 i 200 m molt fàcil d'eliminar l'àcid làctic, seguit de 5-10 minuts de descans.
  4. Prova 2 (200 m esforç màxim): Nedar 200 m amb el màxim esforç absolut. Registra l'hora exacta.
  5. Calcula: Converteix els teus temps a segons i apliqueu la fórmula anterior o utilitzeu l'eina de calculadora CSS a Swim Analytics per obtenir el vostre ritme de 100 m.

Quin hauríeu d'utilitzar?

Utilitzeu el ritme llindar (T-Pace) si:

  • Esteu començant amb l'entrenament basat en el ritme.
  • Només realitzeu nedacions llargues i contínues (aigua oberta 3 km+).
  • No voleu fer front a diverses sessions de proves.

Utilitzeu la velocitat crítica de natació (CSS) si:

  • Triatletes i nedadors de marató: Per evitar "cremar l'espelma" massa aviat en una cursa.
  • Entrenament per intervals: Per establir zones específiques de CSS + 2s o CSS - 1s que funcionin realment.
  • Seguiment del progrés: CSS és més sensible a les millores aeròbiques que una prova genèrica de 1000 m.

Preguntes freqüents sobre ritme llindar i CSS

És el mateix CSS que el ritme llindar?

No. Són conceptes de llindar relacionats, però no són la mateixa prova. El ritme llindar sol prové d'una natació més llarga, mentre que el CSS es calcula a partir de dos esforços i ofereix un àncora més repetible per al treball a intervals.

Què és millor per al triatló i el ritme en aigües obertes?

CSS sol ser el millor punt de partida perquè reflecteix un ritme sostenible sense premiar excessivament la velocitat a curt termini. El ritme llindar encara pot ser útil per a nedars llargs i constants, però el CSS sol ser més fàcil de convertir en objectius de ritme i treball de zona.

Com puc passar del ritme llindar a les zones de natació basades en CSS?

Utilitzeu la vostra propera prova de 400 i 200 m per configurar CSS i, a continuació, fixeu el treball d'interval al voltant d'aquest número. Si ja entrenes amb T-Pace, compara ambdós números i utilitza la guia CSS i les zones d'entrenament de natació per reconstruir els teus conjunts al voltant d'un ritme aeròbic repetible.
