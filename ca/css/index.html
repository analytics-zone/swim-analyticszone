Velocitat crítica de natació (CSS)

El punt de referència pràctic de ritme que fixa zones d'entrenament de natació, conjunts de llindars i anàlisi de càrrega d'entrenament

Resposta ràpida

Velocitat crítica de natació (CSS) és una estimació provada en camp del vostre ritme de natació llindar sostenible. La majoria de nedadors ho calculen a partir de a Contrarellotge 400m + 200mi, a continuació, utilitzeu el resultat per establir zones d'entrenament, conjunts de llindars de ritme i potència sTSS i càrrega d'entrenament anàlisi.

  • Millor ús: entrenament de llindar, ritme repetible i seguiment de càrrega
  • Interpretació típica: aproximadament el ritme més ràpid que pots mantenir aeròbicament per a un treball dur i controlat
  • No l'utilitzeu a cegues: El mal ritme, la recuperació incompleta, les diferents condicions de la piscina o les ajudes a l'arrossegament poden distorsionar el resultat
  • Reprova la cadència: cada 6-8 setmanes, o després d'un bloc d'entrenament significatiu

Aportacions clau

  • Què és CSS: una estimació pràctica del ritme llindar per als nedadors, construïda a partir de proves de camp senzilles
  • Per què importa: CSS converteix l'entrenament de natació de conjectures en un ritme repetible i un millor control de càrrega
  • Què desbloqueja:zones, sTSS, gestió del rendiment, i comprovacions de progrés més clares
  • Què no fa: substituir l'anàlisi de la tècnica, el context d'aigües obertes o interpretació de l'eficiència
  • El millor pas següent: prova bé el CSS i després fes-lo servir per estructurar el teu llindar i el treball aeròbic

Si només manteniu una mètrica de natació com a àncora de ritme, manteniu CSS. No és perfecte i no és una prova de laboratori fisiològica completa, però és pràctica, repetible i directament útil. Aquesta combinació és la raó per la qual CSS continua sent la columna vertebral de l'entrenament de natació basat en dades.

Per a la majoria de nedadors, Velocitat crítica de natació és el pont entre les proves de temps en brut i les decisions d'entrenament quotidianes. Ajuda a respondre cinc preguntes importants:

  • Quina velocitat han de ser realment els conjunts de llindars?
  • Quin rang de ritmes pertany a cadascun zona de bany?
  • Com he de repetir el ritme 100, 200 i 400?
  • Com puc quantificar l'estrès de la natació? sTSS?
  • La meva forma física de natació aeròbica es mou en la direcció correcta?

Quines mesures CSS

CSS és una estimació pràctica del ritme aeròbic dur de natació, normalment derivat d'una contrarellotge de 400 i 200 m. En el llenguatge d'entrenament, sovint es tracta com el vostre ritme llindar per a l'entrenament estructurat de la piscina: prou ràpid com per importar, prou controlat per repetir-lo i prou estable per construir zones al voltant.

🎯 El model mental útil

Penseu en CSS com el vostre ritme de referència operatiu, no és un nombre màgic. És la referència de ritme que fa que la resta de les teves mètriques de natació siguin coherents.

Per què importa CSS

CSS és important perquè connecta la fisiologia amb l'execució. Molts nedadors saben que van fer un "set dur" o una "sessió de tempo", però no poden dir si el ritme era correcte. CSS soluciona aquest problema donant-vos un punt de referència que podeu utilitzar sessió rere sessió.

🏊 Zones d'entrenament

CSS us ofereix un àncora net per zones de bany, el treball fàcil es manté fàcil i el treball llindar es manté com a objectiu.

📊 Càrrega d'entrenament

Sense un CSS utilitzable, sTSS i mètriques de gestió del rendiment perden el seu sentit.

⏱️ Ritmo repetible

CSS us ajuda a ritme 100, 200 i 400 amb més coherència en lloc de confiar només en la sensació.

📈 Comprovacions de progrés

Quan CSS millora, el vostre ritme llindar es mou més ràpidament. Aquest és un fort senyal que la forma física de la natació aeròbica també està millorant.

⚠️ Dependència crítica: Si el vostre CSS és incorrecte, les vostres zones, sTSS i els números de gestió del rendiment aniran a la deriva en la direcció equivocada. Una mala entrada produeix una mala anàlisi.

Per a què serveix CSS i per a què no

CSS és excel·lent per a: conjunts de llindars, treball de qualitat aeròbica, configuració de zones i seguiment de càrrega específic per a la natació.

CSS no és suficient per a: Perfil de sprint, diagnòstic de tècnica de braçada, execució de curses en aigües obertes en condicions variables o interpretació de l'eficiència per si sola.

Utilitzeu CSS per

  • Prescripcions de ritme llindar
  • Repetiu el ritme 100/200/400
  • Personalitzat estructura de la zona
  • sTSS i gestió de càrrega
  • Seguiment del progrés aeròbic al llarg d'una temporada

No utilitzeu CSS sol per

  • Comparant diferents longituds de piscines com si fossin idèntiques
  • Comparant nedadors amb perfils de tècnica molt diferents
  • Avaluar la qualitat del traç sense mecànica del traç
  • Jutjar l'eficiència sense context al voltant LLOBO
  • Suposant que el ritme en aigües obertes coincidirà exactament amb el ritme CSS de la piscina

Com provar CSS en 4 passos pràctics

📋 Protocol estàndard

  1. Escalfeu correctament

    Neda entre 300 i 800 m amb facilitat, inclou exercicis i afegeix uns quants esforços de construcció controlats abans de la prova.

  2. Nedar una contrarellotge de 400 m

    Utilitzeu un inici d'impuls, no una immersió. L'objectiu és el vostre 400 m sostenible més ràpid, no els primers 100 imprudents.

  3. Recuperar-se completament

    Pren 5-10 minuts descans fàcil o complet. La recuperació incompleta és una de les maneres més ràpides d'arruïnar el resultat.

  4. Nedar una contrarellotge de 200 m

    De nou, feu servir un inici automàtic i registreu l'hora amb precisió. El vostre ritme de 200 m hauria de ser significativament més ràpid que el de 400 m.

⚠️ Els errors de prova més comuns

Mal ritme de 400 m

Problema: Surt massa fort, després t'esvaeixes malament.

Resultat: Els 400 m deixa de reflectir la qualitat del ritme sostenible.

Correcció: Apunta a divisions iguals o lleugerament negatives.

Recuperació insuficient

Problema: El cansament fa que els 200 m siguin artificialment lents.

Resultat: CSS es distorsiona i les teves zones d'entrenament es desplacen.

Correcció: Espereu fins que la respiració estigui controlada i la recuperació se senti completa.

Condicions mixtes

Problema: Les diferents longituds de la piscina, els grans canvis d'impuls o les ajudes d'arrossegament alteren l'entorn de prova.

Resultat: Compareu números que no són realment comparables.

Correcció: Mantenir les condicions el més coherents possible d'una prova a una altra.

🔄 Freqüència de reprovació

Torneu a provar CSS cada cop 6-8 setmanes, o després d'un bloc significatiu centrat en la consolidació del llindar, la resistència o la tècnica. La qüestió no és una prova constant. La qüestió és utilitzar un punt de referència de manera suficient per veure el canvi.

Fórmula de càlcul CSS

Fórmula

CSS (m/s) = (D₂ - D₁) / (T₂ - T₁)

On:

  • D₁ = 200 metres
  • D₂ = 400 metres
  • T₁ = Temps durant 200 m (en segons)
  • T₂ = Temps durant 400 m (en segons)

Simplificat per a Ritme per 100 m

CSS Ritme/100 m (segons) = (T₄₀₀ - T₂₀₀) / 2

Exemple treballat

Resultats de la prova

  • Temps 400m: 6:08 (368 segons)
  • Temps 200m: 2:30 (150 segons)

Pas 1: calculeu el ritme CSS

CSS Ritme/100 m = (368 - 150) / 2
CSS Ritme/100m = 218/2
CSS Ritme/100m = 109 segons = 1:49 per 100m

Pas 2: Converteix a velocitat si cal

CSS = 100/109
CSS = 0,917 m/s

Exemple treballat: com utilitzar aquest resultat

Si el vostre CSS és 1:49/100m, les vostres repeticions de llindar s'han d'agrupar a prop d'aquest ritme, el vostre taula de zones s'hauria de construir a partir d'ell, i el vostre sTSS càlculs l'hauria d'utilitzar com a referència de ritme denominador.

Com utilitzar CSS a l'entrenament real

1️⃣ Conjunts de llindar de ritme

Utilitzeu CSS per deixar d'endevinar. Conjunts com 8x100, 5x200 o 3x400 tenen més sentit quan coneixeu l'objectiu de ritme i podeu mantenir-lo net.

2️⃣ Connecta Pace a sTSS

CSS dóna sentit al teu sTSS. Les sessions dures i les sessions fàcils deixen de combinar-se amb la mateixa càrrega de treball vaga.

3️⃣ Seguiu millor el progrés

Si el CSS és més ràpid mentre les vostres sessions es mantenen controlades, és probable que millori la vostra aptitud llindar de natació.

4️⃣ Crea millors hàbits de ritme

CSS t'ensenya a nedar el treball repetit amb control en lloc de convertir cada conjunt dur en un col·lapse de ritme.

Exemples d'idees de sessió orientades a CSS

  • Manteniment del llindar: 8x100 a ritme CSS amb un breu descans
  • Durabilitat del ritme: 5x200 al voltant del ritme CSS amb divisions consistents
  • Suport aeròbic: 3x400 lleugerament més lent que el ritme CSS amb recompte de cops controlat
  • Tècnica + ritme: alterna 100 m a l'esforç CSS amb 50 m de fàcil reinici tècnic

Aquests són exemples, no receptes universals. La qüestió és que CSS us ofereix un llenguatge de ritme estable.

CSS, SWOLF i eficiència

CSS t'ho diu que ràpid pots nedar al llindar. SWOLF i mètriques d'eficiència dir-te una cosa diferent: com t'estàs movent econòmicament per l'aigua. Es complementen, però no són intercanviables.

No compareu SWOLF a cegues

Un SWOLF inferior no sempre és "millor" entre nedadors, sessions, braçades o configuracions de piscina. Ús mètriques d'eficiència com a context juntament amb CSS, qualitat de traç i repetibilitat, no com a eina de classificació autònoma.

Validació científica

Wakayoshi et al. i el cas pràctic de CSS

CSS es va fer servir àmpliament perquè oferia als entrenadors i nedadors un punt de referència amigable amb el camp que feia un bon seguiment amb marcadors de rendiment relacionats amb el llindar sense requerir proves repetides de lactat de laboratori.

  • Pràctic: repetible a partir de proves simples de piscina
  • Útil: prou a prop del ritme rellevant per al llindar per a la programació
  • Actuable: fàcil de tornar a provar i actualitzar durant una temporada
  • Connectat: directament útil per a zones d'entrenament, ritme i seguiment de càrrega

Preguntes freqüents

Amb quina freqüència he de tornar a provar CSS?

Cada 6-8 setmanes és un bon valor predeterminat. Torna a provar més aviat només si l'entrenament ha canviat molt o si les teves zones actuals se senten malament.

És el mateix CSS que el llindar de lactat?

No exactament. CSS és una estimació de camp que s'utilitza com a referència pràctica del ritme de llindar. És útil perquè és repetible i accionable, no perquè substitueixi perfectament les proves de laboratori.

Puc utilitzar CSS per al ritme en aigües obertes?

Utilitzeu-lo com a referència inicial, no com a garantia literal. L'observació, el dibuix, el tall, els vestits de neoprè i la navegació canvien el comportament del ritme en aigües obertes.

Els principiants poden utilitzar CSS?

Sí, però el resultat es fa més útil a mesura que milloren la tècnica i el ritme. Els nedadors molt nous sovint necessiten consistència de braç i ritme abans que el CSS es torni altament estable.

He de comparar el meu CSS directament amb el d'una altra persona?

Només amb precaució. La longitud de la piscina, els impulsos, l'arrossegament, l'habilitat de ritme i la qualitat de la tècnica afecten el resultat. CSS és més potent com a referència personal al llarg del temps.

Utilitzeu CSS com a centre del clúster de mètriques de natació

Si CSS és el vostre àncora de ritme, les pàgines següents a utilitzar són:

  • Zones d'entrenament per traduir CSS en intenció de sessió
  • sTSS calculadora per quantificar l'estrès
  • Càrrega d'entrenament per controlar la forma física, la fatiga i la forma
  • Eficiència i SWOLF per afegir context a la tècnica
