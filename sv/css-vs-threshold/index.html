Kritisk simhastighet (CSS) kontra tröskeltempo: vilken spelar roll?

I flera år var "Threshold Pace" (T-Pace) guldstandarden. År 2026 omdefinierar övergången till Critical Swim Speed ​​(CSS) hur idrottare tar fart på sina pass och förutsäger trötthet under tävlingsdagen.

🎯 Det snabba svaret

T-Pace (Tröskeltempo) är en allmän uppskattning av det tempo du kan hålla i en timme, ofta används för att ställa in breda träningszoner. Det är ett "praktiskt" men mindre granulärt mått.

CSS (Critical Swim Speed) är den "funktionella" tröskeln matematiskt härledd från två olika avstånd (t.ex. 400m och 200m). Det är den teoretiska takten du kan hålla utan att träffa "väggen". Om du är en uthållighetssimmare eller triathlet är din CSS det viktigaste numret i ditt träningsprogram.

2026 års skift: från ansträngning till fysiologi

Att hantera trötthet i poolen är notoriskt svårt jämfört med cykling eller löpning. Utan en effektmätare är tempot vår enda proxy för intensitet. Medan många simmare fortfarande förlitar sig på den gamla skolans 1500 m "time-trial"-takt, har modern coachning flyttat till Tvådistans CSS-modell.

Den största skillnaden är Precision vs. upplevd ansträngning. En T-Pace baserad på en enda lång ansträngning fångar ofta din mentala grus mer än din metaboliska tröskel. CSS, beräknat från två distinkta punkter på din "Pace-Duration Curve", avslöjar den faktiska aeroba gränsen för din motor.

Metrisk jämförelse

Tröskeltempo (T-Pace)

Det "enkla" måttet

T-Pace härleds vanligtvis från en enda 1500m eller 1000m maximal ansträngning. Det representerar din "Redline" för ett kontinuerligt simmande.

  • Fördelar: Enkel att förstå. Ett test, ett nummer. Perfekt för långa, kontinuerliga set.
  • Nackdelar: Mycket känslig för pacingfel. Redogör inte för hur snabbt du bleknar under trötthet. Behandlar varje simmare med samma 1500m-tempo som att ha samma fysiologiska profil.

Kritisk simhastighet (CSS)

Det "funktionella" måttet

CSS är en matematisk härledning baserad på två avstånd (vanligtvis en 400m och en 200m maximal ansträngning).

  • Fördelar: Tar bort pacing bias. Bättre förutsäger ditt beteende i intervaller. Ger ett objektivt "aerobt tak."
  • Nackdelar: Kräver två separata tester. Lite mer komplex matematik (autoberäknat i Swim Analytics). Kräver att testerna görs med konsekvent vila.

Varför deltat är viktigt

"Delta" är skillnaden mellan ditt 400m tempo och ditt 200m tempo. Två simmare kan båda ha en T-Pace på 1:40/100m, men deras CSS-profiler kan se ut så här:

Simmare A ("Diesel")

Litet delta mellan 200m och 400m. Denna simmare har en mycket hög aerob effektivitet men lägre sprinthastighet. Deras CSS ligger väldigt nära deras T-Pace.

Simmare B ("Sprintaren")

Stort delta. Bra 200m fart men tappar avsevärt på 400m. Deras CSS kommer att vara lägre än vad deras T-Pace föreslår. Om de försöker träna i en generisk "T-Pace" kommer de hela tiden att överkoka sitt anaeroba system.

📈 CSS-formeln

CSS = (D1 - D2) / (T1 - T2)

Där D1/D2 är de två avstånden och T1/T2 är tiderna i sekunder. Detta avslöjar "lutningen" för ditt hållbara tempo utan bruset från din första anaeroba våg.

Hur man utför CSS-testet (steg-för-steg)

För att beräkna din CSS måste du utföra två maximala försök på olika dagar (eller med full återhämtning däremellan) i en standardpool:

  1. Uppvärmning: Utför en grundlig, standardiserad uppvärmning (vanligtvis 400m till 600m blandade slag och kroppsbyggnader) för att förbereda din kropp.
  2. Försök 1 (400 m maximal ansträngning): Simma 400m så fort som möjligt. Anteckna den exakta tiden. Tempera dig själv så att du inte bränner dig under de första 100 m.
  3. Aktiv återställning: Simma 100m till 200m mycket lätt för att spola ur mjölksyra, följt av 5–10 minuters vila.
  4. Försök 2 (200 m maximal ansträngning): Simma 200m vid absolut maximal ansträngning. Anteckna den exakta tiden.
  5. Beräkna: Konvertera dina tider till sekunder och använd formeln ovan, eller använd CSS-kalkylatorverktyget i Swim Analytics för att få din 100m-tempo.

Vilket ska du använda?

Använd tröskeltempo (T-Pace) om:

  • Du har precis börjat med tempobaserad träning.
  • Du utför bara långa, kontinuerliga simningar (Öppet vatten 3km+).
  • Du vill inte ta itu med flera testsessioner.

Använd Critical Swim Speed (CSS) om:

  • Triathleter och marathonsimmare: För att undvika att "bränna ljuset" för tidigt i ett lopp.
  • Intervallträning: För att ställa in specifika CSS + 2s eller CSS - 1s zoner som faktiskt fungerar.
  • Spåra framsteg: CSS är känsligare för aeroba förbättringar än ett generiskt 1000m-test.

Vanliga frågor om tröskeltempo vs CSS

Är CSS detsamma som tröskeltempo?

Nej. De är relaterade tröskelbegrepp, men de är inte samma test. Tröskeltempo kommer vanligtvis från en längre simning, medan CSS beräknas från två försök och ger ett mer repeterbart ankare för intervallarbete.

Vilket är bättre för triathlon och pacing i öppet vatten?

CSS är vanligtvis den bättre utgångspunkten eftersom det återspeglar hållbart tempo utan att överbelöna kortsiktig hastighet. Tröskeltempo kan fortfarande vara användbart för långa jämna simningar, men CSS är vanligtvis lättare att omvandla till pacingmål och zonarbete.

Hur går jag från tröskeltempo till CSS-baserade simzoner?

Använd ditt nästa 400m och 200m-test för att ställa in CSS, och ankra sedan intervallet runt det numret. Om du redan tränar med T-Pace, jämför båda siffrorna och använd CSS-guiden plus simträningszoner för att bygga om dina set runt repeterbart aerobt tempo.
