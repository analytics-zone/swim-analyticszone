Viteza critică de înot (CSS) vs. Ritmul pragului: care contează?

Ani de zile, „Threshold Pace” (T-Pace) a fost standardul de aur. În 2026, trecerea la Viteza critică de înot (CSS) redefinește modul în care sportivii își ritmează sesiunile și prevăd oboseala din ziua cursei.

🎯 Răspunsul rapid

T-Pace (ritmul de prag) este o estimare generală a ritmului pe care îl puteți menține timp de o oră, adesea folosită pentru stabilirea unor zone largi de antrenament. Este o metrică „practică”, dar mai puțin granulară.

CSS (viteză critică de înot) este pragul „funcțional” derivat matematic de la două distanțe diferite (de exemplu, 400m și 200m). Este ritmul teoretic pe care îl poți susține fără a lovi „Peretele”. Dacă ești înotător de anduranță sau triatlet, CSS-ul tău este cel mai important număr din programul tău de antrenament.

Schimbarea din 2026: de la efort la fiziologie

Gestionarea oboselii din piscină este notoriu de dificilă în comparație cu mersul cu bicicleta sau alergatul. Fără un contor de putere, ritmul este singurul nostru indicator pentru intensitate. În timp ce mulți înotători se bazează în continuare pe ritmuri de „contra cronometru” de 1500 de metri, antrenamentul modern a trecut la Model CSS pe două distanțe.

Principala diferență este Precizie vs. efort perceput. Un T-Pace bazat pe un singur efort lung captează adesea grija mentală mai mult decât pragul metabolic. CSS, calculat din două puncte distincte de pe „Curba ritm-durată”, dezvăluie limita aerobă reală a motorului dumneavoastră.

Comparație metrică

Ritmul prag (T-Pace)

Metrica „simplu”.

T-Pace este de obicei derivat dintr-un singur efort maxim de 1500m sau 1000m. Reprezintă „linia roșie” pentru o înot continuă.

  • Pro: Simplu de înțeles. Un test, un număr. Excelent pentru seturi lungi și continue.
  • Contra: Foarte sensibil la erorile de ritm. Nu ține cont de cât de repede te estompezi sub oboseală. Tratează fiecare înotător cu același ritm de 1500 m ca având același profil fiziologic.

Viteza critică de înot (CSS)

Metrica „Funcțională”.

CSS este o derivație matematică bazată pe două distanțe (de obicei, un efort maxim de 400 m și 200 m).

  • Pro: Îndepărtează distorsiunea de ritm. Îți prezice mai bine comportamentul la intervale. Oferă un „tavan aerobic” obiectiv.
  • Contra: Necesită două teste separate. Matematică puțin mai complexă (calculată automat în Swim Analytics). Necesită ca testele să fie făcute cu odihnă consistentă.

De ce contează Delta

„Delta” este diferența dintre ritmul tău de 400 m și ritmul tău de 200 m. Doi înotători ar putea avea ambii un ritm T de 1:40/100m, dar profilurile lor CSS ar putea arăta astfel:

Înotatorul A ("Dieselul")

Mică deltă între 200m și 400m. Acest înotător are o eficiență aerobă foarte mare, dar o viteză de sprint mai mică. CSS-ul lor este foarte aproape de T-Pace.

Înotatorul B ("Sprinterul")

Deltă mare. Viteză mare de 200 m, dar scade semnificativ pe cei 400 m. CSS-ul lor va fi mai mic decât sugerează T-Pace. Dacă încearcă să se antreneze la un „T-Pace” generic, își vor supragăti constant sistemul anaerob.

📈 Formula CSS

CSS = (D1 - D2) / (T1 - T2)

Unde D1/D2 sunt cele două distanțe și T1/T2 sunt timpii în secunde. Acest lucru dezvăluie „panta” ritmului tău sustenabil fără zgomotul valului tău anaerob inițial.

Cum se efectuează testul CSS (pas cu pas)

Pentru a calcula CSS, trebuie să efectuați două încercări maxime în zile diferite (sau cu recuperare completă între ele) într-un grup standard:

  1. Încălzire: Efectuați o încălzire amănunțită, standardizată (de obicei, 400 m până la 600 m de mișcări mixte și construcție) pentru a vă pregăti corpul.
  2. Proba 1 (efort maxim de 400 m): Înotați 400 m cât mai repede posibil. Înregistrați ora exactă. Rămâneți-vă ca să nu vă epuizați în primii 100 de metri.
  3. Recuperare activă: Înotați 100 m până la 200 m foarte ușor de îndepărtat acidul lactic, urmat de 5-10 minute de odihnă.
  4. Proba 2 (efort maxim de 200 m): Înotați 200 m la efort maxim absolut. Înregistrați ora exactă.
  5. Calculați: Convertiți-vă timpii în secunde și aplicați formula de mai sus sau utilizați instrumentul de calcul CSS din Swim Analytics pentru a obține ritmul de 100 m.

Pe care ar trebui să utilizați?

Utilizați ritmul de prag (T-Pace) dacă:

  • Abia ai început cu antrenamentul bazat pe ritm.
  • Efectuați doar înotări lungi și continue (în apă deschisă 3 km+).
  • Nu doriți să aveți de-a face cu mai multe sesiuni de testare.

Utilizați viteza critică de înot (CSS) dacă:

  • Triatleți și înotători de maraton: Pentru a evita „aprinderea lumânării” prea devreme într-o cursă.
  • Antrenament pe intervale: Pentru a seta anumite zone CSS + 2s sau CSS - 1s care funcționează efectiv.
  • Urmărirea progresului: CSS este mai sensibil la îmbunătățirile aerobe decât un test generic de 1000 m.

Întrebări frecvente privind ritmul pragului vs CSS

Este CSS la fel cu ritmul de prag?

Nu. Sunt concepte legate de prag, dar nu sunt același test. Ritmul de prag provine de obicei dintr-o înot mai lungă, în timp ce CSS este calculat din două eforturi și oferă o ancora mai repetabilă pentru lucrul pe interval.

Care este mai bun pentru triatlon și ritmul în apă deschisă?

CSS este, de obicei, cel mai bun punct de plecare, deoarece reflectă ritmul durabil, fără a recompensa excesiv viteza pe termen scurt. Ritmul prag poate fi în continuare util pentru înotări lungi și constante, dar CSS este de obicei mai ușor de transformat în ținte de ritm și lucru în zonă.

Cum trec de la ritmul prag la zonele de înot bazate pe CSS?

Utilizați următorul test de 400m și 200m pentru a seta CSS, apoi ancorați lucrul cu intervale în jurul numărului respectiv. Dacă vă antrenați deja cu T-Pace, comparați ambele numere și utilizați ghidul CSS plus zonele de antrenament de înot pentru a vă reconstrui seturile în jurul unui ritm aerobic repetabil.
