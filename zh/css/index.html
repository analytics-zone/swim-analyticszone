临界游泳速度 (CSS)

确定游泳训练区域、阈值设置和训练负荷分析的实用配速基准

快速解答

临界游泳速度 (CSS) 是对您的可持续阈值游泳速度的现场测试估计。大多数游泳运动员通过以下方式计算： 400m + 200m 计时赛，然后使用结果来设置 训练区、步速阈值设置和功率 sTSS 和 训练负荷 分析。

  • 最佳用途： 阈值训练、可重复起搏和负荷跟踪
  • 典型解读： 大致是您在有氧运动中能够维持艰苦、受控工作的最快配速
  • 不要盲目使用： 节奏不佳、恢复不完全、不同的池条件或阻力辅助装置可能会扭曲结果
  • 重新测试节奏： 每 6-8 周一次，或在一次有意义的训练之后

要点

  • 什么是CSS： 通过简单的现场测试构建的实用的游泳运动员阈值速度估计
  • 为什么它很重要： CSS 将游泳训练从猜测转变为可重复的节奏和更好的负荷控制
  • 它解锁什么：区域, sTSS, 绩效管理，以及更清晰的进度检查
  • 它不做什么： 取代技术分析、开放水域环境，或 效率解读
  • 最好的下一步： 很好地测试 CSS，然后用它来构建你的阈值和有氧运动

如果您只保留一项游泳指标作为节奏锚，请保留 CSS。它并不完美，也不是完整的生理实验室测试，但它实用、可重复且直接有用。这种结合就是为什么 CSS 仍然是数据驱动游泳训练的支柱。

对于大多数游泳者来说， 临界游泳速度 是原始计时赛和日常训练决策之间的桥梁。它有助于回答五个重要问题：

  • 阈值设置实际上应该有多快？
  • 每个配速范围属于什么范围 游泳区?
  • 我应该如何配速重复 100 秒、200 秒和 400 秒？
  • 如何量化游泳压力 sTSS?
  • 我的有氧游泳健身是否朝着正确的方向发展？

CSS 衡量什么

CSS 是对有氧游泳配速的实用估计，通常来自 400m 和 200m 计时赛。在教练语言中，它通常被视为结构化泳池训练的阈值配速：足够快，足够重要，足够控制，可以重复，足够稳定，可以在周围建立区域。

方面实用答案
CSS 衡量什么您进行持续有氧运动的现场估计阈值游泳速度
最佳使用设置训练区域、阈值设置、配速目标和 训练负荷
常见错误将一项糟糕的测试视为永久事实，或使用 CSS 而不考虑泳池长度、休息和配速质量
相关指标训练区, sTSS, CTL/ATL/TSB， 和 SWOLF/效率

🎯 有用的心理模型

将 CSS 视为您的 运营速度基准，不是一个神奇的数字。配速参考使您的其余游泳指标保持一致。

为什么 CSS 很重要

CSS 很重要，因为它将生理学与执行力联系起来。许多游泳运动员知道他们做了“硬组”或“节奏训练”，但不能说速度是否正确。 CSS 通过为您提供一个可以在一个又一个会话中使用的基准来解决这个问题。

🏊 训练区

CSS 为您提供了一个干净的锚点 游泳区，因此简单的工作保持简单，阈值工作保持有针对性。

📊 训练负荷

如果没有可用的 CSS， sTSS 和 绩效管理指标 失去意义。

⏱️ 可重复的起搏

CSS 可以帮助您更加一致地配速 100 秒、200 秒和 400 秒，而不是仅仅依靠感觉。

📈 进度检查

当 CSS 改进时，你的阈值速度就会变得更快。这是一个强烈的信号，表明有氧游泳的体能也在改善。

⚠️ 关键依赖： 如果您的 CSS 错误，您的区域、sTSS 和绩效管理数字将会朝错误的方向漂移。错误的输入会产生错误的分析。

CSS 的优点是什么，缺点是什么

CSS 非常适合： 阈值设置、有氧质量工作、区域设置和游泳特定负荷跟踪。

CSS 不足以用于： 冲刺分析、划水技术诊断、可变条件下的开放水域比赛执行或单独解释效率。

使用 CSS 用于

  • 阈值配速处方
  • 重复 100/200/400 步速
  • 个性化 区域结构
  • sTSS 和 负载管理
  • 跟踪整个赛季的有氧运动进展

不要单独使用 CSS

  • 比较不同的池长度，就好像它们是相同的一样
  • 比较具有不同技术特征的游泳运动员
  • 判断笔划质量无需 行程力学
  • 在没有上下文的情况下判断效率 狼
  • 假设开放水域配速与泳池 CSS 配速完全匹配

如何通过 4 个实用步骤测试 CSS

📋 标准协议

  1. 适当热身

    轻松游 300-800 米，包括练习，并在测试前添加一些受控的构建工作。

  2. 参加 400m 计时赛

    使用推动启动，而不是潜水。我们的目标是最快可持续 400 米，而不是鲁莽地跑完前 100 米。

  3. 完全康复

    需要5-10分钟 轻松或完全休息。不完全恢复是破坏结果最快的方法之一。

  4. 参加 200m 计时赛

    再次使用推动启动并精确记录时间。您的 200m 配速应该明显快于您的 400m 配速。

⚠️ 最常见的测试错误

400m配速不佳

问题： 你出去太努力了，然后就褪色得很厉害。

结果： 400m 站反映了可持续配速质量。

使固定： 目标是均匀或轻微负分裂。

恢复不足

问题： 疲劳使200m人为减慢。

结果： CSS 变得扭曲，你的训练区域会发生漂移。

使固定： 等到呼吸得到控制并且感觉完全康复。

混合条件

问题： 不同的水池长度、较大的推力变化或阻力辅助装置都会改变测试环境。

结果： 您比较的数字实际上并没有可比性。

使固定： 在每次测试中尽可能保持条件一致。

🔄 重新测试频率

每隔一段时间重新测试 CSS 6-8周，或在专注于阈值、耐力或技术巩固的有意义的区块之后。重点不是不断的测试。重点是始终如一地使用一个基准来观察变化。

CSS计算公式

公式

CSS (m/s) = (D2 - D1) / (T2 - T1)

在哪里：

  • D₁ = 200 米
  • D2 = 400 米
  • T₁ = 200m 的时间（以秒为单位）
  • T2 = 400m 的时间（以秒为单位）

简化为每 100m 配速

CSS 步速/100m（秒）= (T₄₀₀ - T2₀₀) / 2

工作示例

测试结果

  • 400m时间： 6:08 （368 秒）
  • 200m时间： 2:30 （150 秒）

第 1 步：计算 CSS 速度

CSS 步速/100m = (368 - 150) / 2
CSS 步速/100m = 218 / 2
CSS 步速/100m = 109 秒 = 每 100m 1:49

第 2 步：根据需要转换为速度

CSS = 100 / 109
CSS = 0.917米/秒

工作示例：如何使用该结果

如果你的 CSS 是 1:49/100米，你的阈值重复应该聚集在那个速度附近，你的 区域表 应该从它构建，并且你的 sTSS计算 应将其用作分母速度参考。

如何在实际训练中使用 CSS

1️⃣ 步速阈值设置

使用 CSS 来停止猜测。当您知道配速目标并能干净利落地保持住时，8x100、5x200 或 3x400 等组数会更有意义。

2️⃣ 将 Pace 连接至sTSS

CSS 赋予你的意义 sTSS。艰苦的训练和简单的训练不再混入同样模糊的工作量中。

3️⃣ 更好地跟踪进度

如果 CSS 变得更快，同时您的训练保持可控，您的阈值游泳健康可能会有所改善。

4️⃣ 养成更好的节奏习惯

CSS 教你如何在有控制的情况下进行重复训练，而不是将每组高强度训练都变成节奏崩溃。

面向 CSS 的会话创意示例

  • 阈值维护： 8x100 以 CSS 配速短时间休息
  • 步速耐久性： 5x200 左右的 CSS 配速，并保持一致的分割
  • 有氧支持： 3x400 比 CSS 速度稍慢，且笔划数受控制
  • 技术+节奏： 交替 100m CSS 努力和 50m 轻松技术重置

这些只是示例，并非通用处方。重点是 CSS 为您提供了一种稳定节奏的语言。

CSS、SWOLF 和效率

CSS告诉你 多快 你可以在门槛处游泳。 SWOLF 和效率指标 告诉您一些不同的事情：您在水中移动的经济性如何。它们相互补充，但不可互换。

不要盲目比较SWOLF

对于游泳者、训练、泳姿或泳池设置来说，较低的 SWOLF 并不总是“更好”。使用 效率指标 作为 CSS、笔画质量和可重复性的上下文，而不是作为独立的排名工具。

常见问题解答

我应该多久重新测试一次 CSS？

每 6-8 周是一个很好的默认值。仅当训练发生很大变化或您当前的区域明显感觉不对时才应尽早重新测试。

CSS 与乳酸阈值相同吗？

不完全是。 CSS 是一种现场估计，用作实用的阈值速度基准。它很有用，因为它是可重复的和可操作的，而不是因为它完美地取代了实验室测试。

我可以使用 CSS 进行开放水域配速吗？

将其用作起始参考，而不是字面保证。瞄准、吃水、劈砍、潜水服和导航都会改变开放水域中的配速行为。

初学者可以使用CSS吗？

是的，但随着技术和节奏的提高，结果会变得更有用。游泳新手通常需要在 CSS 变得高度稳定之前保持划水和步速的一致性。

我应该直接将我的 CSS 与其他人的 CSS 进行比较吗？

只能小心翼翼。泳池长度、推球、阻力、节奏技巧和技术质量都会影响结果。随着时间的推移，CSS 作为个人基准是最强大的。
