临界游泳速度（CSS）

数据驱动游泳训练的基础

什么是临界游泳速度（CSS）？

临界游泳速度（CSS）是您可以在不力竭的情况下保持的理论最大游泳速度。它代表您的有氧阈值配速，通常对应于 4 mmol/L 血乳酸，可持续约 30 分钟。CSS 使用 400 米和 200 米计时测试来确定个性化的训练区间。

临界游泳速度（CSS）代表您可以持续保持而不会力竭的理论最大游泳速度。这是您在水中的有氧阈值——乳酸产生等于乳酸清除的强度。

🎯 生理意义

CSS 密切对应于：

  • 乳酸阈值 2（LT2） - 第二通气阈值
  • 最大乳酸稳态（MLSS） - 最高可持续乳酸水平
  • 功能阈值配速（FTP） - 游泳相当于自行车 FTP
  • ~4 mmol/L 血乳酸 - 传统 OBLA 标记

为什么 CSS 很重要

CSS 是解锁所有高级训练负荷分析的基础指标：

  • 训练区间：根据您的生理特征个性化强度区间
  • sTSS 计算：实现准确的训练压力评分量化
  • CTL/ATL/TSB：表现管理图表指标所需
  • 进度追踪：有氧体能改善的客观衡量
⚠️ 关键依赖：没有有效的 CSS 测试，无法计算高级训练负荷指标（sTSS、CTL、ATL、TSB）。不准确的 CSS 将破坏所有后续的训练分析。

CSS 测试协议

📋 标准协议

  1. 热身

    300-800 米轻松游泳、技术练习和渐进式加速，为最大努力做准备。

  2. 400 米计时测试

    从蹬壁出发（不跳水）的最大持续努力。精确记录时间。目标：最快的可持续 400 米。

  3. 完全恢复

    5-10 分钟的轻松游泳或完全休息。这对准确结果至关重要。

  4. 200 米计时测试

    从蹬壁出发的最大努力。精确记录时间。每 100 米应该比 400 米更快。

⚠️ 常见错误

恢复不足

问题：疲劳人为地减慢 200 米时间

结果：计算的 CSS 变得比实际更快，导致过度训练的区间

解决方案：休息直到心率降至 120 bpm 以下或呼吸完全恢复

400 米配速不当

问题：起步太快导致后程大幅减速

结果：400 米时间不能反映真实的可持续配速

解决方案：目标是匀速或负分段（后 200 米 ≤ 前 200 米）

使用跳水出发

问题：增加约 0.5-1.5 秒的优势，使计算产生偏差

解决方案：始终使用蹬壁出发

🔄 重测频率

每 6-8 周重测一次 CSS，以随着体能提高更新训练区间。随着您适应训练，您的区间应逐渐变快。

CSS 计算公式

公式

CSS (m/s) = (D₂ - D₁) / (T₂ - T₁)

其中：

  • D₁ = 200 米
  • D₂ = 400 米
  • T₁ = 200 米时间（秒）
  • T₂ = 400 米时间（秒）

简化为每 100 米配速

CSS 配速/100m (秒) = (T₄₀₀ - T₂₀₀) / 2

计算示例

测试结果：

  • 400 米时间：6:08 (368 秒)
  • 200 米时间：2:30 (150 秒)

第一步：计算 CSS (m/s)

CSS = (400 - 200) / (368 - 150)
CSS = 200 / 218
CSS = 0.917 m/s

第二步：转换为每 100 米配速

配速 = 100 / 0.917
配速 = 109 秒
配速 = 1:49 / 100m

基于 CSS 的训练区间

注意：在游泳中，配速以每距离时间测量。因此，更高的百分比 = 更慢的配速，而更低的百分比 = 更快的配速。这与自行车/跑步相反，在那里更高的百分比 = 更难的努力。

🎯 区域训练的好处

使用基于 CSS 的区间将主观的“感觉”训练转化为客观、可重复的训练。每个区间针对特定的生理适应：

  • 区间 2：建立有氧引擎（每周训练量的 60-70%）
  • 区间 3：提高比赛配速效率（训练量的 15-20%）
  • 区间 4：推高乳酸阈值（训练量的 10-15%）
  • 区间 5：发展最高速度和力量（训练量的 5-10%）

科学验证

Wakayoshi 等 (1992-1993) - 基础研究

Kohji Wakayoshi 在大阪大学的开创性研究确立了 CSS 作为实验室乳酸测试的有效、实用替代方案：

  • 与无氧阈值时的 VO₂ 强相关 (r = 0.818)
  • 与 OBLA 速度极佳相关 (r = 0.949)
  • 预测 400 米表现 (r = 0.864)
  • 对应于 4 mmol/L 血乳酸 - 最大乳酸稳态
  • 距离与时间之间的线性关系 (r² > 0.998)

关键论文：

  1. Wakayoshi K, et al. (1992). "Determination and validity of critical velocity as an index of swimming performance in the competitive swimmer." European Journal of Applied Physiology, 64(2), 153-157.
  2. Wakayoshi K, et al. (1992). "A simple method for determining critical speed as swimming fatigue threshold in competitive swimming." International Journal of Sports Medicine, 13(5), 367-371.
  3. Wakayoshi K, et al. (1993). "Does critical swimming velocity represent exercise intensity at maximal lactate steady state?" European Journal of Applied Physiology, 66(1), 90-95.

🔬 为什么 CSS 有效

CSS 代表了重度运动和极重度运动领域之间的界限。在 CSS 以下，乳酸产生和清除保持平衡——您可以长时间游泳。在 CSS 以上，乳酸会逐渐积累，直到在 20-40 分钟内力竭。

这使得 CSS 成为以下方面的完美强度：

  • 设定 800 米-1500 米项目的可持续比赛配速
  • 制定阈值间歇训练
  • 监测有氧体能的改善
  • 计算训练负荷和恢复需求

如何测试您的 CSS

第一步：充分热身

完成 300-800 米的轻松游泳，包括技术练习和渐进式加速。这为最大努力做好身体准备并防止受伤。

第二步：进行 400 米计时测试

从蹬壁出发（不跳水）以最大持续努力游 400 米。精确记录到秒。目标是匀速或负分段（后 200 米等于或快于前 200 米）。

第三步：完全恢复

进行 5-10 分钟的轻松游泳或完全休息。这种恢复至关重要。等到心率降至 120 bpm 以下且呼吸完全恢复后再继续。

第四步：进行 200 米计时测试

从蹬壁出发以最大努力游 200 米。精确记录时间。这应该比您的 400 米配速每 100 米更快。

第五步：计算您的 CSS

使用公式：CSS 配速/100m = (T400 - T200) / 2。例如：(368s - 150s) / 2 = 109s = 1:49/100m。或者使用本页面的计算器立即获得带有个性化训练区间的结果。

实际应用

1️⃣ 解锁训练负荷指标

CSS 是 sTSS 强度系数计算的分母。没有它，您无法量化训练压力或追踪体能/疲劳趋势。

2️⃣ 个性化训练区间

通用配速表不考虑个人生理差异。基于 CSS 的区间确保每位游泳者都在其最佳强度下训练。

3️⃣ 监测体能进展

每 6-8 周重测一次。CSS 提高（配速变快）表明有氧适应成功。CSS 停滞表明训练需要调整。

4️⃣ 预测比赛表现

CSS 配速近似于您可持续的 30 分钟比赛配速。用它来为 800 米、1500 米和公开水域项目设定现实目标。

5️⃣ 设计阈值训练

经典 CSS 组：8×100 @ CSS 配速（休息 15 秒），5×200 @ 101% CSS（休息 20 秒），3×400 @ 103% CSS（休息 30 秒）。建立乳酸清除能力。

6️⃣ 优化减量策略

在减量前后追踪 CSS。成功的减量应维持或略微提高 CSS，同时减少疲劳（提高 TSB）。
