ល្បឿនហែលទឹកសំខាន់ (CSS) ទល់នឹងកម្រិតកម្រិត៖ តើមួយណាសំខាន់?

អស់រយៈពេលជាច្រើនឆ្នាំ "Threshold Pace" (T-Pace) គឺជាស្តង់ដារមាស។ នៅឆ្នាំ 2026 ការផ្លាស់ប្តូរទៅ Critical Swim Speed ​​(CSS) កំពុងកំណត់ឡើងវិញពីរបៀបដែលអត្តពលិកបង្កើនល្បឿនវគ្គរបស់ពួកគេ និងព្យាករណ៍ពីភាពអស់កម្លាំងនៅថ្ងៃប្រណាំង។

🎯 ចម្លើយរហ័ស

T-Pace (កម្រិតល្បឿន) គឺជាការប៉ាន់ប្រមាណទូទៅនៃល្បឿនដែលអ្នកអាចរក្សាបានរយៈពេលមួយម៉ោង ដែលជារឿយៗត្រូវបានប្រើសម្រាប់ការកំណត់តំបន់ហ្វឹកហាត់ទូលំទូលាយ។ វាគឺជា "ជាក់ស្តែង" ប៉ុន្តែជាម៉ែត្រក្រឡាតិចជាង។

CSS (ល្បឿនហែលដ៏សំខាន់) គឺជាកម្រិត "មុខងារ" ដែលទទួលបានតាមគណិតវិទ្យាពីចម្ងាយពីរផ្សេងគ្នា (ឧទាហរណ៍ 400m និង 200m)។ វា​គឺ​ជា​ល្បឿន​ទ្រឹស្ដី​ដែល​អ្នក​អាច​ទ្រទ្រង់​ដោយ​មិន​បាច់​ប៉ះ​នឹង "ជញ្ជាំង"។ ប្រសិនបើអ្នកជាអ្នកហែលទឹក ឬកីឡាទ្រីយ៉ាត្លុងដែលមានភាពអត់ធ្មត់ CSS របស់អ្នកគឺជាលេខដ៏សំខាន់បំផុតនៅក្នុងកម្មវិធីហ្វឹកហ្វឺនរបស់អ្នក។

ការផ្លាស់ប្តូរឆ្នាំ 2026: ពីកិច្ចខិតខំប្រឹងប្រែងទៅសរីរវិទ្យា

ការ​គ្រប់គ្រង​ភាព​នឿយហត់​ក្នុង​អាង​ទឹក​គឺ​ជា​ការ​ពិបាក​គួរ​ឱ្យ​កត់​សម្គាល់​បើ​ធៀប​នឹង​ការ​ជិះ​កង់ ឬ​ការ​រត់។ បើគ្មានឧបករណ៍វាស់ថាមពលទេ ល្បឿនគឺជាប្រូកស៊ីតែមួយគត់របស់យើងសម្រាប់អាំងតង់ស៊ីតេ។ ខណៈពេលដែលអ្នកហែលទឹកជាច្រើននៅតែពឹងផ្អែកលើល្បឿន 1500 ម៉ែត្រ "សាកល្បងពេលវេលា" របស់សាលាចាស់ ការហ្វឹកហាត់ទំនើបបានផ្លាស់ប្តូរទៅ គំរូ CSS ពីរចម្ងាយ.

ភាពខុសគ្នាសំខាន់គឺ ភាពជាក់លាក់ធៀបនឹងការខិតខំប្រឹងប្រែងដែលយល់ឃើញ. T-Pace ផ្អែកលើការខិតខំប្រឹងប្រែងដ៏យូរមួយ ជារឿយៗចាប់យកអារម្មណ៍ផ្លូវចិត្តរបស់អ្នកច្រើនជាងកម្រិតនៃការរំលាយអាហាររបស់អ្នក។ CSS ដែលគណនាពីចំណុចពីរផ្សេងគ្នានៅលើ "Pace-Duration Curve" របស់អ្នកបង្ហាញពីដែនកំណត់ពិតប្រាកដនៃម៉ាស៊ីនរបស់អ្នក។

ការប្រៀបធៀបម៉ែត្រ

កម្រិតល្បឿន (T-Pace)

មាត្រដ្ឋាន "សាមញ្ញ"

T-Pace ជាធម្មតាបានមកពីការខិតខំប្រឹងប្រែងអតិបរមា 1500m ឬ 1000m។ វាតំណាងឱ្យ "បន្ទាត់ក្រហម" របស់អ្នកសម្រាប់ការហែលទឹកជាបន្តបន្ទាប់។

  • គុណសម្បត្តិ៖ ងាយយល់។ ការធ្វើតេស្តមួយ លេខមួយ។ ល្អសម្រាប់ឈុតបន្តបន្ទាប់។
  • គុណវិបត្តិ៖ មានភាពរសើបខ្លាំងចំពោះកំហុសក្នុងល្បឿន។ មិន​គិត​ថា​តើ​អ្នក​បាត់​បង់​ភាព​នឿយហត់​លឿន​ប៉ុណ្ណា​ទេ។ ចាត់​ទុក​អ្នក​ហែល​ទឹក​គ្រប់​រូប​ក្នុង​ល្បឿន 1500 ម៉ែត្រ​ដូច​គ្នា មាន​ទម្រង់​សរីរវិទ្យា​ដូច​គ្នា។

ល្បឿនហែលទឹកសំខាន់ (CSS)

សូចនាករ "មុខងារ"

CSS គឺ​ជា​ដេរីវេ​គណិតវិទ្យា​ដែល​ផ្អែក​លើ​ចម្ងាយ​ពីរ (ជា​ធម្មតា ៤០០ ម៉ែត្រ និង ២០០ ម៉ែត្រ)។

  • គុណសម្បត្តិ៖ លុបបំបាត់ភាពលំអៀងនៃល្បឿន។ ព្យាករណ៍ពីអាកប្បកិរិយារបស់អ្នកក្នុងចន្លោះពេលកាន់តែប្រសើរ។ ផ្តល់នូវគោលបំណង "ពិដាន aerobic" ។
  • គុណវិបត្តិ៖ ទាមទារការធ្វើតេស្តពីរដាច់ដោយឡែក។ គណិតវិទ្យាស្មុគស្មាញបន្តិច (គណនាដោយស្វ័យប្រវត្តិនៅក្នុង Swim Analytics)។ តម្រូវ​ឱ្យ​មាន​ការ​ធ្វើ​តេ​ស្ត​ដែល​ត្រូវ​បាន​ធ្វើ​ដោយ​ការ​សម្រាក​ជាប់​លាប់​។

ហេតុអ្វីបានជាតំបន់ដីសណ្តមានសារៈសំខាន់

"ដីសណ្ត" គឺជាភាពខុសគ្នារវាងល្បឿន 400 ម៉ែត្ររបស់អ្នក និង 200 ម៉ែត្ររបស់អ្នក។ អ្នកហែលទឹកពីរនាក់អាចមាន T-Pace 1:40/100m ប៉ុន្តែទម្រង់ CSS របស់ពួកគេអាចមើលទៅដូចនេះ៖

អ្នកហែលទឹក A ("The Diesel")

ដីសណ្តតូច ចន្លោះពី ២០០ម ទៅ ៤០០ម។ អ្នក​ហែល​ទឹក​នេះ​មាន​ប្រសិទ្ធ​ភាព​ខ្ពស់​ណាស់ ប៉ុន្តែ​មាន​ល្បឿន​រត់​ទាប​ជាង។ CSS របស់ពួកគេគឺនៅជិត T-Pace របស់ពួកគេ។

អ្នកហែលទឹក B ("The Sprinter")

ដីសណ្តធំ។ ល្បឿន 200m ដ៏អស្ចារ្យ ប៉ុន្តែធ្លាក់ចុះយ៉ាងខ្លាំងនៅលើ 400 ម៉ែត្រ។ CSS របស់ពួកគេនឹងទាបជាង T-Pace បានណែនាំ។ ប្រសិនបើពួកគេព្យាយាមហ្វឹកហាត់នៅ "T-Pace" ទូទៅនោះ ពួកគេនឹងធ្វើម្ហូបហួសពេលប្រព័ន្ធ anaerobic របស់ពួកគេ។

📈 រូបមន្ត CSS

CSS = (D1 - D2) / (T1 - T2)

ដែល D1/D2 ជាចម្ងាយពីរ ហើយ T1/T2 គឺជាដងគិតជាវិនាទី។ នេះបង្ហាញពី "ជម្រាល" នៃល្បឿនប្រកបដោយនិរន្តរភាពរបស់អ្នកដោយគ្មានសំលេងរំខាននៃការកើនឡើង anaerobic ដំបូងរបស់អ្នក។

របៀបធ្វើតេស្ត CSS (មួយជំហានម្តងៗ)

ដើម្បីគណនា CSS របស់អ្នក អ្នកត្រូវធ្វើការសាកល្បងអតិបរិមាចំនួនពីរនៅថ្ងៃផ្សេងៗគ្នា (ឬជាមួយនឹងការងើបឡើងវិញពេញលេញនៅចន្លោះ) នៅក្នុងក្រុមស្តង់ដារ៖

  1. កំដៅឡើង៖ ធ្វើការកម្តៅសាច់ដុំឱ្យបានហ្មត់ចត់ និងមានលក្ខណៈស្តង់ដារ (ជាធម្មតា 400m ទៅ 600m នៃការដាច់សរសៃឈាមខួរក្បាលចម្រុះ និងបង្កើត) ដើម្បីរៀបចំរាងកាយរបស់អ្នក។
  2. ការសាកល្បង 1 (400m អតិបរមាការខិតខំប្រឹងប្រែង): ហែល 400 ម៉ែត្រឱ្យលឿនតាមដែលអាចធ្វើទៅបាន។ កត់ត្រាពេលវេលាពិតប្រាកដ។ រុញខ្លួនអ្នកដើម្បីកុំឱ្យឆេះក្នុង 100 ម៉ែត្រដំបូង។
  3. ការងើបឡើងវិញសកម្ម៖ ហែលទឹកពី 100m ទៅ 200m ងាយស្រួលបំផុតដើម្បីបញ្ចេញអាស៊ីតឡាក់ទិក បន្ទាប់មកសម្រាក 5-10 នាទី។
  4. ការសាកល្បងទី 2 (ការប្រឹងប្រែងអតិបរមា 200 ម)៖ ហែល 200 ម៉ែត្រដោយការខិតខំប្រឹងប្រែងអតិបរមាដាច់ខាត។ កត់ត្រាពេលវេលាពិតប្រាកដ។
  5. គណនា៖ បម្លែងពេលវេលារបស់អ្នកទៅជាវិនាទី ហើយអនុវត្តរូបមន្តខាងលើ ឬប្រើឧបករណ៍គណនា CSS ក្នុង Swim Analytics ដើម្បីទទួលបានល្បឿន 100 ម៉ែត្ររបស់អ្នក។

តើអ្នកគួរប្រើមួយណា?

ប្រើ Threshold Pace (T-Pace) ប្រសិនបើ៖

  • អ្នកទើបតែចាប់ផ្តើមជាមួយនឹងការបណ្តុះបណ្តាលផ្អែកលើល្បឿន។
  • អ្នក​គ្រាន់តែ​ហែល​ទឹក​ជា​បន្តបន្ទាប់ (បើក​ចំហរ 3km+)។
  • អ្នកមិនចង់ដោះស្រាយជាមួយវគ្គសាកល្បងច្រើនទេ។

តើ CSS ដូចគ្នាទៅនឹងកម្រិតល្បឿនដែរឬទេ?

ទេ ពួកវាជាគោលគំនិតកម្រិតដែលទាក់ទង ប៉ុន្តែវាមិនមែនជាការសាកល្បងដូចគ្នានោះទេ។ ល្បឿនកម្រិតជាធម្មតាបានមកពីការហែលទឹកយូរជាងមួយ ខណៈពេលដែល CSS ត្រូវបានគណនាពីការខិតខំប្រឹងប្រែងពីរ និងផ្តល់នូវយុថ្កាដែលអាចធ្វើម្តងទៀតបានបន្ថែមទៀតសម្រាប់ការងារចន្លោះពេល។

តើមួយណាល្អជាងសម្រាប់កីឡាទ្រីយ៉ាត្លុង និងការរត់ក្នុងទឹក?

CSS ជា​ធម្មតា​ជា​ចំណុច​ចាប់​ផ្ដើម​ល្អ​ជាង​ព្រោះ​វា​ឆ្លុះ​បញ្ចាំង​ពី​ល្បឿន​ប្រកប​ដោយ​និរន្តរភាព​ដោយ​មិន​ផ្តល់​រង្វាន់​លើស​ល្បឿន​ក្នុង​រយៈ​ពេល​ខ្លី។ ល្បឿនកម្រិតអាចនៅតែមានប្រយោជន៍សម្រាប់ការហែលទឹកថេរយូរ ប៉ុន្តែជាធម្មតា CSS ងាយស្រួលក្នុងការបំប្លែងទៅជាគោលដៅល្បឿន និងការងារតាមតំបន់។

តើខ្ញុំផ្លាស់ទីពីកម្រិតល្បឿនទៅតំបន់ហែលទឹកដែលមានមូលដ្ឋានលើ CSS ដោយរបៀបណា?
