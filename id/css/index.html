Kecepatan Berenang Kritis (CSS)

Tolok ukur kecepatan praktis yang menjadi landasan zona latihan renang, penetapan ambang batas, dan analisis beban latihan

Jawaban Cepat

Kecepatan Berenang Kritis (CSS) adalah perkiraan kecepatan berenang ambang batas berkelanjutan Anda yang telah teruji di lapangan. Kebanyakan perenang menghitungnya dari a Uji coba waktu 400m + 200m, lalu gunakan hasilnya untuk mengatur zona pelatihan, set ambang batas kecepatan, dan kekuatan sTSS dan beban pelatihan analisis.

  • Penggunaan terbaik: pelatihan ambang batas, tempo berulang, dan pelacakan beban
  • Interpretasi khas: kira-kira kecepatan tercepat yang dapat Anda pertahankan secara aerobik untuk kerja keras dan terkontrol
  • Jangan menggunakannya secara membabi buta: tempo yang buruk, pemulihan yang tidak lengkap, kondisi kolam yang berbeda, atau alat bantu tarik dapat merusak hasil
  • Tes ulang irama: setiap 6-8 minggu, atau setelah blok pelatihan yang berarti

Poin Penting

  • Apa itu CSS: perkiraan kecepatan ambang batas praktis untuk perenang, yang dibuat dari pengujian lapangan sederhana
  • Mengapa ini penting: CSS mengubah latihan berenang dari hanya menebak-nebak menjadi tempo yang berulang dan kontrol beban yang lebih baik
  • Apa yang dibukanya:zona, sTSS, manajemen kinerja, dan pemeriksaan kemajuan yang lebih jelas
  • Apa yang tidak dilakukannya: menggantikan analisis teknik, konteks perairan terbuka, atau interpretasi efisiensi
  • Langkah terbaik berikutnya: uji CSS dengan baik, lalu gunakan untuk menyusun ambang batas dan pekerjaan aerobik Anda

Jika Anda hanya menggunakan satu metrik renang sebagai jangkar kecepatan, pertahankan CSS. Tes ini tidak sempurna dan bukan tes laboratorium fisiologis lengkap, namun praktis, dapat diulang, dan bermanfaat secara langsung. Kombinasi itulah yang menyebabkan CSS tetap menjadi tulang punggung pelatihan renang berbasis data.

Bagi sebagian besar perenang, Kecepatan Berenang Kritis adalah jembatan antara uji waktu mentah dan keputusan pelatihan sehari-hari. Ini membantu menjawab lima pertanyaan penting:

  • Seberapa cepat sebenarnya penetapan ambang batas?
  • Berapa kisaran kecepatan yang dimiliki masing-masing zona berenang?
  • Bagaimana cara saya mengulang kecepatan 100 detik, 200 detik, dan 400 detik?
  • Bagaimana cara mengukur stres renang sTSS?
  • Apakah kebugaran renang aerobik saya bergerak ke arah yang benar?

Apa yang Diukur CSS

CSS adalah perkiraan praktis kecepatan renang aerobik keras, biasanya berasal dari time trial 400m dan 200m. Dalam bahasa kepelatihan, ini sering kali diperlakukan sebagai kecepatan ambang batas untuk pelatihan kumpulan terstruktur: cukup cepat untuk menjadi penting, cukup terkontrol untuk diulang, dan cukup stabil untuk membangun zona di sekitarnya.

🎯 Model Mental yang Berguna

Anggaplah CSS sebagai milik Anda tolok ukur kecepatan operasi, bukan angka ajaib. Ini adalah referensi kecepatan yang membuat metrik renang Anda lainnya menjadi koheren.

Mengapa CSS Penting

CSS penting karena menghubungkan fisiologi dengan eksekusi. Banyak perenang mengetahui bahwa mereka melakukan "set keras" atau "sesi tempo" namun tidak dapat memastikan apakah kecepatannya tepat. CSS memperbaiki masalah itu dengan memberi Anda patokan yang dapat Anda gunakan sesi demi sesi.

🏊 Zona Pelatihan

CSS memberi Anda jangkar yang bersih zona berenang, sehingga pekerjaan mudah tetap mudah dan ambang batas pekerjaan tetap tepat sasaran.

📊 Beban Latihan

Tanpa CSS yang dapat digunakan, sTSS dan metrik manajemen kinerja kehilangan maknanya.

⏱️ Kecepatan Berulang

CSS membantu Anda mencapai kecepatan 100, 200, dan 400 dengan lebih konsisten daripada hanya mengandalkan perasaan.

📈 Pemeriksaan Kemajuan

Saat CSS meningkat, kecepatan ambang batas Anda bergerak lebih cepat. Ini merupakan sinyal kuat bahwa kebugaran renang aerobik juga meningkat.

⚠️ Ketergantungan Kritis: Jika CSS Anda salah, zona, sTSS, dan nomor manajemen kinerja Anda akan menyimpang ke arah yang salah. Masukan yang buruk menghasilkan analisis yang buruk.

Apa Fungsi CSS dan Apa yang Tidak

CSS sangat baik untuk: set ambang batas, kualitas kerja aerobik, pengaturan zona, dan pelacakan beban khusus renang.

CSS tidak cukup untuk: pembuatan profil sprint, diagnosis teknik pukulan, pelaksanaan balapan di perairan terbuka dalam berbagai kondisi, atau menafsirkan efisiensinya sendiri.

Gunakan CSS Untuk

  • Resep ambang batas kecepatan
  • Ulangi kecepatan 100/200/400
  • Dipersonalisasi struktur zona
  • sTSS dan manajemen beban
  • Melacak kemajuan aerobik sepanjang musim

Jangan Gunakan CSS Sendiri Untuk

  • Membandingkan panjang kolam yang berbeda seolah-olah keduanya identik
  • Membandingkan perenang dengan profil teknik yang sangat berbeda
  • Menilai kualitas pukulan tanpa mekanika pukulan
  • Menilai efisiensi tanpa konteks serigala
  • Dengan asumsi kecepatan perairan terbuka akan sama persis dengan kecepatan CSS kumpulan

Cara Menguji CSS dalam 4 Langkah Praktis

📋 Protokol Standar

  1. Lakukan pemanasan dengan benar

    Berenang 300-800m dengan mudah, sertakan latihan, dan tambahkan beberapa upaya membangun yang terkontrol sebelum tes.

  2. Berenang dalam uji waktu 400m

    Gunakan start dorong, bukan menyelam. Sasarannya adalah 400m tercepat dan berkelanjutan, bukan 100m pertama yang sembrono.

  3. Pulihkan sepenuhnya

    Luangkan waktu 5-10 menit istirahat ringan atau total. Pemulihan yang tidak lengkap adalah salah satu cara tercepat untuk merusak hasilnya.

  4. Berenang dalam uji waktu 200m

    Sekali lagi gunakan push start dan catat waktunya dengan tepat. Kecepatan 200m Anda seharusnya jauh lebih cepat daripada kecepatan 400m Anda.

⚠️ Kesalahan Pengujian Paling Umum

Kecepatan 400m yang buruk

Masalah: Anda keluar terlalu keras, lalu memudar dengan buruk.

Hasil: Perhentian 400m mencerminkan kualitas kecepatan yang berkelanjutan.

Perbaiki: Bertujuan untuk pembagian yang genap atau sedikit negatif.

Pemulihan tidak memadai

Masalah: Kelelahan membuat lari 200m menjadi lambat secara artifisial.

Hasil: CSS menjadi terdistorsi dan zona pelatihan Anda melayang.

Perbaiki: Tunggu hingga pernapasan terkontrol dan pemulihan terasa selesai.

Kondisi campuran

Masalah: Panjang kumpulan yang berbeda, perubahan dorongan yang besar, atau alat bantu tarik mengubah lingkungan pengujian.

Hasil: Anda membandingkan angka-angka yang sebenarnya tidak sebanding.

Perbaiki: Jaga kondisi sekonsisten mungkin dari pengujian ke pengujian.

🔄 Frekuensi Pengujian Ulang

Tes ulang CSS setiap 6-8 minggu, atau setelah blok bermakna yang berfokus pada ambang batas, daya tahan, atau konsolidasi teknik. Intinya bukanlah pengujian terus-menerus. Intinya adalah menggunakan satu tolok ukur secara cukup konsisten untuk melihat perubahan.

Rumus Perhitungan CSS

Rumus

CSS (m/s) = (D₂ - D₁) / (T₂ - T₁)

Dimana:

  • D₁ = 200 meter
  • D₂ = 400 meter
  • T₁ = Waktu untuk 200m (dalam detik)
  • T₂ = Waktu untuk 400m (dalam detik)

Disederhanakan untuk Kecepatan per 100m

Kecepatan CSS/100m (detik) = (T₄₀₀ - T₂₀₀) / 2

Contoh yang berhasil

Hasil Tes

  • waktu 400m: 06:08 (368 detik)
  • waktu 200m: 2:30 (150 detik)

Langkah 1: Hitung kecepatan CSS

Kecepatan CSS/100m = (368 - 150) / 2
Kecepatan CSS/100m = 218/2
Kecepatan CSS/100m = 109 detik = 1:49 per 100m

Langkah 2: Ubah ke kecepatan jika diperlukan

CSS = 100/109
CSS = 0,917 m/s

Contoh yang Berhasil: Cara Menggunakan Hasil Itu

Jika CSS Anda 1:49/100m, pengulangan ambang batas Anda harus mendekati kecepatan tersebut, milik Anda tabel zona harus dibangun darinya, dan Anda sTSS perhitungan harus menggunakannya sebagai referensi kecepatan penyebut.

Cara Menggunakan CSS dalam Pelatihan Nyata

1️⃣ Set Ambang Kecepatan

Gunakan CSS untuk berhenti menebak-nebak. Set seperti 8x100, 5x200, atau 3x400 lebih masuk akal jika Anda mengetahui target kecepatan dan dapat menahannya dengan baik.

2️⃣ Hubungkan Kecepatan ke sTSS

CSS memberi arti pada Anda sTSS. Sesi sulit dan sesi mudah berhenti menyatu dalam beban kerja samar yang sama.

3️⃣ Lacak Kemajuan dengan Lebih Baik

Jika CSS menjadi lebih cepat sementara sesi Anda tetap terkontrol, ambang batas kebugaran renang Anda kemungkinan akan meningkat.

4️⃣ Bangun Kebiasaan Kecepatan yang Lebih Baik

CSS mengajarkan Anda untuk berenang berulang kali dengan kontrol alih-alih mengubah setiap set yang sulit menjadi kegagalan.

Contoh Ide Sesi Berorientasi CSS

  • Pemeliharaan ambang batas: 8x100 dengan kecepatan CSS dengan istirahat singkat
  • Daya tahan kecepatan: 5x200 sekitar kecepatan CSS dengan pemisahan yang konsisten
  • Dukungan aerobik: 3x400 sedikit lebih lambat dari kecepatan CSS dengan jumlah pukulan terkontrol
  • Teknik + kecepatan: ganti 100m pada upaya CSS dengan 50m reset teknis yang mudah

Ini hanyalah contoh, bukan resep universal. Intinya adalah CSS memberi Anda bahasa tempo yang stabil.

CSS, SWOLF, dan Efisiensi

CSS memberitahu Anda seberapa cepat Anda bisa berenang di ambang pintu. SWOLF dan metrik efisiensi memberi tahu Anda sesuatu yang berbeda: seberapa ekonomis Anda bergerak di air. Mereka saling melengkapi, namun tidak dapat dipertukarkan.

Jangan Membandingkan SWOLF Secara Buta

SWOLF yang lebih rendah tidak selalu "lebih baik" pada perenang, sesi, pukulan, atau pengaturan kolam renang. Gunakan metrik efisiensi sebagai konteks bersama dengan CSS, kualitas goresan, dan kemampuan pengulangan, bukan sebagai alat pemeringkatan yang berdiri sendiri.

Validasi Ilmiah

Wakayoshi dkk. dan Kasus Praktis untuk CSS

CSS menjadi digunakan secara luas karena menawarkan pelatih dan perenang tolok ukur ramah lapangan yang dapat dilacak dengan baik dengan penanda kinerja terkait ambang batas tanpa memerlukan pengujian laktat laboratorium berulang kali.

  • Praktis: dapat diulang dari pengujian kumpulan sederhana
  • Berguna: cukup dekat dengan kecepatan yang relevan dengan ambang batas untuk pemrograman
  • Dapat ditindaklanjuti: mudah untuk diuji ulang dan diperbarui sepanjang musim
  • Terhubung: langsung berguna untuk zona pelatihan, kecepatan, dan pelacakan beban

Pertanyaan yang Sering Diajukan

Seberapa sering saya harus menguji ulang CSS?

Setiap 6-8 minggu adalah standar yang baik. Tes ulang lebih cepat hanya jika latihan banyak berubah atau zona Anda saat ini jelas terasa salah.

Apakah CSS sama dengan ambang laktat?

Tidak tepat. CSS adalah perkiraan lapangan yang digunakan sebagai tolok ukur kecepatan ambang batas praktis. Hal ini berguna karena dapat diulang dan ditindaklanjuti, bukan karena menggantikan pengujian laboratorium dengan sempurna.

Bisakah saya menggunakan CSS untuk mondar-mandir di perairan terbuka?

Gunakan ini sebagai referensi awal, bukan jaminan literal. Penampakan, penyusunan, pemotongan, pakaian selam, dan navigasi semuanya mengubah perilaku kecepatan di perairan terbuka.

Bisakah pemula menggunakan CSS?

Ya, tapi hasilnya menjadi lebih berguna seiring dengan peningkatan teknik dan kecepatan. Perenang yang sangat baru sering kali membutuhkan konsistensi pukulan dan kecepatan sebelum CSS menjadi sangat stabil.

Haruskah saya membandingkan CSS saya secara langsung dengan CSS orang lain?

Hanya dengan hati-hati. Panjang kolam, push-off, drag, keterampilan mondar-mandir, dan kualitas teknik semuanya mempengaruhi hasil. CSS paling kuat sebagai tolok ukur pribadi dari waktu ke waktu.

  • Beban pelatihan untuk memantau kebugaran, kelelahan, dan bentuk tubuh
