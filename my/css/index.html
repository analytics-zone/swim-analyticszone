အရေးကြီးသော ရေကူးအမြန်နှုန်း (CSS)

Data-Driven Swimming Training ၏ ဖောင်ဒေးရှင်း

Critical Swim Speed (CSS) ဆိုတာဘာလဲ။

အရေးကြီးသော ရေကူးအမြန်နှုန်း (CSS)ပင်ပန်းနွမ်းနယ်မှုမရှိဘဲ ထိန်းထားနိုင်သော သီအိုရီအရ အမြင့်ဆုံး ရေကူးအလျင်ဖြစ်သည်။ ၎င်းသည် ပုံမှန်အားဖြင့် 4 mmol/L blood lactate နှင့် သက်ဆိုင်ပြီး ခန့်မှန်းခြေအားဖြင့် မိနစ် 30 ကြာတည်တံ့သော သင်၏ အေရိုးဗစ်အဆင့်ကို ကိုယ်စားပြုသည်။ CSS ကို 400m နှင့် 200m time trial သုံးပြီး တွက်ချက်ပြီး စိတ်ကြိုက်လေ့ကျင့်မှုဇုန်များကို ဆုံးဖြတ်ပါသည်။

Critical Swim Speed (CSS) သည် သင် အားမကုန်ဘဲ အဆက်မပြတ် ထိန်းသိမ်းနိုင်သော သီအိုရီ အများဆုံး ရေကူးအလျင်ကို ကိုယ်စားပြုသည်။ ၎င်းသည် ရေထဲတွင် သင့်အေရိုးဗစ်အဆင့်ဖြစ်သည်—နို့နို့ထွက်ရှိမှုနှင့် နို့နို့ထွက်ရှိမှုကို ညီမျှသည့်ပြင်းထန်မှုဖြစ်သည်။

🎯 ဇီဝကမ္မဆိုင်ရာ အရေးပါမှု၊

CSS သည်-

  • Lactate Threshold 2 (LT2)- ဒုတိယလေဝင်လေထွက်တံခါးပေါက်
  • အများဆုံး Lactate ပုံမှန်အနေအထား (MLSS)- အမြင့်ဆုံးရေရှည်တည်တံ့သောနို့ရည်အဆင့်
  • လုပ်ဆောင်နိုင်သော အဆင့်သတ်မှတ်နှုန်း (FTP)- စက်ဘီးစီးခြင်း FTP နှင့် ညီမျှသော ရေကူးခြင်း။
  • ~ 4 mmol/L သွေးနို့တိုက်ကျွေးခြင်း။- ရိုးရာ OBLA အမှတ်အသား

ဘာကြောင့် CSS က အရေးကြီးတာလဲ။

CSS သည်အခြေခံမက်ထရစ်အဆင့်မြင့်လေ့ကျင့်ရေးဝန်ခွဲခြမ်းစိတ်ဖြာမှုအားလုံးကိုသော့ဖွင့်ပေးသည်-

  • လေ့ကျင့်ရေးဇုန်များ-သင်၏ဇီဝကမ္မဗေဒအပေါ်အခြေခံ၍ ပြင်းထန်မှုဇုန်များကို စိတ်ကြိုက်ပြင်ဆင်ပါ။
  • sTSS တွက်ချက်မှု-တိကျသောလေ့ကျင့်ရေးစိတ်ဖိစီးမှုရမှတ် အရေအတွက်ကို ဖွင့်ပေးသည်။
  • CTL/ATL/TSB-စွမ်းဆောင်ရည်စီမံခန့်ခွဲမှုဇယား မက်ထရစ်များအတွက် လိုအပ်သည်။
  • တိုးတက်မှု ခြေရာခံခြင်း-အေရိုးဗစ် ကြံ့ခိုင်မှု တိုးတက်စေရန် ရည်ရွယ်ချက် တိုင်းတာခြင်း။
⚠️ အရေးကြီးသော မှီခိုမှု-မှန်ကန်သော CSS စမ်းသပ်မှုမရှိဘဲ၊ အဆင့်မြင့်လေ့ကျင့်ရေးဝန်မက်ထရစ်များ (sTSS၊ CTL၊ ATL၊ TSB) ကို တွက်ချက်၍မရပါ။ မမှန်ကန်သော CSS သည် နောက်ဆက်တွဲလေ့ကျင့်ရေးပိုင်းခြားစိတ်ဖြာချက်အားလုံးကို ပျက်စီးစေပါသည်။

CSS ကို လွယ်ကူသော အဆင့် ၄ ဆင့်ဖြင့် တွက်ချက်နည်း

📋 စံပရိုတိုကော

  1. သွေးပူ

    300-800m လွယ်ကူသောရေကူးခြင်း၊ လေ့ကျင့်ခန်းများနှင့် အမြင့်ဆုံးအားစိုက်ထုတ်မှုအတွက် ကြိုတင်ပြင်ဆင်ခြင်း။

  2. မီတာ 400 အချိန်အစမ်း

    တွန်းတင်ခြင်းမှ အထိရောက်ဆုံး အားထုတ်မှု (ငုပ်ခြင်းမပြုပါ)။ အချိန်ကို ဒုတိယပိုင်း မှတ်တမ်းတင်ပါ။ ပန်းတိုင်- အမြန်ဆုံး ရေရှည်တည်တံ့သော မီတာ ၄၀၀။

  3. ပြီးပြည့်စုံသော ပြန်လည်ရယူခြင်း။

    5-10 မိနစ်လွယ်ကူသောရေကူးခြင်း သို့မဟုတ် ပြီးပြည့်စုံသော အနားယူခြင်း။ တိကျသောရလဒ်များအတွက် ၎င်းသည် အရေးကြီးပါသည်။

  4. 200 မီတာ အချိန်စမ်းသပ်မှု

    တွန်းတင်ခြင်းမှ အမြင့်ဆုံးအားထုတ်မှု။ အချိန်ကိုတိကျစွာမှတ်တမ်းတင်ပါ။ ၎င်းသည် မီတာ ၄၀၀ ထက် မီတာ ၁၀၀ လျှင်မြန်သင့်သည်။

⚠️ အဖြစ်များသော အမှားများ

ပြန်လည်ရယူခြင်း မလုံလောက်ပါ။

ပြဿနာ-ပင်ပန်းနွမ်းနယ်မှုသည် အချိန် 200 မီတာကို အတုယူ၍ နှေးကွေးစေသည်။

ရလဒ်-CSS နဲ့ တွက်တာ ဖြစ်လာတာပါ။လက်တွေ့ထက် ပိုမြန်တယ်။overtrained ဇုန်များသို့ဦးတည်သည်။

ဖြေရှင်းချက်-HR သည် 120 bpm အောက်တွင် ကျဆင်းသွားသည်အထိ သို့မဟုတ် အသက်ရှူခြင်း အပြည့်အဝ ပြန်လည်ကောင်းမွန်လာသည်အထိ အနားယူပါ။

မီတာ ၄၀၀ အကွာအဝေးတွင် ညံ့ဖျင်းသည်။

ပြဿနာ-စတင်ခြင်းသည် အလွန်လျှင်မြန်စွာ နှေးကွေးစေပါသည်။

ရလဒ်-မီတာ ၄၀၀ အချိန်သည် စစ်မှန်သော ရေရှည်တည်တံ့သော အရှိန်အဟုန်ကို ထင်ဟပ်ခြင်းမရှိပါ။

ဖြေရှင်းချက်-ခွဲထွက်ခြင်း သို့မဟုတ် အနုတ်လက္ခဏာခွဲခြင်းအတွက် ရည်ရွယ်သည် (ဒုတိယ 200m ≤ ပထမမီတာ 200)

Dive Starts ကိုအသုံးပြုခြင်း။

ပြဿနာ-~ 0.5-1.5 စက္ကန့်အားသာချက်၊ တွက်ချက်မှုများကို skewing ထည့်ပေးသည်။

ဖြေရှင်းချက်-နံရံမှ push start ကို အမြဲသုံးပါ။

🔄 ပြန်လည်စမ်းသပ်ခြင်းအကြိမ်ရေ

CSS တိုင်းကို ပြန်လည်စမ်းသပ်ပါ။6-8 ပတ်ကြံ့ခိုင်မှု ပိုမိုကောင်းမွန်လာသည်နှင့်အမျှ လေ့ကျင့်ရေးဇုန်များကို အပ်ဒိတ်လုပ်ရန်။ လေ့ကျင့်ရေးနဲ့ လိုက်လျောညီထွေဖြစ်အောင် လိုက်လျောညီထွေဖြစ်အောင် သင့်ဇုန်တွေက တဖြည်းဖြည်း ပိုမြန်လာပါလိမ့်မယ်။

CSS တွက်ချက်မှု ဖော်မြူလာ

ဖော်မြူလာ

CSS (m/s) = (D₂ - D₁) / (T₂ - T₁)

ဘယ်မှာလဲ-

  • D₁ = 200 မီတာ
  • D₂ = 400 မီတာ
  • T₁ = 200m (စက္ကန့်အတွင်း) အချိန်
  • T₂ = 400m (စက္ကန့်အတွင်း) အချိန်

မီတာ 100 လျှင် Pace အတွက် ရိုးရှင်းပါသည်။

CSS Pace/100m (စက္ကန့်) = (T₄₀₀ - T₂₀₀) / 2

အလုပ်လုပ်ပုံနမူနာ

စမ်းသပ်မှုရလဒ်များ

  • မီတာ 400 အချိန်၆:၀၈(၃၆၈)စက္ကန့်၊
  • 200m အချိန်-၂:၃၀စက္ကန့် (၁၅၀)၊

အဆင့် 1- CSS ကို m/s ဖြင့် တွက်ချက်ပါ။

CSS = (400 - 200) / (368 - 150)
CSS = 200/218
CSS =0.917 m/s

အဆင့် 2- မီတာ 100 နှုန်းသို့ အရှိန်ပြောင်းပါ။

ခြေလှမ်း = 100 / 0.917
အရှိန် = 109 စက္ကန့်
အရှိန်အဟုန် =မီတာ 100 လျှင် 1:49

အခမဲ့ CSS ဂဏန်းတွက်စက်

သင်၏အရေးပါသောရေကူးအမြန်နှုန်းနှင့် စိတ်ကြိုက်လေ့ကျင့်ရေးဇုန်များကို ချက်ချင်းတွက်ချက်ပါ။

ဖော်မတ်- မိနစ်-စက္ကန့် (ဥပမာ၊ 6:08)
ဖော်မတ်- မိနစ်-စက္ကန့် (ဥပမာ၊ 2:30)

သင်၏ရလဒ်များ

အရေးကြီးသော ရေကူးအမြန်နှုန်း
--
100m အတွက် CSS Pace
--

သင့်စိတ်ကြိုက်လေ့ကျင့်ရေးဇုန်များ

သင်၏ CSS ကို အခြေခံ၍ 100m နှုန်းဖြင့် သင့်လေ့ကျင့်ရေးဇုန်များ ဖြစ်သည်-

အစားထိုး (ရိုးရှင်းသောနည်းလမ်း)-

Pace = (368 - 150) / 2
အရှိန် = ၂၁၈/၂
အရှိန် = 109 စက္ကန့် =မီတာ 100 လျှင် 1:49

CSS ကိုအခြေခံ၍ လေ့ကျင့်ရေးဇုန်များ

မှတ်ချက်-ရေကူးရာတွင် အရှိန်ကို တိုင်းတာသည်။အကွာအဝေးအလိုက် အချိန်. ထို့ကြောင့်၊မြင့်မားသောရာခိုင်နှုန်း = နှေးကွေးသောအရှိန်, and aရာခိုင်နှုန်းနိမ့် = မြန်သောနှုန်း. ၎င်းသည် % = ပိုမြင့်သော စက်ဘီးစီးခြင်း/ပြေးခြင်းသို့ ပြောင်းပြန်ဖြစ်သည်။

ဇုန် နာမည် CSS Pace ၏ % CSS အတွက် ဥပမာ 1:40/100m ကာကွယ်မှု 12X ဇီဝကမ္မရည်ရွယ်ချက်
၁ ပြန်လည်ထူထောင်ရေး >108% > 1:48/100m ၂-၃/၁၀ တက်ကြွစွာ ပြန်လည်ရယူခြင်း၊ နည်းပညာ သန့်စင်ခြင်း၊ သွေးပူခြင်း/အအေးခံခြင်း
၂ အေရိုးဗစ်အခြေစိုက်စခန်း 104-108% 1:44-1:48/100m ၄-၅/၁၀ အေရိုးဗစ်စွမ်းရည်၊ mitochondrial သိပ်သည်းဆ၊ အဆီဓာတ်တိုးခြင်းကို တည်ဆောက်ပါ။
၃ Tempo/Sweet Spot 99-103% 1:39-1:43/100m ၆-၇/၁၀ ပြိုင်ပွဲအရှိန်အဟုန်ကိုလိုက်လျောညီထွေဖြစ်အောင်၊ အာရုံကြောကြွက်သားစွမ်းဆောင်ရည်
၄ အဆင့် (CSS) 96-100% 1:36-1:40/100m ၇-၈/၁၀ နို့တိုက်ခြင်းအဆင့်ကို မြှင့်တင်ခြင်း၊ ပြင်းထန်မှု မြင့်မားခြင်း။
၅ VO₂max/Anaerobic <96% <1:36/100m ၉-၁၀/၁၀ VO₂max ဖွံ့ဖြိုးတိုးတက်မှု၊ စွမ်းအား၊ နို့တိုက်ကျွေးမှု သည်းခံနိုင်မှု

  • ဇုန် 2-အေရိုးဗစ်အင်ဂျင်ကို တည်ဆောက်ပါ (အပတ်စဉ် ပမာဏ၏ 60-70%)
  • ဇုန် ၃-အပြေးအလွှား စွမ်းဆောင်ရည်ကို မြှင့်တင်ပါ (ထုထည်၏ 15-20%)
  • ဇုန် 4-lactate အဆင့်ကို ပိုမြင့်အောင် တွန်းပါ (ထုထည်၏ 10-15%)
  • ဇုန် 5-ထိပ်တန်းအမြန်နှုန်းနှင့် ပါဝါကို မြှင့်တင်ပါ (ထုထည်၏ 5-10%)

အိုဆာကာတက္ကသိုလ်ရှိ Kohji Wakayoshi ၏ ဟောပြောမှုလေ့လာမှုများက CSS ကို ဓာတ်ခွဲခန်းနို့တိုက်ကျွေးခြင်းအတွက် မှန်ကန်ပြီး လက်တွေ့ကျသော အစားထိုးတစ်ခုအဖြစ် တည်ထောင်ခဲ့သည်-

  • anaerobic အဆင့်တွင် VO₂ နှင့် ပြင်းထန်သောဆက်စပ်မှု(r=0.818)၊
  • OBLA တွင် အလျင်နှင့် အလွန်ကောင်းမွန်သော ဆက်စပ်မှု(r=0.949)
  • မီတာ 400 စွမ်းဆောင်ရည်ကို ခန့်မှန်းသည်။(r=0.864)၊
  • 4 mmol/L သွေး lactate နှင့် ကိုက်ညီသည်။- အများဆုံး lactate တည်တည်ငြိမ်ငြိမ်
  • အကွာအဝေးနှင့် အချိန်အကြား မျဉ်းသားသော ဆက်ဆံရေး(r² > 0.998)

🔬 CSS ဘာကြောင့် အလုပ်လုပ်တာလဲ။

CSS သည် ၎င်းအကြား နယ်နိမိတ်ကို ကိုယ်စားပြုသည်။ပြင်းထန်ပြီး ပြင်းထန်သောလေ့ကျင့်ခန်းများ. CSS အောက်တွင်၊ နို့တိုက်ကျွေးခြင်း ထုတ်လုပ်မှုနှင့် ကင်းရှင်းမှုသည် ဟန်ချက်ညီစွာ ရှိနေသည်- သင်သည် အချိန်ကြာမြင့်စွာ ရေကူးနိုင်သည်။ CSS အထက်တွင်၊ lactate သည် မိနစ် 20-40 အတွင်း ပင်ပန်းနွမ်းနယ်သည်အထိ တဖြည်းဖြည်း စုပုံလာသည်။

၎င်းသည် CSS အတွက် ပြီးပြည့်စုံသော ပြင်းထန်မှုကို ဖြစ်စေသည်-

  • 800m မှ 1500m ပွဲစဉ်များအတွက် ရေရှည်တည်တံ့သော ပြိုင်ပွဲအရှိန်ကို သတ်မှတ်ခြင်း။
  • ကြားကာလ လေ့ကျင့်မှု အဆင့်သတ်မှတ်ခြင်း။
  • အေရိုးဗစ် ကြံ့ခိုင်မှု တိုးတက်မှုကို စောင့်ကြည့်ခြင်း။
  • လေ့ကျင့်ရေးဝန်နှင့် ပြန်လည်ရယူရန် လိုအပ်ချက်များကို တွက်ချက်ခြင်း။

သင်၏ CSS ကိုစမ်းသပ်နည်း

အဆင့် 1: သင့်လျော်စွာပူနွေး

လေ့ကျင့်ခန်းများနှင့် အဆင့်ဆင့်တည်ဆောက်မှုများ အပါအဝင် လွယ်ကူသော ရေကူး မီတာ 300 မှ 800 မီတာအထိ အပြီးသတ်ပါ။ ဒါက သင့်ခန္ဓာကိုယ်ကို အမြင့်ဆုံးအားစိုက်ထုတ်ပြီး ထိခိုက်ဒဏ်ရာရတာကို ကာကွယ်ပေးပါတယ်။

အဆင့် 2- မီတာ 400 အချိန်စမ်းသပ်မှုကို လုပ်ဆောင်ပါ။

တွန်းတင်ခြင်းမှ အမြင့်ဆုံး မီတာ 400 တွင် ရေကူးပါ (ငုပ်ခြင်းမပြုပါ)။ သင်၏အချိန်ကို ဒုတိယအဖြစ် မှတ်တမ်းတင်ပါ။ ခွဲခြမ်းများပင် သို့မဟုတ် အနုတ်ပိုင်းခွဲခြင်းအတွက် ရည်ရွယ်သည် (ပထမမီတာ 200 နှင့် ပထမမီတာ 200 နှင့် ညီမျှသော သို့မဟုတ် ပိုမြန်သည်)။

အဆင့် 3: လုံးဝပြန်လည်ရယူပါ။

လွယ်ကူစွာ ရေကူးခြင်းဖြင့် 5-10 မိနစ်ခန့် အနားယူပါ သို့မဟုတ် ပြီးပြည့်စုံစွာ အနားယူပါ။ ဤပြန်လည်ရယူခြင်းသည် အရေးကြီးပါသည်။ သင့်နှလုံးခုန်နှုန်း 120 bpm အောက်တွင် ကျဆင်းသွားသည်အထိ စောင့်ဆိုင်းပြီး အသက်ရှုမှု အပြည့်အဝ ပြန်လည်ကောင်းမွန်လာသည်အထိ စောင့်ပါ။

အဆင့် 4- မီတာ 200 အချိန်စမ်းသပ်မှုကို လုပ်ဆောင်ပါ။

တွန်းတင်ရာမှ အမြင့်ဆုံး မီတာ 200 ရေကူးပါ။ သင်၏အချိန်ကိုတိကျစွာမှတ်တမ်းတင်ပါ။ ၎င်းသည် သင်၏ မီတာ ၄၀၀ နှုန်းထက် မီတာ 100 တွင် ပိုမြန်သင့်သည်။

အဆင့် 5- သင်၏ CSS ကို တွက်ချက်ပါ။

ဖော်မြူလာကို အသုံးပြုပါ- CSS Pace/100m = (T400 - T200) / 2. ဥပမာ- (368s - 150s) / 2 = 109s = 1:49/100m။ သို့မဟုတ် စိတ်ကြိုက်လေ့ကျင့်ရေးဇုန်များဖြင့် ချက်ချင်းရလဒ်များရရှိရန် ဤစာမျက်နှာရှိ ဂဏန်းပေါင်းစက်ကို အသုံးပြုပါ။

လက်တွေ့အသုံးချမှုများ

1️⃣ Training Load Metrics ကိုဖွင့်ပါ။

CSS သည် sTSS အတွက် Intensity Factor တွက်ချက်မှုတွင် ပိုင်းခြေဖြစ်သည်။ ၎င်းမရှိလျှင် သင်သည် လေ့ကျင့်ခန်း၏ဖိစီးမှုကို တွက်ချက်ခြင်း သို့မဟုတ် ကြံ့ခိုင်မှု/ပင်ပန်းနွမ်းနယ်မှုလမ်းကြောင်းများကို ခြေရာခံနိုင်မည်မဟုတ်ပေ။

2️⃣ လေ့ကျင့်ရေးဇုန်များကို စိတ်ကြိုက်ပြင်ဆင်ပါ။

ယေဘူယျ အရှိန်အဟုန် ဇယားများသည် တစ်ဦးချင်း ဇီဝကမ္မဗေဒ အတွက် ထည့်မတွက်ပါ။ CSS အခြေပြုဇုန်များသည် ရေကူးသူရထားတိုင်းကို ၎င်းတို့၏ အကောင်းဆုံးပြင်းထန်မှုဖြင့် သေချာစေသည်။

3️⃣ ကြံ့ခိုင်မှုတိုးတက်မှုကို စောင့်ကြည့်ပါ။

6-8 ပတ်တစ်ကြိမ်ပြန်စစ်ပါ။ CSS ကို မြှင့်တင်ခြင်း (ပိုမိုမြန်ဆန်သော အရှိန်အဟုန်) သည် အောင်မြင်သော အေရိုးဗစ် လိုက်လျောညီထွေရှိမှုကို ဖော်ပြသည်။ CSS သည် လေ့ကျင့်ချိန်ညှိမှု လိုအပ်ကြောင်း အကြံပြုထားသည်။

4️⃣ ပြိုင်ပွဲစွမ်းဆောင်ရည်ကို ခန့်မှန်းပါ။

CSS ခြေလှမ်းသည် သင်၏ ရေရှည်တည်တံ့သော မိနစ် 30 ပြိုင်ပွဲအရှိန်အဟုန်ကို ခန့်မှန်းပါသည်။ မီတာ 800၊ 1500 မီတာနှင့် အဖွင့်ရေပွဲများအတွက် လက်တွေ့ကျသောပန်းတိုင်များချမှတ်ရန် ၎င်းကိုအသုံးပြုပါ။

5️⃣ Design Threshold လေ့ကျင့်ခန်းများ

ဂန္တဝင် CSS အစုံများ- 8 × 100 @ CSS ခြေလှမ်း (15 စက္ကန့်ကျန်), 5 × 200 @ 101% CSS (20s ကျန်), 3 × 400 @ 103% CSS (30s ကျန်သည်)။ lactate ရှင်းလင်းရေးစွမ်းရည်ကိုတည်ဆောက်ပါ။

6️⃣ Taper Strategy ကို အကောင်းဆုံးလုပ်ပါ။

အတိုင်းအတာ မတိုင်မီနှင့် ပြီးနောက် CSS ကို ခြေရာခံပါ။ အောင်မြင်သော taper သည် ပင်ပန်းနွမ်းနယ်မှုကို လျော့ကျစေပြီး CSS ကို ထိန်းသိမ်းသည် သို့မဟုတ် အနည်းငယ် တိုးတက်စေသည် (TSB တိုးသည်)။

သင်၏ CSS အသိပညာကိုအသုံးချပါ။

ယခု သင် Critical Swim Speed ကို နားလည်ပြီး သင်၏ လေ့ကျင့်မှုကို အကောင်းဆုံးဖြစ်အောင် လုပ်ဆောင်ရန် နောက်အဆင့်များကို လုပ်ဆောင်ပါ။

  • လေ့ကျင့်ရေးစိတ်ဖိစီးမှုရမှတ် (TSS) တွက်ချက်ပါလေ့ကျင့်ခန်းပြင်းထန်မှုကို တွက်ချက်ရန် သင်၏ CSS ကို အခြေခံထားသည်။
  • လေ့ကျင့်ရေးဇုန် 7 ခုကို လေ့လာပါ။တိကျသော လိုက်လျောညီထွေဖြစ်အောင် လေ့ကျင့်ခန်းများကို မည်သို့တည်ဆောက်ရမည်ကို လေ့လာပါ။
  • ဖော်မြူလာအားလုံးကို ကြည့်ပါ။ပြီးပြည့်စုံသောပွင့်လင်းမြင်သာမှုအတွက် Swim Analytics တွင်အသုံးပြုသည်။
  • Swim Analytics ကို ဒေါင်းလုဒ်လုပ်ပါ။CSS၊ TSS နှင့် စွမ်းဆောင်ရည်လမ်းကြောင်းများကို အလိုအလျောက်ခြေရာခံရန်
