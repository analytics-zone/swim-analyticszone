အရေးကြီးသော ရေကူးအမြန်နှုန်း (CSS) နှင့် Threshold Pace- ဘယ်အရာက အရေးကြီးလဲ။

"Threshold Pace" (T-Pace) သည် နှစ်ပေါင်းများစွာ ရွှေစံနှုန်းဖြစ်ခဲ့သည်။ 2026 ခုနှစ်တွင်၊ Critical Swim Speed ​​(CSS) သို့ ကူးပြောင်းမှုသည် အားကစားသမားများသည် ၎င်းတို့၏ အစည်းအဝေးများကို အရှိန်မြှင့်ပုံနှင့် ပြိုင်ပွဲနေ့တွင် ပင်ပန်းနွမ်းနယ်မှုကို ကြိုတင်ခန့်မှန်းသည်။

🎯 အမြန်အဖြေ

T-Pace (အဆင့်သတ်မှတ်နှုန်း) ကျယ်ပြန့်သော လေ့ကျင့်ရေးဇုန်များ သတ်မှတ်ရာတွင် သုံးလေ့ရှိသော တစ်နာရီကြာ သင်ထိန်းထားနိုင်သော အရှိန်အဟုန်၏ ယေဘူယျ ခန့်မှန်းချက်ဖြစ်သည်။ ၎င်းသည် "လက်တွေ့" ဖြစ်သော်လည်း အသေးစိပ်မက်ထရစ်ဖြစ်သည်။

CSS (အရေးပါသော ရေကူးအမြန်နှုန်း) ကွဲပြားခြားနားသောအကွာအဝေးနှစ်ခုမှ သင်္ချာနည်းအရ ဆင်းသက်လာသော "functional" threshold (ဥပမာ၊ 400m နှင့် 200m)။ "Wall" ကို မထိဘဲ ထိန်းထားနိုင်တဲ့ သီအိုရီ အရှိန်အဟုန် ပါပဲ။ အကယ်၍ သင်သည် ခံနိုင်ရည်ရှိသော ရေကူးသမား သို့မဟုတ် သုံးပွင့်ဆိုင်ကစားသမားဖြစ်ပါက၊ သင်၏ CSS သည် သင်၏လေ့ကျင့်ရေးအစီအစဉ်တွင် အရေးကြီးဆုံးနံပါတ်ဖြစ်သည်။

2026 အပြောင်းအလဲ- အားထုတ်မှုမှ ဇီဝကမ္မဗေဒအထိ

ရေကူးကန်ထဲတွင် ပင်ပန်းနွမ်းနယ်မှုကို စီမံခန့်ခွဲခြင်းသည် စက်ဘီးစီးခြင်း သို့မဟုတ် ပြေးခြင်းထက် ခက်ခဲသည်။ ပါဝါမီတာမပါဘဲ၊ အရှိန်သည် ကျွန်ုပ်တို့၏ တစ်ခုတည်းသော ပြင်းထန်မှုဆိုင်ရာ ပရောက်စီဖြစ်သည်။ ရေကူးသမားအများစုသည် ကျောင်းဟောင်း မီတာ 1500 "အချိန်-အစမ်း" အရှိန်အဟုန်ကို အားကိုးနေကြသော်လည်း ခေတ်မီ လေ့ကျင့်ရေးသည် အဆိုပါနေရာသို့ ပြောင်းရွှေ့သွားခဲ့သည်။ Two-Distance CSS Model.

အဓိက ကွာခြားချက်ကတော့ တိကျမှုနှင့် ရှုမြင်မှု အားထုတ်မှု. ရှည်လျားသောအားထုတ်မှုတစ်ခုတည်းအပေါ်အခြေခံသည့် T-Pace သည် သင်၏ဇီဝဖြစ်စဉ်အဆင့်ထက် သင့်စိတ်ပိုင်းဆိုင်ရာ တုန်လှုပ်မှုကို မကြာခဏ ဖမ်းယူသည်။ သင်၏ "Pace-Duration Curve" တွင် မတူညီသောအချက်နှစ်ချက်မှ တွက်ချက်ထားသော CSS သည် သင့်အင်ဂျင်၏ အမှန်တကယ် အေရိုးဗစ်ကန့်သတ်ချက်ကို ဖော်ပြသည်။

မက်ထရစ် နှိုင်းယှဉ်မှု

Threshold Pace (T-Pace)

"ရိုးရှင်း" မက်ထရစ်

T-Pace သည် အများအားဖြင့် 1500m သို့မဟုတ် 1000m အမြင့်ဆုံးအားထုတ်မှုမှ ဆင်းသက်လာသည်။ ဆက်တိုက်ရေကူးခြင်းအတွက် ၎င်းသည် သင်၏ "Redline" ကို ကိုယ်စားပြုသည်။

  • အားသာချက်များ နားလည်ရန်ရိုးရှင်း။ စာမေးပွဲတစ်ခု၊ နံပါတ်တစ်ခု။ ရှည်လျားပြီး အဆက်မပြတ် အတွဲများ အတွက် အထူးကောင်းမွန်ပါသည်။
  • အားနည်းချက်များ- အရှိန်အဟုန်ဖြင့် အမှားအယွင်းများအတွက် အလွန်အကဲဆတ်သည်။ ပင်ပန်းနွမ်းနယ်မှုအောက်တွင် မည်မျှ လျင်မြန်စွာ ပျောက်ပျက်သွားသည်ကို မတွက်ဆပါ။ ရေကူးသူတိုင်းကို တူညီသော ဇီဝကမ္မဆိုင်ရာ ပရိုဖိုင်ရှိခြင်းကဲ့သို့ တူညီသော မီတာ ၁၅၀၀ နှုန်းဖြင့် ဆက်ဆံပါ။

အရေးကြီးသော ရေကူးအမြန်နှုန်း (CSS)

"Functional" မက်ထရစ်

CSS သည် အကွာအဝေး နှစ်ခုကို အခြေခံ၍ သင်္ချာဆိုင်ရာ ဆင်းသက်လာမှုတစ်ခု (ပုံမှန်အားဖြင့် မီတာ 400 နှင့် 200 မီတာ အကွာအဝေး) ဖြစ်သည်။

  • အားသာချက်များ အရှိန်အဟုန် ဘက်လိုက်မှုကို ဖယ်ရှားသည်။ ကြားကာလများတွင် သင့်အပြုအမူကို ပိုမိုကောင်းမွန်စွာ ခန့်မှန်းနိုင်သည်။ ရည်ရွယ်ချက် "အေရိုးဗစ်မျက်နှာကျက်" ကိုပေးသည်။
  • အားနည်းချက်များ- သီးခြားစမ်းသပ်မှုနှစ်ခုလိုအပ်သည်။ အနည်းငယ်ပိုရှုပ်ထွေးသောသင်္ချာ (Swim Analytics တွင် အလိုအလျောက်တွက်ချက်သည်)။ တသမတ်တည်း အနားယူခြင်းဖြင့် စမ်းသပ်မှုများ ပြုလုပ်ရန် လိုအပ်သည်။

မြစ်ဝကျွန်းပေါ်ဒေသက ဘာကြောင့် အရေးကြီးတာလဲ။

"Delta" သည် သင်၏ မီတာ 400 နှင့် သင်၏ မီတာ 200 အကြား ခြားနားချက်ဖြစ်သည်။ ရေကူးသမားနှစ်ဦးစလုံးတွင် T-Pace သည် 1:40/100m ရှိနိုင်သော်လည်း ၎င်းတို့၏ CSS ပရိုဖိုင်များသည် ဤကဲ့သို့ဖြစ်နေနိုင်သည်-

ရေကူးသမား A ("ဒီဇယ်")

မြစ်ဝကျွန်းပေါ်ဒေသငယ် မီတာ ၂၀၀ မှ ၄၀၀ မီတာကြား။ ဤရေကူးသမားသည် အလွန်မြင့်မားသော အေရိုးဗစ် စွမ်းဆောင်ရည်ရှိသော်လည်း အပြေးနှုန်း နိမ့်သည်။ ၎င်းတို့၏ CSS သည် ၎င်းတို့၏ T-Pace နှင့် အလွန်နီးစပ်ပါသည်။

ရေကူးသမား B ("စပရင်တာ")

မြစ်ဝကျွန်းပေါ်ဒေသကြီး။ မီတာ 200 အရှိန်ကောင်းသော်လည်း မီတာ 400 တွင် သိသိသာသာ ကျဆင်းသွားသည်။ ၎င်းတို့၏ CSS သည် ၎င်းတို့၏ T-Pace အကြံပြုထားသည်ထက် နိမ့်နေမည်ဖြစ်သည်။ ယေဘူယျ "T-Pace" တွင် လေ့ကျင့်ရန် ကြိုးစားပါက ၎င်းတို့၏ anaerobic စနစ်သည် အဆက်မပြတ် ချက်ပြုတ်နေမည်ဖြစ်သည်။

📈 CSS ဖော်မြူလာ

CSS = (D1 - D2) / (T1 - T2)

D1/D2 သည် အကွာအဝေးနှစ်ခုဖြစ်ပြီး T1/T2 သည် စက္ကန့်ပိုင်းအတွင်း အကြိမ်များဖြစ်သည်။ ၎င်းသည် သင်၏ ကနဦး anaerobic surge ၏ ဆူညံသံမပါဘဲ သင်၏ ရေရှည်တည်တံ့သော အရှိန်အဟုန်၏ "လျှောစောက်" ကို ဖော်ပြသည်။

CSS Test ကို ဘယ်လိုလုပ်ဆောင်ရမလဲ (တစ်ဆင့်ပြီးတစ်ဆင့်)

သင်၏ CSS ကို တွက်ချက်ရန်၊ သင်သည် မတူညီသော ရက်များတွင် အများဆုံး စမ်းသပ်မှု နှစ်ခုကို လုပ်ဆောင်ရန် လိုအပ်သည်-

  1. သွေးပူလေ့ကျင့်ခန်း: သင့်ခန္ဓာကိုယ်ကို ပြင်ဆင်ရန်အတွက် စေ့စေ့စပ်စပ်၊ စံပြုသွေးပူခြင်း (ပုံမှန်အားဖြင့် မီတာ 400 မှ 600 မီတာအထိ ရောစပ်ထားသော လေဖြတ်ခြင်းနှင့် တည်ဆောက်ခြင်း) ကို ပြုလုပ်ပါ။
  2. စမ်းသပ်မှု 1 (400m အများဆုံးအားထုတ်မှု)- မီတာ 400 အမြန်ဆုံးရေကူးပါ။ အချိန်အတိအကျ မှတ်တမ်းတင်ပါ။ ပထမဆုံး မီတာ 100 မှာ မီးမလောင်ရအောင် သင်ကိုယ်တိုင် အရှိန်မြှင့်လိုက်ပါ။
  3. အသက်ဝင်သော ပြန်လည်ရယူခြင်း- 100m မှ 200m အထိ ရေကူးပြီး lactic acid ကို ဖယ်ရှားရန် အလွန်လွယ်ကူပြီး 5-10 မိနစ် အနားယူပါ။
  4. စမ်းသပ်မှု 2 (200m အများဆုံးအားထုတ်မှု)- အမြင့်ဆုံးအားစိုက်ထုတ်ပြီး မီတာ 200 ရေကူးပါ။ အချိန်အတိအကျ မှတ်တမ်းတင်ပါ။
  5. တွက်ချက်ရန်- သင်၏အချိန်များကို စက္ကန့်များအဖြစ်ပြောင်းပြီး အထက်ဖော်ပြပါဖော်မြူလာကိုအသုံးပြုပါ သို့မဟုတ် သင်၏ 100m အရှိန်နှုန်းရရှိရန် Swim Analytics တွင် CSS ဂဏန်းတွက်စက်ကိရိယာကို အသုံးပြုပါ။

ဘယ်ဟာကို သုံးသင့်သလဲ

အကယ်၍ Threshold Pace (T-Pace) ကိုသုံးပါ-

  • သင်သည် အရှိန်အဟုန်ကို အခြေခံသော လေ့ကျင့်မှုဖြင့် စတင်နေပြီဖြစ်သည်။
  • သင်သည် ရှည်လျားပြီး ဆက်တိုက်ရေကူးခြင်းကိုသာ လုပ်ဆောင်ပါ (Open water 3km+)။
  • စမ်းသပ်မှုများစွာကို သင်မကိုင်တွယ်ချင်ပါ။

အကယ်၍ Critical Swim Speed ​​(CSS) ကိုသုံးပါ-

  • သုံးရက်မြောက် အားကစားသမားများနှင့် မာရသွန်ရေကူးသမားများ "ဖယောင်းတိုင်ကို မီးရှို့ခြင်း" ရှောင်ရန် ပြိုင်ပွဲတစ်ခုတွင် စောလွန်းသည်။
  • ကြားကာလလေ့ကျင့်ရေး- အမှန်တကယ်အလုပ်လုပ်သော သီးခြား CSS + 2s သို့မဟုတ် CSS - 1s ဇုန်များကို သတ်မှတ်ရန်။
  • ခြေရာခံခြင်း တိုးတက်မှု- CSS သည် ယေဘူယျ 1000m စမ်းသပ်မှုထက် အေရိုးဗစ်တိုးတက်မှုအတွက် ပိုထိခိုက်လွယ်သည်။

နံပါတ်၊ ၎င်းတို့သည် ဆက်စပ် သတ်မှတ်ချက် သဘောတရားများ ဖြစ်သော်လည်း ၎င်းတို့သည် တူညီသော စမ်းသပ်မှု မဟုတ်ပါ။ Threshold pace သည် များသောအားဖြင့် ပိုရှည်သော ရေကူးခြင်းမှ လာလေ့ရှိပြီး CSS ကို ကြိုးပမ်းမှု နှစ်ခုမှ တွက်ချက်ပြီး ကြားကာလ အလုပ်အတွက် နောက်ထပ် ထပ်ခါတလဲလဲ လုပ်နိုင်သော ကျောက်ဆူးကို ပေးပါသည်။

triathlon နှင့် open-water pacing အတွက် ဘယ်ဟာ ပိုကောင်းလဲ။

CSS သည် အများအားဖြင့် ပိုမိုကောင်းမွန်သော အစမှတ်ဖြစ်ပြီး အချိန်တိုအတွင်း အရှိန်အဟုန်ပြင်းစွာ မရရှိဘဲ ရေရှည်တည်တံ့သော အရှိန်အဟုန်ကို ထင်ဟပ်နေသောကြောင့် ဖြစ်သည်။ Threshold pace သည် ရေရှည်တည်ငြိမ်သောရေကူးခြင်းအတွက် အသုံးဝင်နိုင်ဆဲဖြစ်သော်လည်း CSS သည် pacing targets နှင့် zone work အဖြစ်သို့ ပြောင်းလဲရန် ပိုမိုလွယ်ကူပါသည်။

အဆင့်သတ်မှတ်နှုန်းမှ CSS အခြေပြု ရေကူးဇုန်များသို့ မည်သို့ရွှေ့ရမည်နည်း။

CSS ကိုသတ်မှတ်ရန် သင်၏နောက်ထပ် 400m နှင့် 200m စမ်းသပ်မှုကို အသုံးပြုပါ၊ ထို့နောက် ထိုနံပါတ်တစ်ဝိုက်တွင် ကျောက်ချသည့်ကြားကာလကို လုပ်ဆောင်ပါ။ သင် T-Pace ဖြင့် လေ့ကျင့်ပြီးပါက၊ နံပါတ်နှစ်ခုလုံးကို နှိုင်းယှဉ်ပြီး ထပ်ခါတလဲလဲ အေရိုးဗစ် အရှိန်ဖြင့် သင့်အစုံကို ပြန်လည်တည်ဆောက်ရန် CSS လမ်းညွှန်နှင့် ရေကူးလေ့ကျင့်ရေးဇုန်များကို အသုံးပြုပါ။

သင်၏ရေကူးကန်ဇုန်များကို ခန့်မှန်းခြင်းကို ရပ်လိုက်ပါ။
