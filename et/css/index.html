Kriitiline ujumiskiirus (CSS)

Andmepõhise ujumistreeningu alus

Mis on kriitiline ujumiskiirus (CSS)?

Kriitiline ujumiskiirus (CSS)on teoreetiline maksimaalne ujumiskiirus, mida saate säilitada ilma kurnatuseta. See esindab teie aeroobset läve tempot, mis vastab tavaliselt 4 mmol/L vere laktaadile ja püsib umbes 30 minutit. CSS arvutatakse 400 m ja 200 m ajasõidu abil, et määrata isikupärastatud treeningtsoonid.

Kriitiline ujumiskiirus (CSS) tähistab teoreetilist maksimaalset ujumiskiirust, mida saate pidevalt ilma kurnatuseta hoida. See on teie aeroobne lävi vees – intensiivsus, kus laktaadi tootmine võrdub laktaadi kliirensiga.

🎯 Füsioloogiline tähtsus

CSS vastab täpselt:

  • Laktaadi lävi 2 (LT2)- Teine ventilatsioonilävi
  • Maksimaalne laktaadi püsiseisund (MLSS)- Kõrgeim jätkusuutlik laktaaditase
  • Funktsionaalne lävitempo (FTP)- Ujumine on samaväärne rattasõiduga FTP
  • ~4 mmol/L vere laktaati- Traditsiooniline OBLA marker

Miks CSS on oluline?

CSS onalusmõõdikmis avab kogu täiustatud treeningkoormuse analüüsi:

  • Treeningtsoonid:Isikupärastab intensiivsustsoone vastavalt teie füsioloogiale
  • sTSS arvutus:Võimaldab täpse treeningstressi skoori kvantifitseerida
  • CTL/ATL/TSB:Nõutav jõudlusjuhtimise diagrammi mõõdikute jaoks
  • Edenemise jälgimine:Aeroobse vormi parandamise objektiivne mõõt
⚠️ Kriitiline sõltuvus:Ilma kehtiva CSS-testita ei saa arvutada täiustatud treeningkoormuse mõõdikuid (sTSS, CTL, ATL, TSB). Ebatäpne CSS rikub kogu järgneva koolitusanalüüsi.

Kuidas arvutada CSS-i nelja lihtsa sammuga

📋 Standardprotokoll

  1. Soojendus

    300–800 m lihtne ujumine, harjutused ja järkjärguline ülesehitamine, et valmistuda maksimaalseks pingutuseks.

  2. 400 m ajasõit

    Maksimaalne püsiv pingutus tõuke algusest (sukeldumata). Salvestage aeg sekundini. Eesmärk: kiireim jätkusuutlik 400m.

  3. Täielik taastumine

    5-10 minutitkerge ujumine või täielik puhkus. See on täpsete tulemuste jaoks kriitilise tähtsusega.

  4. 200 m ajasõit

    Maksimaalne pingutus tõuke algusest. Salvestage aeg täpselt. See peaks olema 100 m kohta kiirem kui 400 m.

⚠️ Levinud vead

Ebapiisav taastumine

Probleem:Väsimus aeglustab kunstlikult 200m aega

Tulemus:Arvutatud CSS muutubkiiremini kui tegelikkus, mis viib ületreenitud tsoonideni

Lahendus:Puhka, kuni pulss langeb alla 120 löögi minutis või kuni hingamine on täielikult taastunud

Kehv tempo 400 m peal

Probleem:Liiga kiire käivitamine põhjustab dramaatilist aeglustumist

Tulemus:400 m aeg ei peegelda tõeliselt jätkusuutlikku tempot

Lahendus:Püüdke saavutada ühtlast lõhenemist või negatiivset lõhet (teine 200 m ≤ esimene 200 m)

Sukeldumiskäivituste kasutamine

Probleem:Lisab ~0,5-1,5 sekundilise eelise, kallutab arvutused

Lahendus:Kasutage alati seinast tõukekäivitust

🔄 Kordustestimise sagedus

Testige CSS-i uuesti iga kord6-8 nädalattreeningtsoone värskendada, kui sobivus paraneb. Treeninguga kohanedes peaksid teie tsoonid järk-järgult kiiremaks muutuma.

CSS-i arvutusvalem

Valem

CSS (m/s) = (D2 – D1) / (T2 – T1)

Kus:

  • D₁ = 200 meetrit
  • D₂ = 400 meetrit
  • T₁ = aeg 200 m (sekundites)
  • T₂ = aeg 400 m (sekundites)

Lihtsustatud tempo jaoks 100 m kohta

CSS tempo/100 m (sekundites) = (T₄00 - T₂00) / 2

Töötatud näide

Testi tulemused:

  • 400m aeg:6:08(368 sekundit)
  • 200m aeg:2:30(150 sekundit)

1. samm: arvutage CSS m/s

CSS = (400–200) / (368–150)
CSS = 200/218
CSS =0,917 m/s

2. samm: teisendage tempoks 100 m kohta

Tempo = 100 / 0,917
Tempo = 109 sekundit
Tempo =1:49 100 m kohta

Märkus.Ujumises mõõdetakse tempot kuiaeg distantsi kohta. Seetõttu asuurem protsent = aeglasem tempoja amadalam protsent = kiirem tempo. See on vastupidine rattasõidule/jooksmisele, kus suurem % = suurem pingutus.

🎯 Tsoonipõhised treeningu eelised

CSS-põhiste tsoonide kasutamine muudab subjektiivse "tunde" treeningu objektiivseks korratavaks treeninguks. Iga tsoon on suunatud konkreetsetele füsioloogilistele kohanemistele:

  • 2. tsoon:Ehitage aeroobne mootor (60-70% nädala mahust)
  • 3. tsoon:Suurendage võistlustempo efektiivsust (15–20% mahust)
  • 4. tsoon:Lükake laktaadi lävi kõrgemale (10-15% mahust)
  • 5. tsoon:Arendada tipptasemel kiirust ja võimsust (5–10% mahust)

🔬 Miks CSS töötab?

CSS tähistab piiriraskete ja raskete treeningute valdkondades. CSS-i all on laktaadi tootmine ja kliirens tasakaalus – võite ujuda pikema aja jooksul. Üle CSS-i koguneb laktaat järk-järgult kuni ammendumiseni 20-40 minuti jooksul.

See muudab CSS-i ideaalseks intensiivsuseks:

  • Jätkusuutliku võistlustempo määramine 800–1500 m ürituste jaoks
  • Läve intervalltreeningu määramine
  • Aeroobse vormi paranemise jälgimine
  • Treeningkoormuse ja taastumisvajaduste arvutamine

Kuidas oma CSS-i testida

1. samm: soojendage korralikult

Läbige 300–800 meetrit lihtsat ujumist, sealhulgas harjutusi ja järkjärgulisi harjutusi. See valmistab teie keha ette maksimaalseks pingutuseks ja väldib vigastusi.

2. samm: sooritage 400 m ajasõit

Ujuge tõukestardist (sukeldumata) maksimaalse püsiva pingutusega 400 meetrit. Salvestage oma aeg sekundini. Püüdke saavutada ühtlast lõhenemist või negatiivset vahet (teine ​​200 m on võrdne või kiirem kui esimene 200 m).

3. samm: taastage täielikult

Puhka 5-10 minutit kerge ujumise või täieliku puhkusega. See taastumine on kriitilise tähtsusega. Enne jätkamist oodake, kuni teie pulss langeb alla 120 löögi minutis ja hingamine on täielikult taastunud.

4. samm: sooritage 200 m ajasõit

Ujuge tõukestardist 200 meetrit maksimaalse pingutusega. Pange oma aeg täpselt kirja. See peaks olema 100 m kohta kiirem kui teie 400 m tempo.

5. samm: arvutage oma CSS

Kasutage valemit: CSS Pace/100m = (T400 - T200) / 2. Näide: (368s - 150s) / 2 = 109s = 1:49/100m. Või kasutage sellel lehel olevat kalkulaatorit, et saada koheseid tulemusi isikupärastatud treeningtsoonidega.

Praktilised rakendused

1️⃣ Avage treeningu koormuse mõõdikud

CSS on sTSS intensiivsusteguri arvutamise nimetaja. Ilma selleta ei saa te treeningstressi kvantifitseerida ega jälgida fitnessi/väsimuse trende.

2️⃣ Isikupärastage treeningtsoone

Üldised tempograafikud ei võta arvesse individuaalset füsioloogiat. CSS-põhised tsoonid tagavad, et iga ujuja treenib oma optimaalse intensiivsusega.

3️⃣ Jälgige treeningu edenemist

Testige uuesti iga 6-8 nädala järel. CSS-i (kiirem tempo) parandamine näitab edukat aeroobset kohanemist. Seisev CSS soovitab koolitusvajadust kohandada.

4️⃣ Ennustage võistlustulemusi

CSS-i tempo on ligikaudne teie jätkusuutlikule 30-minutilisele võistlustempole. Kasutage seda realistlike eesmärkide seadmiseks 800 m, 1500 m ja avaveeüritustele.

5️⃣ Disainläve treeningud

Klassikalised CSS-i komplektid: 8×100 @ CSS tempo (15 s puhkust), 5×200 @ 101% CSS (20 s puhkust), 3×400 @ 103% CSS (30 s puhkust). Suurendage laktaadi kliirensi võimet.

6️⃣ Optimeerige koonuse strateegiat

Jälgige CSS-i enne ja pärast kitsenemist. Edukas koonus säilitab või parandab veidi CSS-i, vähendades samas väsimust (suurenenud TSB).
