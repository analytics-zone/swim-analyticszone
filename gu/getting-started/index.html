સ્વિમ ઍનલિટિક્સ સાથે પ્રારંભ કરવું

સ્વિમિંગ પ્રદર્શન, CSS પરીક્ષણ અને તાલીમ લોડ એનાલિટિક્સ ટ્રૅક કરવા માટેની તમારી સંપૂર્ણ માર્ગદર્શિકા

ડેટા-ડ્રિવન સ્વિમિંગમાં આપનું સ્વાગત છે

સ્વિમ ઍનલિટિક્સ તમારા સ્વિમિંગ વર્કઆઉટ્સને ક્રિયાત્મક આંતરદૃષ્ટિનો ઉપયોગ કરીને પરિવર્તિત કરે છેક્રિટિકલ સ્વિમ સ્પીડ (CSS),ટ્રેનિંગ સ્ટ્રેસ સ્કોર (sTSS), અનેપર્ફોર્મન્સ મેનેજમેન્ટ ચાર્ટ (PMC)મેટ્રિક્સ આ માર્ગદર્શિકા તમને 4 સરળ પગલાઓમાં પ્રથમ સેટઅપથી અદ્યતન તાલીમ લોડ વિશ્લેષણ સુધી લઈ જશે.

ઝડપી શરૂઆત (5 મિનિટ)

1

ડાઉનલોડ અને ઇન્સ્ટોલ કરો

App Store પરથી સ્વિમ ઍનલિટિક્સ ડાઉનલોડ કરો અને Apple Health ઍક્સેસ કરવાની પરવાનગી આપો. એપ્લિકેશન સ્વિમિંગ વર્કઆઉટ્સને આપમેળે સમન્વયિત કરે છે - મેન્યુઅલ લોગિંગની જરૂર નથી.

એપ ડાઉનલોડ કરો →
2

CSS ટેસ્ટ કરો

તમારી ક્રિટિકલ સ્વિમ સ્પીડ સ્થાપિત કરવા માટે 400m અને 200m સમયની અજમાયશ પૂર્ણ કરો. આ તમામ મેટ્રિક્સનો પાયો છે—CSS વિના, sTSS અને તાલીમ ઝોનની ગણતરી કરી શકાતી નથી.

CSS ટેસ્ટ પ્રોટોકોલ ↓
3

CSS પરિણામો દાખલ કરો

એપ્લિકેશનમાં તમારા 400m અને 200m વખત ઇનપુટ કરો. સ્વિમ ઍનલિટિક્સ CSS, પેસ ઝોનની ગણતરી કરે છે અને તમારા શરીરવિજ્ઞાન માટે તમામ મેટ્રિક્સને વ્યક્તિગત કરે છે. દર 6-8 અઠવાડિયે અપડેટ કરો કારણ કે ફિટનેસ સુધરે છે.

4

ટ્રૅકિંગ વર્કઆઉટ્સ શરૂ કરો

Apple Watch અને Health એપ વડે સ્વિમ કરો. સ્વિમ એનાલિટિક્સ આપમેળે વર્કઆઉટ્સ આયાત કરે છે, sTSS ની ગણતરી કરે છે, CTL/ATL/TSB અપડેટ કરે છે અને પ્રગતિને ટ્રેક કરે છે. કોઈ મેન્યુઅલ ડેટા એન્ટ્રીની જરૂર નથી.

CSS ટેસ્ટિંગ પ્રોટોકોલ પૂર્ણ કરો

📋 તમને જે જોઈએ છે

  • પૂલ ઍક્સેસ:25m અથવા 50m પૂલ (25yd સ્વીકાર્ય)
  • સમય:સ્ટોપવોચ, પેસ ક્લોક અથવા એપલ વોચ
  • વોર્મ-અપ સમય:પરીક્ષણ પહેલાં 15-20 મિનિટ
  • પુનઃપ્રાપ્તિ:ટ્રાયલ વચ્ચે 5-10 મિનિટ
  • પ્રયાસ:મહત્તમ ટકાઉ ગતિ (ઓલ-આઉટ સ્પ્રિન્ટ નહીં)

⏱️ ટેસ્ટ દિવસની શરતો

  • આરામ કર્યો:24-48 કલાક પહેલાં સખત તાલીમ નહીં
  • હાઇડ્રેટેડ:સારી રીતે હાઇડ્રેટેડ, સામાન્ય આહાર
  • પૂલ તાપમાન:26-28°C (79-82°F) આદર્શ - ખૂબ ઠંડુ અથવા ગરમ પાણી પ્રભાવને અસર કરી શકે છે અને પરિણામોને ત્રાંસુ કરી શકે છે
  • દિવસનો સમય:જ્યારે તમે સામાન્ય રીતે શ્રેષ્ઠ તાલીમ આપો છો
  • સાધન:તાલીમની જેમ જ (ગોગલ્સ, કેપ, સૂટ)

સ્ટેપ-બાય-સ્ટેપ CSS ટેસ્ટ

વોર્મ-અપ

15-20 મિનિટ

400-800m સરળ સ્વિમિંગ, કવાયત અને પ્રગતિશીલ બિલ્ડ-અપ્સ. વધતી ગતિએ 2-3×50 શામેલ કરો (60%, 75%, 85% પ્રયાસ). ટેસ્ટ પહેલા 2-3 મિનિટ આરામ કરો.

અજમાયશ 1

400m મહત્તમ પ્રયત્નો

શરૂઆત દબાણ કરો(ડાઈવ નહીં). સૌથી ઝડપી ગતિએ 400m તરીને તમે સંપૂર્ણ અંતર સુધી ટકી શકો. આ સ્પ્રિન્ટ નથી: સમગ્ર અંતર માટે તમારી જાતને ગતિ આપો. માં રેકોર્ડ સમયmm:ssફોર્મેટ (દા.ત., 6:08).

પેસિંગ ટીપ:100m સ્પ્લિટ્સ માટે પણ લક્ષ્ય રાખો. બીજું 200m હોવું જોઈએ ≤ પ્રથમ 200m (નકારાત્મક વિભાજન આદર્શ).
પુનઃપ્રાપ્તિ

5-10 મિનિટ

જટિલ તબક્કો:સરળ સ્વિમિંગ અથવા સંપૂર્ણ આરામ. હૃદયના ધબકારા 120 bpm ની નીચે ન આવે અને શ્વાસ સંપૂર્ણપણે સ્વસ્થ થાય ત્યાં સુધી રાહ જુઓ. લક્ષ્ય: વાતચીત શ્વાસ અને અન્ય મહત્તમ પ્રયાસ માટે તૈયાર લાગણી. અપૂરતી પુનઃપ્રાપ્તિ = અચોક્કસ CSS.

અજમાયશ 2

200m મહત્તમ પ્રયત્નો

શરૂઆત દબાણ કરો(ડાઈવ નહીં). 200m માટે મહત્તમ ટકાઉ પ્રયત્નો. આ 400 મીટર કરતાં 100 મીટર દીઠ સખત લાગવું જોઈએ. માં રેકોર્ડ સમયmm:ssફોર્મેટ (દા.ત., 2:30).

માન્યતા તપાસ:200m ગતિ/100m 400m ગતિ/100m કરતાં 3-6 સેકન્ડ વધુ ઝડપી હોવી જોઈએ. જો નહિં, તો પુનઃપ્રાપ્તિ અપૂરતી હતી અથવા ગતિ બંધ હતી.
કૂલ-ડાઉન

10-15 મિનિટ

300-500m સરળ સ્વિમિંગ, સ્ટ્રેચિંગ. તમારો સમય તરત જ રેકોર્ડ કરો - મેમરી પર વિશ્વાસ ન કરો.

⚠️ CSS ટેસ્ટની સામાન્ય ભૂલો

  • 400m પર ખૂબ ઝડપથી બહાર જવું:બ્લોઅપમાં પરિણામો, અચોક્કસ CSS. સમ પેસિંગનો ઉપયોગ કરો.
  • ટ્રાયલ વચ્ચે અપૂરતી પુનઃપ્રાપ્તિ:થાક 200m ધીમો પાડે છે, CSSને કૃત્રિમ રીતે ઝડપી બનાવે છે → ઓવરટ્રેઇન્ડ ઝોન.
  • ડાઇવનો ઉપયોગ શરૂ થાય છે:0.5-1.5 સેકન્ડનો ફાયદો ઉમેરે છે, ગણતરીઓ સ્કૂઇંગ કરે છે. હંમેશા દિવાલથી દબાણ કરો.
  • થાક લાગે ત્યારે પરીક્ષણ:ભારે તાલીમ લોડ 24-48 કલાક પહેલાં = હતાશ પરિણામો. તાજી થાય ત્યારે ટેસ્ટ કરો.
  • તરત જ રેકોર્ડિંગ નથી:મેમરી અવિશ્વસનીય છે. કૂલ-ડાઉન પહેલાં સમય લખો.

સ્વિમ ઍનલિટિક્સમાં CSS પરિણામો દાખલ કરવું

પગલું 1: CSS સેટિંગ્સ ખોલો

સ્વિમ એનાલિટિક્સ એપ્લિકેશનમાં, પર જાઓસેટિંગ્સ → ક્રિટિકલ સ્વિમ સ્પીડ. "CSS ટેસ્ટ કરો" અથવા "CSS અપડેટ કરો" પર ટૅપ કરો.

પગલું 2: ઇનપુટ ટાઇમ્સ

તમારો 400m સમય દાખલ કરો (દા.ત.,6:08) અને 200m સમય (દા.ત.,2:30). બતાવેલ ચોક્કસ ફોર્મેટનો ઉપયોગ કરો. "ગણતરી કરો" ને ટેપ કરો.

પગલું 3: પરિણામોની સમીક્ષા કરો

એપ્લિકેશન દર્શાવે છે:

  • CSS ઝડપ:0.917 m/s
  • CSS ગતિ:1:49/100 મી
  • તાલીમ ક્ષેત્રો:7 વ્યક્તિગત ઝોન (ઝોન 1-7)
  • sTSS આધારરેખા:હવે તમામ વર્કઆઉટ્સ માટે સક્ષમ છે

પગલું 4: સાચવો અને સમન્વયિત કરો

"સેવ CSS" ને ટેપ કરો. એપ્લિકેશન તરત જ:

  • તાલીમ ઝોનની પુનઃ ગણતરી કરે છે
  • પાછલા 90 દિવસો માટે sTSS ને પૂર્વવર્તી રીતે અપડેટ કરે છે
  • CTL/ATL/TSB ગણતરીઓને સમાયોજિત કરે છે
  • ઝોન-આધારિત વર્કઆઉટ વિશ્લેષણને સક્ષમ કરે છે

💡 પ્રો ટીપ: ઐતિહાસિક CSS પરીક્ષણ

જો તમે પહેલાથી જ તમારા CSSને અગાઉના પરીક્ષણોથી જાણો છો, તો તમે તે સમયને સીધા જ ઇનપુટ કરી શકો છો. જો કે, સૌથી સચોટ પરિણામો માટે, દર 6-8 અઠવાડિયે એક નવેસરથી પરીક્ષણ કરો. જેમ જેમ તાલીમ આગળ વધે તેમ તેમ તમારું CSS સુધરવું જોઈએ (ઝડપી થવું).

તમારા મેટ્રિક્સને સમજવું

ક્રિટિકલ સ્વિમ સ્પીડ (CSS)

તે શું છે:તમારી એરોબિક થ્રેશોલ્ડ ગતિ—સૌથી ઝડપી ગતિ જે તમે થાક વિના ~30 મિનિટ સુધી જાળવી શકો છો.

તેનો અર્થ શું છે:CSS = 1:49/100m એટલે કે તમે સતત થ્રેશોલ્ડ પ્રયત્નો માટે 1:49 ગતિ પકડી શકો છો.

કેવી રીતે ઉપયોગ કરવો:તમામ તાલીમ ઝોન અને sTSS ગણતરી માટેનો આધાર. દર 6-8 અઠવાડિયામાં અપડેટ કરો.

તાલીમ ઝોન

તેઓ શું છે:તમારા CSS પર આધારિત 7 તીવ્રતા રેન્જ, પુનઃપ્રાપ્તિ (ઝોન 1) થી સ્પ્રિન્ટ (ઝોન 7) સુધી.

તેઓનો અર્થ શું છે:દરેક ઝોન ચોક્કસ શારીરિક અનુકૂલન (એરોબિક બેઝ, થ્રેશોલ્ડ, VO₂max) ને લક્ષ્ય બનાવે છે.

કેવી રીતે ઉપયોગ કરવો:માળખાગત તાલીમ માટે ઝોન પ્રિસ્ક્રિપ્શનોને અનુસરો. એપ્લિકેશન દરેક વર્કઆઉટ માટે સમય-ઇન-ઝોન બતાવે છે.

સ્વિમિંગ ટ્રેનિંગ સ્ટ્રેસ સ્કોર (sTSS)

તે શું છે:તીવ્રતા અને અવધિને સંયોજિત વર્કઆઉટ તણાવની માત્રા. CSS ગતિએ 1 કલાક = 100 sTSS.

તેનો અર્થ શું છે:sTSS 50 = સરળ પુનઃપ્રાપ્તિ, sTSS 100 = મધ્યમ, sTSS 200+ = ખૂબ સખત સત્ર.

કેવી રીતે ઉપયોગ કરવો:તાલીમ લોડનું સંચાલન કરવા માટે દૈનિક/સાપ્તાહિક sTSS ને ટ્રૅક કરો. મહત્તમ દર અઠવાડિયે 5-10 sTSS વધારવાનું લક્ષ્ય રાખો.

sTSS માર્ગદર્શિકા →

CTL / ATL / TSB

તેઓ શું છે:

  • CTL:ક્રોનિક ટ્રેનિંગ લોડ (ફિટનેસ) - 42-દિવસ સરેરાશ sTSS
  • ATL:એક્યુટ ટ્રેનિંગ લોડ (થાક) - 7-દિવસ સરેરાશ sTSS
  • TSB:ટ્રેનિંગ સ્ટ્રેસ બેલેન્સ (ફોર્મ) = CTL - ATL

કેવી રીતે ઉપયોગ કરવો:સકારાત્મક TSB = તાજા/ટેપર્ડ, નકારાત્મક TSB = થાકેલું. જ્યારે TSB = +5 થી +25 હોય ત્યારે રેસ.

📊 તમારા પ્રથમ સપ્તાહના લક્ષ્યો

CSS દાખલ કર્યા પછી અને 3-5 વર્કઆઉટ્સ પૂર્ણ કર્યા પછી:

  • sTSS મૂલ્યો તપાસો:ખાતરી કરો કે તેઓ પ્રયત્નોની ધારણા સાથે મેળ ખાય છે (સરળ ~50, મધ્યમ ~100, સખત ~150+)
  • ઝોન વિતરણની સમીક્ષા કરો:શું તમે ઝોન 2 (એરોબિક બેઝ) માં 60-70% ખર્ચ કરો છો?
  • બેઝલાઇન CTL સ્થાપિત કરો:તમારા પ્રથમ સપ્તાહની સરેરાશ sTSS પ્રારંભિક ફિટનેસ બેઝલાઇન બની જાય છે
  • પેટર્ન ઓળખો:કયા વર્કઆઉટ્સ સૌથી વધુ sTSS જનરેટ કરે છે? શું તમે પર્યાપ્ત રીતે સ્વસ્થ થઈ રહ્યા છો?

સામાન્ય વપરાશકર્તા જર્ની (પ્રથમ 8 અઠવાડિયા)

અઠવાડિયું 1-2: બેઝલાઇન સ્થાપિત કરો

  • CSS પરીક્ષણ કરો અને પરિણામો દાખલ કરો
  • 3-5 સામાન્ય તાલીમ વર્કઆઉટ્સ પૂર્ણ કરો
  • sTSS મૂલ્યો અને ઝોન વિતરણનું અવલોકન કરો
  • પ્રારંભિક CTL (ફિટનેસ સ્તર) સ્થાપિત કરો
  • ધ્યેય:મેટ્રિક્સ સમજો, હજુ સુધી કોઈ ફેરફાર નથી

અઠવાડિયું 3-4: ઝોન લાગુ કરો

  • વર્કઆઉટ પ્લાનિંગમાં CSS ઝોનનો ઉપયોગ કરો
  • એરોબિક સેટ માટે ઇરાદાપૂર્વક ઝોન 2 તરવું
  • સાપ્તાહિક sTSS ટોટલને ટ્રૅક કરો (સતતતા માટેનું લક્ષ્ય)
  • મોનિટર TSB (થોડું નકારાત્મક = તાલીમ હોવું જોઈએ)
  • ધ્યેય:ઝોન દ્વારા ટ્રેન, લાગણી નથી

અઠવાડિયું 5-6: પ્રગતિશીલ ઓવરલોડ

  • બેઝલાઇનથી સાપ્તાહિક sTSS ને 5-10% વધારો
  • Add 1 threshold (Zone 4) session per week
  • CTL ધીમે ધીમે વધવું જોઈએ (ફિટનેસ સુધારી રહ્યું છે)
  • ATL સખત અઠવાડિયામાં વધી શકે છે (સામાન્ય)
  • ધ્યેય:નિયંત્રિત ફિટનેસ પ્રગતિ

અઠવાડિયું 7-8: ફરીથી પરીક્ષણ કરો અને ગોઠવો

  • બીજી CSS ટેસ્ટ કરો (ઝડપી હોવી જોઈએ)
  • એપ્લિકેશનમાં ઝોન અપડેટ કરો (ગતિ સુધારે છે)
  • CTL અઠવાડિયું 1 વિ અઠવાડિયું 8 ની તુલના કરો (+10-20 હોવી જોઈએ)
  • પ્રગતિની સમીક્ષા કરો: શું સમય ઘટી રહ્યો છે? સરળ લાગે છે?
  • ધ્યેય:તાલીમની અસરકારકતાને માન્ય કરો

✅ સફળતા સૂચકાંકો

સ્વિમ એનાલિટિક્સ સાથે 8 અઠવાડિયાની સંરચિત તાલીમ પછી, તમારે જોવું જોઈએ:

  • CSS સુધારણા:1-3% ઝડપી CSS ગતિ (દા.ત., 1:49 → 1:47)
  • CTL વધારો:+15-25 પોઈન્ટ (દા.ત., 30 → 50 CTL)
  • સુસંગત sTSS:10-15% તફાવતની અંદર સાપ્તાહિક કુલ
  • વધુ સારી પેસિંગ:વધુ સમાન વિભાજન, વધુ સારા પ્રયત્નો કેલિબ્રેશન
  • સુધારેલ પુનઃપ્રાપ્તિ:TSB ચક્ર અનુમાનિત રીતે (-10 થી +5)

મુશ્કેલીનિવારણ અને FAQs

વર્કઆઉટ પ્રયત્નો માટે મારું sTSS ખૂબ ઊંચું/નીચું લાગે છે

કારણ:CSS જૂનું અથવા અચોક્કસ છે.

ઉકેલ:CSSનું ફરીથી પરીક્ષણ કરો. જો તમે થાકેલા અથવા નબળી ગતિએ પરીક્ષણ કર્યું હોય, તો CSS ખોટું હશે. તમામ ડાઉનસ્ટ્રીમ મેટ્રિક્સ માટે યોગ્ય CSS પરીક્ષણ મહત્વપૂર્ણ છે.

એપ્લિકેશન "કોઈ CSS ગોઠવેલ નથી" બતાવે છે

કારણ:CSS પરીક્ષણ પૂર્ણ થયું નથી અથવા સાચવ્યું નથી.

ઉકેલ:સેટિંગ્સ → ક્રિટિકલ સ્વિમ સ્પીડ → પરફોર્મ ટેસ્ટ પર જાઓ. 400m અને 200m બંને વખત દાખલ કરો, પછી સાચવો પર ટૅપ કરો.

એપલ વોચમાંથી વર્કઆઉટ્સ સમન્વયિત થતા નથી

કારણ:હેલ્થ એપની પરવાનગી આપવામાં આવી નથી અથવા વર્કઆઉટને "સ્વિમિંગ" તરીકે વર્ગીકૃત કરવામાં આવ્યું નથી.

ઉકેલ:સેટિંગ્સ તપાસો → ગોપનીયતા → આરોગ્ય → સ્વિમ વિશ્લેષણ → વર્કઆઉટ્સ માટે વાંચવાની મંજૂરી આપો. ખાતરી કરો કે Apple Watch વર્કઆઉટ પ્રકાર "પૂલ સ્વિમ" અથવા "ઓપન વોટર સ્વિમ" છે.

સતત તાલીમ છતાં CTL વધતું નથી

કારણ:sTSS કુલ ખૂબ ઓછી અથવા અસંગત આવર્તન.

ઉકેલ:CTL એ 42-દિવસની એક્સપોનેન્શિયલ વેઇટેડ એવરેજ છે. તે ધીમે ધીમે વધે છે. સાપ્તાહિક sTSS માં 5-10% વધારો, અને સતત CTL વૃદ્ધિ માટે 4+ વર્કઆઉટ્સ/અઠવાડિયે જાળવો.

મારે કેટલી વાર CSS રિટેસ્ટ કરવું જોઈએ?

ભલામણ:બેઝ/બિલ્ડ તબક્કા દરમિયાન દર 6-8 અઠવાડિયે, પીક સીઝન દરમિયાન દર 4-6 અઠવાડિયે. માંદગી, ઈજા, લાંબો વિરામ, અથવા જ્યારે ઝોન સતત ખૂબ સરળ/અઘરું લાગે ત્યારે ફરીથી પરીક્ષણ કરો. ભારે તાલીમ અઠવાડિયા દરમિયાન અથવા સ્પર્ધાના 10 દિવસની અંદર પરીક્ષણ કરવાનું ટાળો.

શું હું અન્ય સ્ટ્રોક માટે સ્વિમ એનાલિટિક્સનો ઉપયોગ કરી શકું?

હા, મર્યાદાઓ સાથે:CSS નું સામાન્ય રીતે ફ્રીસ્ટાઇલમાં પરીક્ષણ કરવામાં આવે છે. IM/બેકસ્ટ્રોક/બ્રેસ્ટસ્ટ્રોક વર્કઆઉટ માટે, sTSS ની ગણતરી ફ્રીસ્ટાઇલ CSSના આધારે કરવામાં આવે છે. વધુ ચોકસાઈ માટે સ્ટ્રોક-વિશિષ્ટ CSS પરીક્ષણો કરવાનું વિચારો.
