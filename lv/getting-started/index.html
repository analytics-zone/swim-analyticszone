Darba sākšana ar Swim Analytics

Pilnīgs ceļvedis peldēšanas veiktspējas izsekošanai, CSS testēšanai un treniņu slodzes analīzei

Laipni lūdzam uz datiem balstītā peldēšanā

Swim Analytics pārveido jūsu peldēšanas treniņus praktiski izmantojamos ieskatos, izmantojotKritiskais peldēšanas ātrums (CSS),Treniņa stresa rādītājs (sTSS), unVeiktspējas pārvaldības diagramma (PMC)metriku. Šajā rokasgrāmatā 4 vienkāršās darbībās tiks parādīta informācija no pirmās iestatīšanas līdz uzlabotai treniņu slodzes analīzei.

Ātrais sākums (5 minūtes)

1

Lejupielādēt un instalēt

Lejupielādējiet Swim Analytics no App Store un piešķiriet atļauju piekļūt Apple Health. Programma automātiski sinhronizē peldēšanas treniņus — nav nepieciešama manuāla reģistrēšana.

Lejupielādēt lietotni →
2

Veiciet CSS testu

Pabeidziet 400 m un 200 m laika izmēģinājumu, lai noteiktu savu kritisko peldēšanas ātrumu. Tas ir visu rādītāju pamats — bez CSS sTSS un treniņu zonas nevar aprēķināt.

CSS pārbaudes protokols ↓
3

Ievadiet CSS rezultātus

Ievadiet 400 m un 200 m laikus lietotnē. Swim Analytics aprēķina CSS, tempa zonas un personalizē visus rādītājus atbilstoši jūsu fizioloģijai. Atjauniniet ik pēc 6–8 nedēļām, kad fiziskā sagatavotība uzlabojas.

4

Sāciet treniņu izsekošanu

Peldieties, izmantojot Apple Watch un lietotni Health. Swim Analytics automātiski importē treniņus, aprēķina sTSS, atjaunina CTL/ATL/TSB un izseko progresam. Nav nepieciešama manuāla datu ievade.

Pabeigt CSS testēšanas protokolu

📋 Kas jums nepieciešams

  • Piekļuve baseinam:25 m vai 50 m baseins (pieņemams 25 m)
  • Laiks:Hronometrs, tempa pulkstenis vai Apple Watch
  • Iesildīšanās laiks:15-20 minūtes pirms testa
  • Atgūšana:5-10 minūtes starp izmēģinājumiem
  • Piepūle:Maksimāli ilgtspējīgs temps (nevis pilnīgs sprints)

⏱️ Testa dienas nosacījumi

  • Atpūties:Bez smagas apmācības 24-48 stundas iepriekš
  • Hidratēts:Labi hidratēts, normāla ēšana
  • Baseina temperatūra:26-28°C (79-82°F) ideāli - ļoti auksts vai silts ūdens var ietekmēt veiktspēju un izkropļot rezultātus
  • Diennakts laiks:Kad parasti trenējies vislabāk
  • Aprīkojums:Tāpat kā treniņā (brilles, cepure, uzvalks)

Soli pa solim CSS pārbaude

Iesildīšanās

15-20 minūtes

400–800 m viegla peldēšana, vingrinājumi un pakāpeniska celšana. Iekļaujiet 2–3 × 50 pieaugošā tempā (60%, 75%, 85% piepūles). Pirms testa atpūtieties 2-3 minūtes.

1. izmēģinājums

400 m maksimālā piepūle

Push start(bez niršanas). Peldiet 400 m visātrākajā tempā, ko varat izturēt visu distanci. Šis NAV sprints: veiciet tempu visas distances garumā. Ierakstīt laikumm:ssformātā (piemēram, 6:08).

Padoms par ritmu:Mērķējiet pat 100 m šķelšanās. Otrajiem 200 m jābūt ≤ pirmajiem 200 m (ideāls negatīvais sadalījums).
Atveseļošanās

5-10 minūtes

KRITISKĀ FĀZE:Viegla peldēšana vai pilnīga atpūta. Pagaidiet, līdz sirdsdarbība nokrītas zem 120 sitieniem minūtē un elpošana pilnībā atjaunojas. Mērķis: sarunvalodas elpošana un gatavības sajūta vēl vienam maksimālam piepūlei. Nepietiekama atkopšana = neprecīzs CSS.

2. izmēģinājums

200 m maksimālā piepūle

Push start(bez niršanas). Maksimāli ilgtspējīga piepūle 200 m. Tam vajadzētu justies grūtāk uz 100 m nekā 400 m. Ierakstīt laikumm:ssformātā (piemēram, 2:30).

Validācijas pārbaude:200 m tempam/100 m jābūt par 3-6 sekundēm ātrākam nekā 400 m tempam/100 m. Ja nē, atveseļošanās bija nepietiekama vai temps bija izslēgts.
Atdzesē

10-15 minūtes

300-500m viegla peldēšana, stiepšanās. Nekavējoties ierakstiet laiku — neuzticieties atmiņai.

⚠️ Biežākās CSS pārbaudes kļūdas

  • Pārāk ātri iziet 400 m:Rezultāts ir sprādziens, neprecīzs CSS. Izmantojiet vienmērīgu ritmu.
  • Nepietiekama atveseļošanās starp izmēģinājumiem:Nogurums palēnina 200 m, padarot CSS mākslīgi ātru → pārtrenētas zonas.
  • Niršanas izmantošana sākas:Pievieno 0,5-1,5s priekšrocību, šķību aprēķinus. Vienmēr spiediet no sienas.
  • Pārbaude, kad esat noguris:Liela treniņu slodze 24-48h pirms = depresīvi rezultāti. Pārbaudi, kad tas ir svaigs.
  • Netiek ierakstīts uzreiz:Atmiņa ir neuzticama. Pierakstiet laikus pirms atdzesēšanas.

CSS rezultātu ievadīšana pakalpojumā Swim Analytics

1. darbība. Atveriet CSS iestatījumus

Lietotnē Swim Analytics dodieties uzIestatījumi → Kritiskais peldēšanas ātrums. Pieskarieties "Veikt CSS pārbaudi" vai "Atjaunināt CSS".

2. darbība: ievades laiki

Ievadiet savu 400 m laiku (piem.,6:08) un 200 m laikā (piem.,2:30). Izmantojiet precīzu parādīto formātu. Pieskarieties "Aprēķināt".

3. darbība. Pārskatiet rezultātus

Lietotne parāda:

  • CSS ātrums:0,917 m/s
  • CSS temps:1:49/100m
  • Treniņu zonas:7 personalizētas zonas (1.-7. zona)
  • sTSS bāzes līnija:Tagad iespējots visiem treniņiem

4. darbība. Saglabāt un sinhronizēt

Pieskarieties "Saglabāt CSS". Lietotne nekavējoties:

  • Pārrēķina treniņu zonas
  • Ar atpakaļejošu datumu atjaunina sTSS pēdējo 90 dienu laikā
  • Pielāgo CTL/ATL/TSB aprēķinus
  • Iespējo uz zonām balstītu treniņu analīzi

💡 Pro padoms: vēsturiska CSS pārbaude

Ja jūs jau zināt savu CSS no iepriekšējām pārbaudēm, varat ievadīt šos laikus tieši. Tomēr, lai iegūtu visprecīzākos rezultātus, veiciet jaunu testu ik pēc 6-8 nedēļām. Jūsu CSS ir jāuzlabo (jākļūst ātrākam) apmācības gaitā.

Jūsu metrikas izpratne

Kritiskais peldēšanas ātrums (CSS)

Kas tas ir:Jūsu aerobā sliekšņa temps — ātrākais ātrums, ko varat izturēt ~30 minūtes bez pārguruma.

Ko tas nozīmē:CSS = 1:49/100 m nozīmē, ka varat noturēt 1:49 tempu, ilgstoši cenšoties sasniegt slieksni.

Kā lietot:Pamats visām treniņu zonām un sTSS aprēķiniem. Atjauniniet ik pēc 6-8 nedēļām.

Uzziniet CSS →

Treniņu zonas

Kas tie ir:7 intensitātes diapazoni, pamatojoties uz jūsu CSS — no atkopšanas (1. zona) līdz sprintam (7. zona).

Ko tie nozīmē:Katra zona ir vērsta uz specifiskiem fizioloģiskiem pielāgojumiem (aerobā bāze, slieksnis, VO₂max).

Kā lietot:Sekojiet zonu priekšrakstiem strukturētai apmācībai. Lietotne parāda laiku zonā katram treniņam.

Treniņu zonas →

Peldēšanas apmācības stresa rādītājs (sTSS)

Kas tas ir:Kvantificēts treniņu stress, kas apvieno intensitāti un ilgumu. 1 stunda CSS tempā = 100 sTSS.

Ko tas nozīmē:sTSS 50 = viegla atveseļošanās, sTSS 100 = mērena, sTSS 200+ = ļoti smaga sesija.

Kā lietot:Sekojiet katru dienu/nedēļu sTSS, lai pārvaldītu treniņu slodzi. Mērķējiet 5-10 sTSS palielinājumu nedēļā max.

sTSS rokasgrāmata →

CTL / ATL / TSB

Kas tie ir:

  • CTL:Hroniska treniņu slodze (fitness) - 42 dienu vidējais sTSS
  • ATL:Akūta treniņu slodze (nogurums) - 7 dienu vidējais sTSS
  • TSB:Treniņa stresa līdzsvars (forma) = CTL - ATL

Kā lietot:Pozitīvais TSB = svaigs/konusveida, negatīvs TSB = noguris. Sacensības, kad TSB = +5 līdz +25.

📊 Jūsu pirmās nedēļas mērķi

Pēc CSS ievadīšanas un 3-5 treniņu pabeigšanas:

  • Pārbaudiet sTSS vērtības:Apstipriniet, ka tie atbilst piepūles uztverei (viegli ~50, vidēji ~100, smagi ~150+)
  • Pārskatiet zonu sadalījumu:Vai jūs tērējat 60-70% 2. zonā (aerobā bāze)?
  • Izveidojiet CTL bāzes līniju:Jūsu pirmās nedēļas vidējais sTSS kļūst par sākotnējo fitnesa bāzes līmeni
  • Identificējiet modeļus:Kuri treniņi rada visaugstāko sTSS? Vai atveseļojaties atbilstoši?

Tipisks lietotāja ceļojums (pirmās 8 nedēļas)

1.–2. nedēļa: izveidojiet bāzes līniju

  • Veiciet CSS testu un ievadiet rezultātus
  • Pabeidziet 3-5 parastos treniņu treniņus
  • Ievērojiet sTSS vērtības un zonu sadalījumu
  • Izveidojiet sākotnējo CTL (fitnesa līmeni)
  • Mērķis:Izprotiet metriku, pagaidām nekādu izmaiņu

3.–4. nedēļa: pielietojiet zonas

  • Treniņu plānošanā izmantojiet CSS zonas
  • Apzināti peldiet 2. zonu aerobikas komplektiem
  • Izsekojiet iknedēļas sTSS kopsummas (mērķējiet uz konsekvenci)
  • Monitors TSB (jābūt nedaudz negatīvam = apmācība)
  • Mērķis:Trenējies pa zonām, nevis jūt

5.-6. nedēļa: progresējoša pārslodze

  • Palieliniet iknedēļas sTSS par 5-10% no sākotnējā līmeņa
  • Pievienojiet 1 sliekšņa (4. zonas) sesiju nedēļā
  • CTL vajadzētu pakāpeniski palielināties (fitnesa uzlabošana)
  • ATL var palielināties grūtās nedēļās (parasti)
  • Mērķis:Kontrolēta fitnesa progresēšana

7.–8. nedēļa: atkārtota pārbaude un pielāgošana

  • Veikt otro CSS testu (jābūt ātrākam)
  • Atjaunināt zonas lietotnē (uzlabojas temps)
  • Salīdziniet CTL 1. nedēļu ar 8. nedēļu (jābūt +10-20)
  • Pārskatiet progresu: vai laiks samazinās? Vai jūtaties vieglāk?
  • Mērķis:Apstipriniet apmācības efektivitāti

✅ Veiksmes rādītāji

Pēc 8 nedēļu strukturētas apmācības ar Swim Analytics jums vajadzētu redzēt:

  • CSS uzlabojumi:Par 1–3% ātrāks CSS temps (piemēram, 1:49 → 1:47)
  • CTL pieaugums:+15–25 punkti (piemēram, 30 → 50 CTL)
  • Konsekventa sTSS:Nedēļas kopsummas 10–15% novirzes robežās
  • Labāks temps:Vienmērīgāki sadalījumi, labāka piepūles kalibrēšana
  • Uzlabota atveseļošanās:TSB cikls paredzams (-10 līdz +5)

Problēmu novēršana un bieži uzdotie jautājumi

Mans sTSS šķiet pārāk augsts/zems treniņu piepūlei

Iemesls:CSS ir novecojis vai neprecīzs.

Risinājums:Atkārtoti pārbaudiet CSS. Ja pārbaudījāt, kad esat noguris vai slikts temps, CSS būs nepareizs. Pareiza CSS pārbaude ir būtiska visiem pakārtotajiem rādītājiem.

Lietotne parāda "Nav konfigurēta CSS"

Iemesls:CSS pārbaude nav pabeigta vai nav saglabāta.

Risinājums:Dodieties uz Iestatījumi → Kritiskais peldēšanas ātrums → Veikt pārbaudi. Ievadiet 400 m un 200 m laikus, pēc tam pieskarieties Saglabāt.

Treniņi netiek sinhronizēti no Apple Watch

Iemesls:Veselības lietotnei nav piešķirtas atļaujas vai treniņam nav piešķirta kategorija "Peldēšana".

Risinājums:Pārbaudiet Iestatījumi → Konfidencialitāte → Veselība → Peldēšanas analīze → Atļaut lasīt treniņiem. Pārliecinieties, vai Apple Watch treniņa veids ir “Peldēšana baseinā” vai “Peldēšana atklātā ūdenī”.

CTL nepalielinās, neskatoties uz konsekventu apmācību

Iemesls:sTSS kopējā frekvence ir pārāk zema vai nekonsekventa.

Risinājums:CTL ir 42 dienu eksponenciāli svērtais vidējais rādītājs. Tas paceļas lēnām. Palieliniet iknedēļas sTSS par 5–10% un uzturiet 4+ treniņus nedēļā, lai nodrošinātu konsekventu CTL izaugsmi.

Cik bieži man vajadzētu atkārtoti pārbaudīt CSS?

Ieteikums:Ik pēc 6–8 nedēļām bāzes/būvēšanas fāzēs, ik pēc 4–6 nedēļām sezonas laikā. Atkārtoti pārbaudiet pēc slimības, traumas, ilga pārtraukuma vai tad, kad zonas jūtas pārāk vieglas/grūtas. Izvairieties no testēšanas smagu treniņu nedēļu laikā vai 10 dienu laikā pēc sacensībām.

Vai es varu izmantot Swim Analytics citiem sitieniem?

Jā, ar ierobežojumiem:CSS parasti tiek pārbaudīts brīvajā stilā. IM/uz muguras/brūts treniņiem sTSS tiek aprēķināts, pamatojoties uz brīvā stila CSS. Lai iegūtu lielāku precizitāti, apsveriet iespēju veikt insultam raksturīgus CSS testus.
