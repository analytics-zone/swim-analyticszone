Критична швидкість плавання (CSS)

Основа навчання плаванню на основі даних

Що таке критична швидкість плавання (CSS)?

Критична швидкість плавання (CSS)це теоретична максимальна швидкість плавання, яку ви можете підтримувати без виснаження. Він представляє ваш пороговий аеробний темп, який зазвичай відповідає 4 ммоль/л лактату в крові та зберігається протягом приблизно 30 хвилин. CSS розраховується за допомогою гонок на час на 400 і 200 метрів для визначення персоналізованих тренувальних зон.

Критична швидкість плавання (CSS) представляє теоретичну максимальну швидкість плавання, яку ви можете підтримувати постійно без виснаження. Це ваш аеробний поріг у воді — інтенсивність, при якій вироблення лактату дорівнює кліренсу лактату.

🎯 Фізіологічне значення

CSS точно відповідає:

  • Лактатний поріг 2 (LT2)- Другий вентиляційний поріг
  • Максимальний рівноважний стан лактату (MLSS)- Найвищий стійкий рівень лактату
  • Функціональний пороговий темп (FTP)- Плавання еквівалентно їзді на велосипеді FTP
  • ~4 ммоль/л лактату крові- Традиційний маркер OBLA

Чому CSS важливий

CSS - цефундаментальна метрикаякий розблоковує весь розширений аналіз тренувального навантаження:

  • Тренувальні зони:Персоналізує зони інтенсивності відповідно до вашої фізіології
  • Розрахунок sTSS:Забезпечує точну кількісну оцінку тренувального стресу
  • CTL/ATL/TSB:Необхідний для показників діаграми керування ефективністю
  • Відстеження прогресу:Об’єктивна міра покращення аеробної підготовленості
⚠️ Критична залежність:Без дійсного тесту CSS не можна розрахувати показники навантаження для розширеного навчання (sTSS, CTL, ATL, TSB). Неточний CSS зіпсує весь подальший аналіз навчання.

Як розрахувати CSS за 4 простих кроки

📋 Стандартний протокол

  1. Розминка

    Легке плавання на 300–800 м, вправи та поступове нарощування, щоб підготуватися до максимальних зусиль.

  2. Гонка на час 400 м

    Максимальне тривале зусилля від старту від поштовху (без занурення). Запис часу з точністю до секунди. Мета: найшвидший стійкий 400 м.

  3. Повне одужання

    5-10 хвилинлегкого плавання або повного відпочинку. Це КРИТИЧНО ВАЖЛИВО для точних результатів.

  4. Гонка на час на 200 метрів

    Максимальне зусилля від поштовху. Точний запис часу. Це має бути швидше на 100 м, ніж на 400 м.

рішення:Відпочиньте, поки ЧСС не впаде нижче 120 уд/хв або поки дихання повністю не відновиться

проблема:Занадто швидкий старт викликає різке сповільнення

рішення:Прагніть до рівномірних розривів або від’ємного розриву (другі 200 м ≤ перші 200 м)

🔄 Частота повторного тестування

Кожного разу тестуйте CSS6-8 тижнівоновлювати зони тренувань у міру покращення фізичної форми. Ваші зони повинні поступово ставати швидшими в міру адаптації до тренувань.

Формула розрахунку CSS

Формула

CSS (м/с) = (D₂ - D₁) / (T₂ - T₁)

Де:

  • D₁ = 200 метрів
  • D₂ = 400 метрів
  • T₁ = Час на 200 м (у секундах)
  • T₂ = Час на 400 м (у секундах)

Спрощено для темпу на 100 м

Темп CSS/100 м (секунд) = (T₄₀₀ - T₂₀₀) / 2

Спрацьований приклад

Результати тесту:

  • 400 м час:6:08(368 секунд)
  • 200 м час:2:30(150 секунд)

Крок 1: обчисліть CSS у м/с

CSS = (400 - 200) / (368 - 150)
CSS = 200/218
CSS =0,917 м/с

Крок 2: Перетворення в темп на 100 м

Темп = 100 / 0,917
Темп = 109 секунд
Темп =1:49 на 100м

🎯 Переваги зонального навчання

Використання зон на основі CSS перетворює суб’єктивне «відчуття» тренування в об’єктивне, повторюване тренування. Кожна зона націлена на певні фізіологічні адаптації:

  • Зона 2:Створення аеробного двигуна (60-70% тижневого обсягу)
  • Зона 3:Підвищення ефективності гоночного темпу (15-20% обсягу)
  • Зона 4:Підвищити лактатний поріг (10-15% об'єму)
  • Зона 5:Розвивайте максимальну швидкість і потужність (5-10% від обсягу)

0,9-1,2 м/с

<0,9 м/с

Побудова аеробної бази та техніки. Менше 1-2 років постійного навчання.

Фундаментальні дослідження Kohji Wakayoshi в Університеті Осаки показали, що CSS є дійсною практичною альтернативою лабораторному тестуванню лактату:

  • Сильна кореляція з VO₂ на анаеробному порозі(r = 0,818)
  • Чудова кореляція зі швидкістю на OBLA(r = 0,949)
  • Прогнозує продуктивність на 400 м(r = 0,864)
  • Відповідає 4 ммоль/л лактату крові- максимальний рівноважний стан лактату
  • Лінійна залежність між відстанню та часом(r² > 0,998)

🔬 Чому CSS працює

CSS представляє межу міждомени важких і важких фізичних навантажень. Нижче CSS виробництво лактату та кліренс залишаються збалансованими — ви можете плавати тривалий час. Понад CSS лактат поступово накопичується до виснаження протягом 20-40 хвилин.

Це робить CSS ідеальною інтенсивністю для:

  • Встановлення стабільного темпу змагань на дистанції 800–1500 метрів
  • Призначення порогових інтервальних тренувань
  • Моніторинг покращення аеробної форми
  • Розрахунок тренувального навантаження та потреб у відновленні

Як перевірити свій CSS

Крок 1. Правильно розігрійтеся

Пройдіть 300-800 метрів легкого плавання, включно з вправами та поступовими нарощуваннями. Це готує ваше тіло до максимальних зусиль і запобігає травмам.

Крок 2: Виконайте гонку на час на 400 метрів

Пропливіть 400 метрів із максимальним постійним зусиллям із поштовху (без пірнання). Записуйте свій час з точністю до секунди. Прагніть до рівномірних шпагатів або негативного шпагату (другі 200 м дорівнюють першим або швидше за перші 200 м).

Крок 3: Повністю відновитись

Відпочинок 5-10 хвилин легким плаванням або повний відпочинок. Це відновлення є КРИТИЧНИМ. Перш ніж продовжувати, зачекайте, поки частота серцевих скорочень впаде нижче 120 ударів на хвилину і дихання повністю відновиться.

Крок 4: Виконайте гонку на час на 200 метрів

Проплисти 200 метрів з максимальним зусиллям з поштовху. Точно записуйте свій час. Це має бути швидше на 100 м, ніж ваш темп на 400 м.

Крок 5: обчисліть свій CSS

Використовуйте формулу: CSS Pace/100m = (T400 - T200) / 2. Приклад: (368s - 150s) / 2 = 109s = 1:49/100m. Або скористайтеся калькулятором на цій сторінці, щоб отримати миттєві результати за допомогою персоналізованих тренувальних зон.

Практичні застосування

1️⃣ Розблокуйте показники тренувального навантаження

CSS є знаменником у розрахунку коефіцієнта інтенсивності для sTSS. Без нього ви не зможете кількісно оцінити стрес під час тренувань або відстежувати тенденції фізичної форми/втоми.

2️⃣ Персоналізуйте тренувальні зони

Загальні діаграми темпу не враховують індивідуальну фізіологію. Зони на основі CSS гарантують, що кожен плавець тренується з оптимальною інтенсивністю.

3️⃣ Відстежуйте прогрес фізичної форми

Перетестуйте кожні 6-8 тижнів. Покращення CSS (швидший темп) свідчить про успішну аеробну адаптацію. Застій CSS говорить про те, що навчання потребує коригування.

4️⃣ Прогнозуйте результати перегонів

Темп CSS приблизно відповідає вашому стійкому темпу 30-хвилинної гонки. Використовуйте його для встановлення реалістичних цілей на дистанціях на 800 м, 1500 м і на відкритій воді.

5️⃣ Розробка порогових тренувань

Класичні набори CSS: 8×100 @ темп CSS (15 секунд відпочинку), 5×200 @ 101% CSS (20 секунд відпочинку), 3×400 @ 103% CSS (30 секунд відпочинку). Збільште здатність очищати лактат.

6️⃣ Оптимізуйте стратегію конусності

Відстежуйте CSS до та після звуження. Успішна конусність зберігає або трохи покращує CSS, одночасно зменшуючи втому (підвищений TSB).
