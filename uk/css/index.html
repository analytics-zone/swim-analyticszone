Критична швидкість плавання (CSS)

Практичний контрольний показник темпу, який фіксує тренувальні зони плавання, набори порогів і аналіз тренувального навантаження

Швидка відповідь

Критична швидкість плавання (CSS) це перевірена в польових умовах оцінка вашого сталого порогового темпу плавання. Більшість плавців обчислюють його за a Гонка на час 400м + 200м, а потім використайте результат для встановлення навчальні зони, набори порогових значень темпу та потужність sTSS і тренувального навантаження аналіз.

  • Найкраще використання: порогове тренування, повторювана стимуляція та відстеження навантаження
  • Типове тлумачення: приблизно найшвидший темп, який ви можете підтримувати аеробно для важкої, контрольованої роботи
  • Не використовуйте його наосліп: погана стимуляція, неповне відновлення, різні умови басейну або засоби перетягування можуть спотворити результат
  • Повторний тест каденції: кожні 6-8 тижнів або після суттєвого тренувального блоку

Ключові висновки

  • Що таке CSS: практична оцінка порогового темпу для плавців, побудована на основі простих польових тестів
  • Чому це важливо: CSS перетворює тренування з плавання з здогадок на повторюваний темп і кращий контроль навантаження
  • Що це відкриває:зони, sTSS, управління продуктивністю, а також більш чіткі перевірки прогресу
  • Чого він не робить: замінити аналіз техніки, контекст відкритої води або інтерпретація ефективності
  • Найкращий наступний крок: добре протестуйте CSS, а потім використовуйте його, щоб структурувати свій поріг і аеробну роботу

Якщо ви зберігаєте лише одну метрику плавання як прив’язку темпу, збережіть CSS. Він не ідеальний і не є повним фізіологічним лабораторним тестом, але практичний, повторюваний і безпосередньо корисний. Саме завдяки цій комбінації CSS залишається основою тренувань з плавання на основі даних.

Для більшості плавців, Критична швидкість плавання є мостом між необробленими випробуваннями на час і щоденними рішеннями щодо тренувань. Це допоможе відповісти на п’ять важливих питань:

  • Наскільки швидко насправді мають бути встановлені порогові значення?
  • Який діапазон темпів належить кожному зона плавання?
  • Який темп повторити 100, 200 і 400 с?
  • Як я кількісно оцінюю стрес під час плавання за допомогою sTSS?
  • Чи мій аеробний плавальний фітнес рухається в правильному напрямку?

Що вимірює CSS

CSS – це практична оцінка темпу жорсткого аеробічного плавання, як правило, виходить із забігу на час на 400 і 200 метрів. У тренерській мові це часто розглядають як ваш пороговий темп для структурованого тренування в басейні: досить швидкий, щоб мати значення, достатньо контрольований, щоб повторювати, і достатньо стабільний, щоб будувати зони.

🎯 Корисна ментальна модель

Вважайте CSS своїм контрольний показник темпу роботи, не магічне число. Це орієнтир темпу, який робить решту ваших показників плавання узгодженими.

Чому CSS важливий

CSS має значення, оскільки він пов’язує фізіологію з виконанням. Багато плавців знають, що вони виконували «важкий сет» або «темповий сеанс», але не можуть сказати, чи був правильний темп. CSS вирішує цю проблему, надаючи вам тест, який ви можете використовувати сеанс за сеансом.

🏊 Тренувальні зони

CSS дає вам чистий якір для зони для плавання, тож легка робота залишається легкою, а порогова робота залишається цільовою.

📊 Тренувальне навантаження

Без придатного для використання CSS, sTSS і показники управління продуктивністю втрачають своє значення.

⏱️ Повторюваний темп

CSS допомагає вам досягати 100, 200 і 400 секунд з більшою послідовністю, а не покладатися лише на відчуття.

📈 Перевірки прогресу

Коли CSS покращується, ваш пороговий темп рухається швидше. Це вагомий сигнал про те, що підготовка до аеробного плавання також покращується.

⚠️ Критична залежність: Якщо ваш CSS неправильний, ваші зони, sTSS і показники керування продуктивністю рухатимуться в неправильному напрямку. Поганий вхід призводить до поганого аналізу.

Для чого корисний CSS, а для чого ні

CSS чудово підходить для: встановлення порогових значень, якість аеробної роботи, налаштування зони та відстеження навантаження, характерного для плавання.

CSS недостатньо для: профілювання спринту, діагностика техніки гребка, виконання гонки на відкритій воді в змінних умовах або самостійна інтерпретація ефективності.

Використовуйте CSS для

  • Приписи порогового темпу
  • Повторіть темп 100/200/400
  • Персоналізований зонна структура
  • sTSS і управління навантаженням
  • Відстеження аеробного прогресу протягом сезону

Не використовуйте лише CSS для

  • Порівняння різних довжин басейнів, ніби вони ідентичні
  • Порівняння плавців із дуже різними профілями техніки
  • Оцінка якості удару без механіка удару
  • Оцінка ефективності без контексту ВОЛЬФ
  • Припущення, що темп у відкритій воді точно відповідатиме темпу CSS басейну

Як перевірити CSS у 4 практичних кроки

📋 Стандартний протокол

  1. Як слід розігрітися

    Легко пропливіть 300-800 м, включіть вправи та додайте кілька контрольованих зусиль перед тестом.

  2. Пропливти гонку на час на 400 метрів

    Використовуйте поштовховий старт, а не занурення. Мета — ваш найшвидший стійкий пробіг на 400 метрів, а не безрозсудна перша сотня.

  3. Повністю одужати

    Прийміть 5-10 хвилин легкий або повний відпочинок. Неповне відновлення – один з найшвидших способів зіпсувати результат.

  4. Пропливти гонку на час на 200 метрів

    Знову скористайтеся кнопкою запуску та точно запишіть час. Ваш темп на 200 метрів має бути значно швидшим, ніж на 400 метрів.

⚠️ Найпоширеніші помилки тестування

Поганий темп на 400 м

проблема: Ви занадто сильно виходите, потім сильно зникаєте.

Результат: 400 м перестає відображати стабільний темп.

Виправити: Прагніть до рівномірного або злегка негативного розподілу.

Недостатнє відновлення

проблема: Втома робить дистанцію 200 метрів штучно сповільненою.

Результат: CSS спотворюється, а ваші навчальні зони дрейфують.

Виправити: Зачекайте, поки дихання не встановиться, і ви відчуєте повне одужання.

Змішані умови

проблема: Різна довжина басейну, значні зміни відштовхування або засоби перетягування змінюють середовище тестування.

Результат: Ви порівнюєте числа, які насправді не порівнюються.

Виправити: Зберігайте умови якомога послідовнішими від тесту до тесту.

🔄 Частота повторного тестування

Кожного разу тестуйте CSS 6-8 тижнів, або після значущого блоку, зосередженого на консолідації порога, витривалості чи техніки. Справа не в постійному тестуванні. Суть полягає в тому, що достатньо послідовно використовувати один контрольний показник, щоб побачити зміни.

Формула розрахунку CSS

Формула

CSS (м/с) = (D₂ - D₁) / (T₂ - T₁)

Де:

  • D₁ = 200 метрів
  • D₂ = 400 метрів
  • T₁ = Час на 200 м (у секундах)
  • T₂ = Час на 400 м (у секундах)

Спрощено для темпу на 100 м

Темп CSS/100 м (секунд) = (T₄₀₀ - T₂₀₀) / 2

Спрацьований приклад

Результати тестування

  • 400 м час: 6:08 (368 секунд)
  • 200 м час: 2:30 (150 секунд)

Крок 1: розрахуйте швидкість CSS

Темп CSS/100 м = (368 - 150) / 2
Темп CSS/100 м = 218 / 2
Темп CSS/100 м = 109 секунд = 1:49 на 100 м

Спрацьований приклад: як використовувати цей результат

Якщо ваш CSS є 1:49/100м, ваше порогове число повторів має бути приблизно таким, як ваш зональний стіл має будуватися з нього, і ваш sTSS розрахунки слід використовувати його як знаменник темпу.

  • Зона 4: де CSS стає безпосередньо корисним для наборів порогів
  • Зона 5: де CSS перестає бути цілою історією, а риси спринту/потужності важливіші

Як використовувати CSS у реальному навчанні

1️⃣ Набори порогових значень темпу

Використовуйте CSS, щоб припинити вгадувати. Такі підходи, як 8x100, 5x200 або 3x400, мають більше сенсу, коли ви знаєте цільовий темп і можете його чітко тримати.

2️⃣ Підключіть Pace до sTSS

CSS надає значення вашому sTSS. Важкі та легкі сесії перестають змішуватися з одним і тим же невизначеним навантаженням.

3️⃣ Краще відстежуйте прогрес

Якщо CSS стає швидшим, а ваші сеанси залишаються під контролем, ваша порогова фізична підготовка до плавання, ймовірно, покращиться.

4️⃣ Створіть кращі звички темпу

CSS вчить вас виконувати повторну роботу з контролем замість того, щоб перетворювати кожну складну серію на згортання.

Приклади ідей для сеансів, орієнтованих на CSS

  • Порогове обслуговування: 8x100 в темпі CSS з коротким відпочинком
  • Витривалість темпу: 5x200 навколо швидкості CSS із послідовним розподілом
  • Аеробна підтримка: 3x400 трохи повільніше, ніж темп CSS з контрольованою кількістю штрихів
  • Техніка + темп: чергуйте 100 м із зусиллям CSS із 50 м із легким технічним скиданням

Це приклади, а не універсальні рецепти. Справа в тому, що CSS дає вам стабільну мову темпу.

CSS, SWOLF і ефективність

CSS говорить вам як швидко ви можете плавати на порозі. SWOLF і показники ефективності сказати вам дещо інше: наскільки економно ви рухаєтеся водою. Вони доповнюють один одного, але не є взаємозамінними.

Не порівнюйте SWOLF наосліп

Нижчий SWOLF не завжди є «кращим» для плавців, сеансів, гребків або налаштувань басейну. використання показники ефективності як контекст разом із CSS, якістю обведення та повторюваністю, а не як окремий інструмент ранжирування.

Наукова перевірка

Wakayoshi та ін. і практичний приклад CSS

CSS став широко використовуватися, тому що він пропонував тренерам і плавцям зручний контрольний тест, який добре відстежувався з пороговими маркерами продуктивності без необхідності повторного лабораторного тестування лактату.

  • Практичний: повторювані з простого тестування пулу
  • Корисно: досить близько до порогової частоти для програмування
  • Діяльність: легко повторно тестувати та оновлювати протягом сезону
  • Підключено: безпосередньо корисний для тренувальних зон, темпу та відстеження навантаження

Часті запитання

Як часто я повинен повторно тестувати CSS?

Кожні 6-8 тижнів є хорошим стандартом. Повторюйте тест раніше, лише якщо тренування сильно змінилося або ваші поточні зони явно здаються неправильними.

Чи CSS те саме, що лактатний поріг?

Не точно. CSS — це польова оцінка, яка використовується як практичний еталон порогового значення темпу. Це корисно, тому що воно повторюване та дієве, а не тому, що воно ідеально замінює лабораторні тести.

Чи можу я використовувати CSS для пейсингу на відкритій воді?

Використовуйте його як вихідний орієнтир, а не як буквальну гарантію. Прицілювання, драфтінг, чоп, гідрокостюми та навігація змінюють поведінку темпу у відкритій воді.

Чи можуть новачки використовувати CSS?

Так, але результат стає кориснішим у міру вдосконалення техніки та темпу. Зовсім новим плавцям часто потрібен постійний гребок і темп, перш ніж CSS стане дуже стабільним.

Чи варто мені безпосередньо порівнювати свій CSS із чужим?

Тільки обережно. Довжина басейну, відштовхування, перетягування, навички темпу та якість техніки впливають на результат. CSS є найпотужнішим як особистий еталон у часі.
