Критична швидкість плавання (CSS) проти порогового темпу: що має значення?

Протягом багатьох років "Threshold Pace" (T-Pace) був золотим стандартом. У 2026 році перехід до критичної швидкості плавання (CSS) переосмислює те, як спортсмени визначають темп своїх тренувань і прогнозують втому під час змагань.

🎯 Швидка відповідь

T-Pace (порогова швидкість) це загальна оцінка темпу, який ви можете підтримувати протягом години, часто використовується для встановлення широких тренувальних зон. Це «практичний», але менш детальний показник.

CSS (критична швидкість плавання) це «функціональний» поріг, отриманий математично з двох різних відстаней (наприклад, 400 м і 200 м). Це теоретичний темп, який ви можете підтримувати, не врізаючись у «стіну». Якщо ви плавець або триатлоніст на витривалість, ваш CSS є найважливішим числом у вашій програмі тренувань.

Зміна 2026: від зусиль до фізіології

Подолати втому в басейні, як відомо, важко порівняно з їздою на велосипеді чи бігом. Без вимірювача потужності темп є нашим єдиним показником інтенсивності. У той час як багато плавців все ще покладаються на стару школу на 1500 метрів у «гонках на час», сучасне навчання перейшло до Модель CSS з двома відстанями.

Головна відмінність полягає в тому Точність проти очікуваних зусиль. T-Pace, заснований на одному тривалому зусиллі, часто фіксує ваші розумові здібності більше, ніж метаболічний поріг. CSS, розрахований за двома різними точками на вашій «кривій темпу й тривалості», показує фактичне аеробне обмеження вашого двигуна.

Метричне порівняння

Пороговий темп (T-Pace)

«Проста» метрика

T-Pace зазвичай розраховується на основі одного максимального зусилля на дистанції 1500 або 1000 метрів. Він представляє вашу «червону лінію» для безперервного плавання.

  • Плюси: Простий для розуміння. Один тест, одне число. Чудово підходить для довгих безперервних серій.
  • Мінуси: Дуже чутливий до помилок ритму. Не враховує, як швидко ви зникаєте від втоми. Ставиться до кожного плавця з однаковим темпом на 1500 метрів, як до тих, хто має однаковий фізіологічний профіль.

Критична швидкість плавання (CSS)

«Функціональна» метрика

CSS — це математичний висновок, заснований на двох відстанях (зазвичай 400 і 200 метрів з максимальним зусиллям).

  • Плюси: Усуває зміщення темпу. Краще прогнозує вашу поведінку в інтервалах. Забезпечує об'єктивну «аеробну стелю».
  • Мінуси: Вимагає двох окремих тестів. Трохи складніша математика (автоматично розраховується в Swim Analytics). Вимагає виконання тестів з постійним відпочинком.

Чому Delta має значення

«Дельта» — це різниця між вашим темпом на 400 і 200 метрів. Два плавці можуть мати T-Pace 1:40/100 м, але їхні профілі CSS можуть виглядати так:

Плавець А ("Дизель")

Невелика дельта між 200 і 400 м. Цей плавець має дуже високу аеробну ефективність, але нижчу швидкість спринту. Їхній CSS дуже близький до T-Pace.

Плавець B ("Спринтер")

Велика дельта. Чудова швидкість на 200 м, але суттєво падає на 400 м. Їх CSS буде нижчим, ніж пропонує T-Pace. Якщо вони намагатимуться тренуватися із стандартним «T-Pace», вони постійно перенапружуватимуть свою анаеробну систему.

📈 Формула CSS

CSS = (D1 - D2) / (T1 - T2)

Де D1/D2 — дві відстані, а T1/T2 — час у секундах. Це показує «нахил» вашого сталого темпу без шуму вашого початкового анаеробного сплеску.

Як виконати тест CSS (покроково)

Щоб розрахувати свій CSS, вам потрібно виконати дві максимальні спроби в різні дні (або з повним відновленням між ними) у стандартному пулі:

  1. Розминка: Виконайте ретельну стандартизовану розминку (зазвичай від 400 до 600 метрів змішаних гребків і налаштувань), щоб підготувати своє тіло.
  2. Випробування 1 (максимальне зусилля 400 м): Пропливіть 400 м якомога швидше. Запишіть точний час. Дотримуйтеся темпу, щоб не згоріти на перших 100 м.
  3. Активне відновлення: Пропливіть від 100 до 200 метрів дуже легко, щоб вимити молочну кислоту, після чого відпочиньте 5–10 хвилин.
  4. Випробування 2 (максимальне зусилля 200 м): Проплисти 200 м з максимальним зусиллям. Запишіть точний час.
  5. обчислити: Перетворіть свій час у секунди та застосуйте наведену вище формулу або скористайтеся інструментом калькулятора CSS у Swim Analytics, щоб визначити свій темп на 100 метрів.

Які ви повинні використовувати?

Використовуйте пороговий темп (T-Pace), якщо:

  • Ви тільки починаєте тренуватися з темпом.
  • Ви виконуєте лише довгі безперервні запливи (відкрита вода 3 км+).
  • Ви не хочете мати справу з кількома сеансами тестування.

Використовуйте критичну швидкість плавання (CSS), якщо:

  • Триатлоністи та плавці-марафонці: Щоб не «спалити свічку» занадто рано в гонці.
  • Інтервальний тренінг: Щоб встановити конкретні зони CSS + 2 або CSS - 1, які дійсно працюють.
  • Відстеження прогресу: CSS більш чутливий до аеробних покращень, ніж звичайний тест на 1000 метрів.

Пороговий темп проти CSS Поширені запитання

Чи є CSS те саме, що пороговий темп?

Ні. Це пов’язані порогові поняття, але це не той самий тест. Пороговий темп зазвичай виходить з одного тривалого запливу, тоді як CSS обчислюється з двох зусиль і дає більш повторюваний якір для інтервальної роботи.

Що краще для триатлону та кроку у відкритій воді?

Зазвичай CSS є кращою відправною точкою, оскільки він відображає стійкий темп без надмірної винагороди за короткострокову швидкість. Пороговий темп все ще може бути корисним для тривалого рівномірного плавання, але CSS зазвичай легше перетворити на цілі темпу та роботу в зоні.

Як мені перейти від порогового темпу до зон плавання на основі CSS?

Використовуйте свій наступний тест на 400 і 200 метрів, щоб встановити CSS, а потім прив’яжіть інтервал до цього числа. Якщо ви вже тренуєтеся з T-Pace, порівняйте обидві цифри та скористайтеся довідником CSS і тренувальними зонами плавання, щоб перебудувати свої підходи навколо повторюваного темпу аеробіки.
