מהירות שחייה קריטית (CSS) לעומת קצב סף: איזה מהם חשוב?

במשך שנים, "קצב הסף" (T-Pace) היה תקן הזהב. בשנת 2026, המעבר למהירות שחייה קריטית (CSS) מגדיר מחדש את האופן שבו ספורטאים מקצבים את האימונים שלהם ומנבאים עייפות ביום המירוץ.

🎯 התשובה המהירה

T-Pace (קצב סף) הוא אומדן כללי של הקצב שאתה יכול לשמור במשך שעה, משמש לעתים קרובות להגדרת אזורי אימון רחבים. זהו מדד "מעשי" אך פחות פרטני.

CSS (מהירות שחייה קריטית) הוא הסף ה"פונקציונלי" הנגזר מתמטית משני מרחקים שונים (למשל, 400 מ' ו-200 מ'). זה הקצב התיאורטי שאתה יכול לעמוד בו מבלי לפגוע ב"קיר". אם אתה שחיין סיבולת או טריאתלט, ה-CSS שלך הוא המספר החשוב ביותר בתוכנית האימונים שלך.

השינוי של 2026: ממאמץ לפיזיולוגיה

ניהול עייפות בבריכה ידוע לשמצה קשה בהשוואה לרכיבה על אופניים או ריצה. ללא מד כוח, הקצב הוא הפרוקסי היחיד שלנו לעוצמה. בעוד שחיינים רבים עדיין מסתמכים על קצבי "מבחן זמן" של 1500 מ', האימון המודרני עבר ל- מודל CSS דו-מרחקים.

ההבדל העיקרי הוא דיוק לעומת מאמץ נתפס. T-Pace המבוסס על מאמץ אחד ארוך לרוב לוכד את החריגות המנטלית שלך יותר מהסף המטבולי שלך. CSS, המחושב משתי נקודות שונות על "עקומת הקצב-משך", חושף את הגבול האירובי האמיתי של המנוע שלך.

השוואה מדדים

קצב סף (T-Pace)

המדד ה"פשוט".

T-Pace נגזר בדרך כלל ממאמץ מרבי בודד של 1500 מ' או 1000 מ'. זה מייצג את "הקו האדום" שלך לשחייה רציפה.

  • יתרונות: פשוט להבנה. מבחן אחד, מספר אחד. נהדר עבור סטים ארוכים ורציפים.
  • חסרונות: רגישות גבוהה לשגיאות קצב. לא מסביר כמה מהר אתה דועך תחת עייפות. מתייחס לכל שחיין באותו קצב של 1500 מטר כמו בעל אותו פרופיל פיזיולוגי.

מהירות שחייה קריטית (CSS)

המדד "פונקציונלי".

CSS היא גזירה מתמטית המבוססת על שני מרחקים (בדרך כלל מאמץ מרבי של 400 מ' ו-200 מ').

  • יתרונות: מסיר הטיית קצב. טוב יותר מנבא את ההתנהגות שלך במרווחים. מספק "תקרה אירובית" אובייקטיבית.
  • חסרונות: דורש שתי בדיקות נפרדות. מתמטיקה קצת יותר מורכבת (חישוב אוטומטי ב-Swim Analytics). דורש שהבדיקות ייעשו במנוחה עקבית.

למה הדלתא חשובה

ה"דלתא" היא ההבדל בין קצב ה-400 מ' שלך לקצב ה-200 מ' שלך. לשני שחיינים אולי יש קצב T של 1:40/100 מ', אבל פרופילי ה-CSS שלהם עשויים להיראות כך:

שחיין A ("הדיזל")

דלתא קטנה בין 200 מ' ל-400 מ'. לשחיין זה יש יעילות אירובית גבוהה מאוד אך מהירות ספרינט נמוכה יותר. ה-CSS שלהם קרוב מאוד ל-T-Pace שלהם.

שחיין B ("האצן")

דלתא גדולה. מהירות נהדרת של 200 מ' אבל יורדת משמעותית ב-400 מ'. ה-CSS שלהם יהיה נמוך ממה שה-T-Pace מציע. אם הם ינסו להתאמן ב-"T-Pace" גנרי, הם כל הזמן יבשלו יתר על המידה את המערכת האנאירובית שלהם.

📈 נוסחת ה-CSS

CSS = (D1 - D2) / (T1 - T2)

כאשר D1/D2 הם שני המרחקים ו-T1/T2 הם הזמנים בשניות. זה חושף את ה"שיפוע" של הקצב בר-קיימא שלך ללא הרעש של הנחשול האנאירובי הראשוני שלך.

כיצד לבצע את מבחן ה-CSS (שלב אחר שלב)

כדי לחשב את ה-CSS שלך, עליך לבצע שני ניסויים מקסימליים בימים שונים (או עם התאוששות מלאה ביניהם) במאגר סטנדרטי:

  1. חימום: בצע חימום יסודי וסטנדרטי (בדרך כלל 400 מ' עד 600 מ' של משיכות ומבנה מעורב) כדי להכין את הגוף שלך.
  2. ניסוי 1 (מאמץ מרבי של 400 מטר): שחו 400 מ' הכי מהר שאפשר. רשום את השעה המדויקת. קצב את עצמך כדי שלא תישרף ב-100 המ' הראשונים.
  3. שחזור פעיל: שחו 100 מ' עד 200 מ' בקלות רבה כדי לשטוף חומצת חלב, ולאחר מכן 5-10 דקות של מנוחה.
  4. ניסוי 2 (200 מ' מאמץ מרבי): שחו 200 מ' במאמץ מרבי מוחלט. רשום את השעה המדויקת.
  5. חשב: המר את הזמנים שלך לשניות והחל את הנוסחה שלמעלה, או השתמש בכלי מחשבון CSS ב-Swim Analytics כדי לקבל את קצב הקצב שלך ל-100 מטר.

באיזה כדאי להשתמש?

השתמש בקצב סף (T-Pace) אם:

  • אתה רק מתחיל עם אימון מבוסס קצב.
  • אתה מבצע רק שחייה ארוכות ורצופות (מים פתוחים 3 ק"מ+).
  • אתה לא רוצה להתמודד עם מפגשי בדיקה מרובים.

השתמש במהירות שחייה קריטית (CSS) אם:

  • טריאתלטים ושחייני מרתון: כדי להימנע מ"שריפת הנר" מוקדם מדי במירוץ.
  • אימון אינטרוולים: כדי להגדיר אזורי CSS + 2s או CSS - 1s ספציפיים שבאמת עובדים.
  • מעקב אחר התקדמות: CSS רגיש יותר לשיפורים אירוביים מאשר בדיקה גנרית של 1000 מטר.

שאלות נפוצות על קצב סף לעומת CSS

האם CSS זהה לקצב הסף?

לא. הם מושגי סף קשורים, אבל הם לא אותו מבחן. קצב הסף מגיע בדרך כלל משחייה אחת ארוכה יותר, בעוד ש-CSS מחושב משני מאמצים ונותן עוגן שניתן לחזור עליו לעבודת אינטרוולים.

מה עדיף לטריאתלון ולקצב במים פתוחים?

CSS היא בדרך כלל נקודת ההתחלה הטובה יותר מכיוון שהיא משקפת קצב בר קיימא מבלי לתגמל יתר על המידה מהירות לטווח קצר. קצב סף עדיין יכול להיות שימושי לשחיות יציבות ארוכות, אבל בדרך כלל קל יותר להמיר CSS למטרות קצב ולעבודה באזור.

איך אני עובר מקצב סף לאזורי שחייה מבוססי CSS?

השתמש במבחן 400 ו-200 מטר הבא שלך כדי להגדיר CSS, ולאחר מכן עגן מרווחי עוגן סביב המספר הזה. אם אתה כבר מתאמן ב-T-Pace, השווה את שני המספרים והשתמש במדריך ה-CSS בתוספת אזורי אימוני שחייה כדי לבנות מחדש את הסטים שלך סביב קצב אירובי שניתן לחזור עליו.
