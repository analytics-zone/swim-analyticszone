중요한 수영 속도(CSS)

수영 훈련 구역, 임계값 세트 및 훈련 부하 분석을 고정하는 실용적인 속도 벤치마크

빠른 답변

중요한 수영 속도(CSS) 지속 가능한 역치 수영 속도에 대한 현장 테스트 추정치입니다. 대부분의 수영선수는 다음과 같이 계산합니다. 400m + 200m 타임 트라이얼, 그 결과를 사용하여 설정 훈련 구역, 페이스 임계값 세트 및 파워 sTSS 그리고 훈련 부하 분석.

  • 최고의 사용: 임계값 훈련, 반복 가능한 속도 및 부하 추적
  • 일반적인 해석: 힘들고 통제된 작업을 위해 유산소 운동으로 유지할 수 있는 대략 가장 빠른 속도
  • 맹목적으로 사용하지 마십시오: 잘못된 페이스, 불완전한 회복, 다른 풀 조건 또는 드래그 보조 장치로 인해 결과가 왜곡될 수 있습니다.
  • 케이던스를 다시 테스트하세요. 6~8주마다 또는 의미 있는 훈련 블록 이후

주요 시사점

  • CSS란 무엇입니까: 간단한 현장 테스트를 통해 구축된 수영 선수를 위한 실제 역치 속도 추정치
  • 중요한 이유: CSS는 수영 훈련을 추측에서 반복 가능한 속도 및 더 나은 부하 제어로 전환합니다.
  • 잠금 해제 내용:구역, sTSS, 성과 관리, 더욱 명확한 진행 상황 확인
  • 하지 않는 일: 기술 분석, 개방 수역 컨텍스트를 대체하거나 효율성 해석
  • 가장 좋은 다음 단계: CSS를 잘 테스트한 다음 이를 사용하여 역치 및 유산소 운동을 구성하세요.

속도 기준으로 하나의 수영 지표만 유지하는 경우 CSS를 유지하세요. 완벽하지도 않고 완전한 생리학적 실험실 테스트도 아니지만 실용적이고 반복 가능하며 직접적으로 유용합니다. 이러한 조합이 CSS가 데이터 기반 수영 훈련의 중추로 남아 있는 이유입니다.

대부분의 수영선수들에게는 중요한 수영 속도 원시 타임 트라이얼과 일상적인 훈련 결정 사이의 다리입니다. 이는 다섯 가지 중요한 질문에 답하는 데 도움이 됩니다.

  • 임계값 세트는 실제로 얼마나 빨라야 합니까?
  • 각각에 속하는 페이스 범위 수영 구역?
  • 100초, 200초, 400초 반복의 속도를 어떻게 맞춰야 하나요?
  • 수영 스트레스를 어떻게 정량화하나요? sTSS?
  • 나의 유산소 수영 체력은 올바른 방향으로 움직이고 있나요?

CSS가 측정하는 것

CSS는 고강도 유산소 수영 속도의 실제 추정치입니다., 일반적으로 400m 및 200m 타임 트라이얼에서 파생됩니다. 코칭 언어에서는 구조화된 풀 트레이닝을 위한 임계 속도로 간주되는 경우가 많습니다. 중요할 만큼 빠르고, 반복할 수 있을 만큼 제어되며, 영역을 구축할 수 있을 만큼 안정적입니다.

측면실용적인 답변
CSS가 측정하는 것지속적인 유산소 운동을 위한 현장 추정 역치 수영 페이스
최고의 사용트레이닝 영역, 임계값 세트, 페이스 목표 설정 및 훈련 부하
흔한 실수하나의 잘못된 테스트를 영구적인 진실로 취급하거나 풀 길이, 휴식 및 속도 품질을 고려하지 않고 CSS를 사용합니다.
관련 지표훈련 구역, sTSS, CTL/ATL/TSB, 그리고 SWOLF / 효율성

🎯 유용한 정신 모델

CSS를 당신의 것으로 생각하십시오 작동 속도 벤치마크, 마법의 숫자가 아닙니다. 나머지 수영 지표를 일관되게 만드는 것은 속도 기준입니다.

CSS가 중요한 이유

CSS는 생리학과 실행을 연결하기 때문에 중요합니다. 많은 수영선수들은 자신이 "하드 세트" 또는 "템포 세션"을 수행했다는 것을 알고 있지만 속도가 올바른지 여부는 말할 수 없습니다. CSS는 세션마다 사용할 수 있는 벤치마크를 제공하여 이 문제를 해결합니다.

🏊 훈련 구역

CSS는 다음에 대한 깨끗한 앵커를 제공합니다. 수영 구역, 따라서 쉬운 작업은 쉽게 유지되고 임계값 작업은 목표로 유지됩니다.

📊 훈련 부하

사용할 수 있는 CSS가 없으면 sTSS 그리고 성과 관리 지표 의미를 잃습니다.

⏱️ 반복 가능한 속도

CSS는 느낌에만 의존하는 대신 100초, 200초, 400초의 속도를 더욱 일관성 있게 유지하는 데 도움이 됩니다.

📈 진행 상황 확인

CSS가 향상되면 임계 속도가 더 빨라집니다. 이는 유산소 수영 체력도 향상되고 있다는 강력한 신호입니다.

⚠️ 중요한 종속성: CSS가 잘못된 경우 영역, sTSS 및 성과 관리 수치가 잘못된 방향으로 표류하게 됩니다. 잘못된 입력은 잘못된 분석을 낳습니다.

CSS가 좋은 점과 그렇지 않은 점

CSS는 다음에 탁월합니다. 임계값 설정, 유산소 수준의 작업, 구역 설정 및 수영별 부하 추적.

CSS는 다음 용도로는 충분하지 않습니다. 스프린트 프로파일링, 뇌졸중 기술 진단, 다양한 조건에서의 오픈 워터 경주 실행 또는 효율성 자체 해석.

CSS 사용

  • 임계 속도 처방
  • 100/200/400 속도 반복
  • 맞춤형 구역 구조
  • sTSS 그리고 부하 관리
  • 시즌 전반에 걸쳐 유산소 진행 상황 추적

CSS를 단독으로 사용하지 마십시오

  • 서로 다른 수영장 길이를 마치 동일한 것처럼 비교
  • 매우 다른 기술 프로필을 가진 수영 선수 비교
  • 없이 스트로크 품질을 판단 뇌졸중 역학
  • 주변 맥락 없이 효율성 판단 스울프
  • 개방 수역 속도가 풀 CSS 속도와 정확히 일치한다고 가정

4가지 실제 단계로 CSS를 테스트하는 방법

📋 표준 프로토콜

  1. 제대로 워밍업을 해라

    300~800m를 쉽게 수영하고, 훈련을 포함하고, 테스트 전에 몇 가지 제어된 빌드 노력을 추가하세요.

  2. 400m 타임 트라이얼 수영

    다이빙이 아닌 푸시 스타트를 사용하십시오. 목표는 무모한 첫 100m가 아닌 가장 빠르고 지속 가능한 400m를 달리는 것입니다.

  3. 완전히 복구

    5~10분 정도 소요 쉽거나 완전한 휴식. 불완전한 복구는 결과를 망치는 가장 빠른 방법 중 하나입니다.

  4. 200m 타임 트라이얼 수영

    다시 푸시 스타트를 사용하고 시간을 정확하게 기록하십시오. 200m 페이스는 400m 페이스보다 의미 있게 더 빨라야 합니다.

⚠️ 가장 일반적인 테스트 오류

400m 속도가 좋지 않음

문제: 당신은 너무 세게 나가서 심하게 퇴색됩니다.

결과: 400m는 지속 가능한 속도 품질을 반영하지 않습니다.

수정: 균등하거나 약간 음수로 분할하는 것을 목표로 하세요.

불충분한 회복

문제: 피로는 200m를 인위적으로 느리게 만듭니다.

결과: CSS가 왜곡되고 훈련 영역이 표류합니다.

수정: 호흡이 조절되고 회복이 완료되었다고 느낄 때까지 기다리십시오.

혼합된 조건

문제: 다양한 수영장 길이, 큰 푸시오프 변경 또는 드래그 보조 장치로 인해 테스트 환경이 변경됩니다.

결과: 실제로 비교할 수 없는 숫자를 비교합니다.

수정: 테스트마다 조건을 가능한 한 일관되게 유지하십시오.

🔄 재테스트 빈도

CSS를 다음 간격으로 다시 테스트하세요. 6~8주, 또는 역치, 지구력 또는 기술 강화에 초점을 맞춘 의미 있는 블록 이후. 요점은 지속적인 테스트가 아닙니다. 요점은 변화를 확인할 수 있을 만큼 일관되게 하나의 벤치마크를 사용하는 것입니다.

CSS 계산 공식

공식

CSS (m/s) = (D² - D₁) / (T² - T₁)

어디에:

  • D₁ = 200미터
  • D² = 400미터
  • T₁ = 200m에 대한 시간(초)
  • T² = 400m를 달리는 시간(초)

100m당 페이스에 대해 단순화됨

CSS 속도/100m(초) = (T₄₀₀ - T₂₀₀) / 2

실제 사례

테스트 결과

  • 400m 시간: 6:08 (368초)
  • 200m 시간: 2:30 (150초)

1단계: CSS 속도 계산

CSS 속도/100m = (368 - 150) / 2
CSS 페이스/100m = 218 / 2
CSS 속도/100m = 109초 = 100m당 1:49

2단계: 필요한 경우 속도로 변환

CSS = 100 / 109
CSS = 0.917m/초

실제 사례: 해당 결과를 사용하는 방법

CSS가 다음과 같은 경우 1:49/100m, 임계값 반복이 해당 속도 근처에 모여야 합니다. 구역 테이블 그것으로부터 구축되어야 하며, sTSS 계산 이를 분모 속도 기준으로 사용해야 합니다.

실제 훈련에서 CSS를 사용하는 방법

1️⃣ 페이스 임계값 세트

추측을 중단하려면 CSS를 사용하세요. 8x100, 5x200 또는 3x400과 같은 세트는 페이스 목표를 알고 이를 깔끔하게 유지할 수 있을 때 더 적합합니다.

2️⃣ 페이스를 sTSS에 연결

CSS는 당신의 sTSS. 어려운 세션과 쉬운 세션은 동일한 모호한 작업 부하에 혼합되지 않습니다.

3️⃣ 진행 상황을 더 잘 추적하세요

세션이 제어되는 동안 CSS가 더 빨라지면 역치 수영 체력이 향상될 가능성이 높습니다.

4️⃣ 더 나은 속도 습관 구축

CSS는 모든 어려운 세트를 속도 붕괴로 바꾸는 대신 컨트롤을 사용하여 반복 작업을 수영하는 방법을 가르칩니다.

샘플 CSS 기반 세션 아이디어

  • 임계값 유지 관리: CSS 속도로 8x100(짧은 휴식 포함)
  • 페이스 내구성: 일관된 분할로 CSS 속도를 중심으로 5x200
  • 유산소 지원: 스트로크 수를 제어하면 CSS 속도보다 약간 느린 3x400
  • 기술 + 속도: 50m의 쉬운 기술 재설정으로 CSS 노력으로 100m를 번갈아 가며

이는 일반적인 처방이 아닌 예입니다. 요점은 CSS가 안정적인 속도 언어를 제공한다는 것입니다.

CSS, SWOLF 및 효율성

CSS가 알려줍니다 얼마나 빨리 문지방에서 수영할 수 있어요. SWOLF 및 효율성 지표 뭔가 다른 것을 말해 보세요. 물 속에서 얼마나 경제적으로 이동하고 있는지 말입니다. 그들은 서로를 보완하지만 상호 교환이 불가능합니다.

SWOLF를 맹목적으로 비교하지 마십시오

낮은 SWOLF가 수영, 세션, 영법 또는 수영장 설정 전반에 걸쳐 항상 "더 나은" 것은 아닙니다. 사용 효율성 지표 독립형 순위 도구가 아닌 CSS, 획 품질 및 반복성과 함께 컨텍스트로 사용됩니다.

과학적 검증

Wakayoshi 외. CSS의 실제 사례

CSS는 코치와 수영 선수에게 반복적인 실험실 젖산 테스트 없이 역치 관련 성과 지표를 잘 추적하는 현장 친화적인 벤치마크를 제공했기 때문에 널리 사용되었습니다.

  • 실용적: 간단한 풀 테스트에서 반복 가능
  • 유용함: 프로그래밍을 위한 임계값 관련 속도에 충분히 가깝습니다.
  • 실행 가능: 시즌 전반에 걸쳐 재테스트 및 업데이트가 용이함
  • 연결됨: 훈련 영역, 속도 및 부하 추적에 직접적으로 유용합니다.

자주 묻는 질문

CSS를 얼마나 자주 다시 테스트해야 합니까?

6~8주마다가 기본값입니다. 훈련이 많이 변경되었거나 현재 영역이 분명히 잘못되었다고 느끼는 경우에만 더 빨리 다시 테스트하십시오.

CSS는 젖산염 역치와 동일합니까?

정확히는 아닙니다. CSS는 실제 임계 속도 벤치마크로 사용되는 필드 추정치입니다. 이는 실험실 테스트를 완벽하게 대체하기 때문이 아니라 반복 가능하고 실행 가능하기 때문에 유용합니다.

오픈워터 속도에 CSS를 사용할 수 있나요?

문자 그대로의 보증이 아닌 시작 참조로 사용하십시오. 관찰, 제도, 절단, 잠수복 및 항해는 모두 개방 수역에서 속도가 작동하는 방식을 변화시킵니다.

초보자도 CSS를 사용할 수 있나요?

예, 하지만 기술과 속도가 향상되면 결과가 더 유용해집니다. 아주 초보 수영 선수들은 CSS가 고도로 안정되기 전에 영법과 속도의 일관성이 필요한 경우가 많습니다.

내 CSS를 다른 사람의 CSS와 직접 비교해야 합니까?

조심스럽게. 풀 길이, 푸시오프, 드래그, 속도 조절 기술 및 기술 품질이 모두 결과에 영향을 미칩니다. CSS는 시간이 지남에 따라 개인 벤치마크로서 가장 강력합니다.
