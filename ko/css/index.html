임계 수영 속도 (CSS)

데이터 기반 수영 훈련의 토대

임계 수영 속도(CSS)란 무엇인가요?

임계 수영 속도(CSS)는 탈진 없이 유지할 수 있는 이론적 최대 수영 속도입니다. 유산소 역치 페이스를 나타내며, 일반적으로 혈중 젖산 4 mmol/L에 해당하고 약 30분 동안 지속할 수 있습니다. CSS는 400m와 200m 타임 트라이얼을 사용하여 개인화된 훈련 존을 결정합니다.

임계 수영 속도(CSS)는 탈진 없이 지속적으로 유지할 수 있는 이론적 최대 수영 속도를 나타냅니다. 물속의 유산소 역치—젖산 생성이 젖산 제거와 균형을 이루는 강도입니다.

생리적 의미

CSS는 다음과 밀접하게 대응합니다:

  • 젖산 역치 2 (LT2) - 두 번째 환기 역치
  • 최대 젖산 정상 상태 (MLSS) - 지속 가능한 최고 젖산 수준
  • 기능적 역치 페이스 (FTP) - 사이클링 FTP에 해당하는 수영
  • 혈중 젖산 ~4 mmol/L - 전통적인 OBLA 마커

CSS가 중요한 이유

CSS는 모든 고급 훈련 부하 분석을 가능하게 하는 핵심 지표입니다:

  • 훈련 존: 개인 생리를 기반으로 강도 존을 개인화
  • sTSS 계산: 정확한 훈련 부하 점수 정량화 가능
  • CTL/ATL/TSB: 퍼포먼스 관리 차트 지표에 필요
  • 진행 상황 추적: 유산소 체력 향상의 객관적 측정
중요 의존성: 유효한 CSS 테스트 없이는 고급 훈련 부하 지표(sTSS, CTL, ATL, TSB)를 계산할 수 없습니다. 부정확한 CSS는 이후 모든 훈련 분석을 왜곡합니다.

4단계로 CSS 계산하는 방법

표준 프로토콜

  1. 워밍업

    최대 노력을 위한 준비로 300-800m의 이지 스윔, 드릴, 점진적 빌드업.

  2. 400m 타임 트라이얼

    푸시 스타트(다이빙 없음)부터 최대 지속 노력. 초 단위로 시간 기록. 목표: 지속 가능한 최대 400m.

  3. 완전한 회복

    5-10분의 이지 스윔 또는 완전 휴식. 정확한 결과를 위해 이것이 매우 중요합니다.

  4. 200m 타임 트라이얼

    푸시 스타트부터 최대 노력. 정확하게 시간 기록. 100m당 페이스가 400m보다 빨라야 합니다.

흔한 실수

불충분한 회복

문제: 피로로 인해 200m 시간이 인위적으로 느려짐

결과: 계산된 CSS가 실제보다 빨라져 과훈련 존으로 이어짐

해결책: 심박수가 120bpm 이하로 떨어지거나 호흡이 완전히 회복될 때까지 휴식

400m에서의 잘못된 페이싱

문제: 너무 빠르게 시작하면 급격한 속도 저하 발생

결과: 400m 시간이 진정한 지속 가능한 페이스를 반영하지 못함

해결책: 균등한 분할 또는 네거티브 스플릿 목표 (후반 200m ≤ 전반 200m)

다이빙 스타트 사용

문제: ~0.5-1.5초 이점을 추가하여 계산 왜곡

해결책: 항상 벽에서 푸시 스타트 사용

재테스트 빈도

체력이 향상됨에 따라 훈련 존을 업데이트하기 위해 6-8주마다 CSS를 재테스트하세요. 훈련에 적응함에 따라 존이 점진적으로 빨라져야 합니다.

CSS 계산 공식

공식

CSS (m/s) = (D₂ - D₁) / (T₂ - T₁)

여기서:

  • D₁ = 200 미터
  • D₂ = 400 미터
  • T₁ = 200m 시간 (초)
  • T₂ = 400m 시간 (초)

100m당 페이스로 단순화

CSS 페이스/100m (초) = (T₄₀₀ - T₂₀₀) / 2

계산 예시

테스트 결과:

  • 400m 시간: 6:08 (368초)
  • 200m 시간: 2:30 (150초)

1단계: m/s로 CSS 계산

CSS = (400 - 200) / (368 - 150)
CSS = 200 / 218
CSS = 0.917 m/s

2단계: 100m당 페이스로 변환

페이스 = 100 / 0.917
페이스 = 109초
페이스 = 100m당 1:49

개인화된 훈련 존

CSS를 기반으로 한 100m당 페이스 훈련 존:

대안 (단순화된 방법):

존 기반 훈련의 장점

CSS 기반 존은 주관적인 '감각' 훈련을 객관적이고 반복 가능한 워크아웃으로 전환합니다. 각 존은 특정 생리적 적응을 목표로 합니다:

  • 존 2: 유산소 엔진 구축 (주간 볼륨의 60-70%)
  • 존 3: 레이스 페이스 효율성 향상 (볼륨의 15-20%)
  • 존 4: 젖산 역치 향상 (볼륨의 10-15%)
  • 존 5: 최고 속도와 파워 개발 (볼륨의 5-10%)

수준별 일반적인 CSS 값

엘리트 장거리 수영 선수

1.5-1.8 m/s
100m당 0:56-1:07

최대 100m 속도의 80-85%를 나타냅니다. 수년간의 구조적 훈련을 받은 국가/국제 수준 선수.

경쟁력 있는 에이지 그룹

1.2-1.5 m/s
100m당 1:07-1:23

고등학교 대표팀, 대학 수영 선수, 경쟁적인 마스터스. 주 5-6일 정기 구조적 훈련.

트라이애슬리트 & 피트니스 수영 선수

0.9-1.2 m/s
100m당 1:23-1:51

주 3-4일 정기 훈련. 탄탄한 기술. 세션당 2000-4000m 완료.

입문 수영 선수

<0.9 m/s
>100m당 1:51

유산소 기초와 기술 구축 중. 1-2년 미만의 지속적인 훈련.

과학적 검증

Wakayoshi et al. (1992-1993) - 기초 연구

오사카 대학의 Kohji Wakayoshi의 획기적인 연구는 CSS를 실험실 젖산 테스트의 유효하고 실용적인 대안으로 확립했습니다:

  • 무산소 역치에서 VO₂와 강한 상관관계 (r = 0.818)
  • OBLA에서 속도와 우수한 상관관계 (r = 0.949)
  • 400m 퍼포먼스 예측 (r = 0.864)
  • 혈중 젖산 4 mmol/L에 해당 - 최대 젖산 정상 상태
  • 거리와 시간 간의 선형 관계 (r² > 0.998)

핵심 논문:

  1. Wakayoshi K, et al. (1992). "Determination and validity of critical velocity as an index of swimming performance in the competitive swimmer." European Journal of Applied Physiology, 64(2), 153-157.
  2. Wakayoshi K, et al. (1992). "A simple method for determining critical speed as swimming fatigue threshold in competitive swimming." International Journal of Sports Medicine, 13(5), 367-371.
  3. Wakayoshi K, et al. (1993). "Does critical swimming velocity represent exercise intensity at maximal lactate steady state?" European Journal of Applied Physiology, 66(1), 90-95.

CSS가 효과적인 이유

CSS는 중등도와 고강도 운동 영역의 경계를 나타냅니다. CSS 이하에서는 젖산 생성과 제거가 균형을 유지하며 장시간 수영이 가능합니다. CSS 이상에서는 20-40분 내에 탈진에 이를 때까지 젖산이 점진적으로 축적됩니다.

이로 인해 CSS는 다음을 위한 완벽한 강도입니다:

  • 800m-1500m 종목의 지속 가능한 레이스 페이스 설정
  • 임계 인터벌 훈련 처방
  • 유산소 체력 향상 모니터링
  • 훈련 부하 및 회복 필요성 계산

CSS 테스트 방법

1단계: 워밍업 실시

드릴과 점진적 빌드업을 포함하여 300-800미터의 이지 스윔을 완료하세요. 이것은 최대 노력을 위한 준비를 하고 부상을 예방합니다.

2단계: 400m 타임 트라이얼 수행

푸시 스타트(다이빙 없음)부터 최대 지속 노력으로 400미터를 수영하세요. 초 단위로 시간을 기록하세요. 균등한 분할 또는 네거티브 스플릿(두 번째 200m가 첫 번째 200m보다 빠르거나 동일)을 목표로 하세요.

3단계: 완전히 회복

이지 스윔이나 완전 휴식으로 5-10분 휴식하세요. 이 회복은 매우 중요합니다. 계속하기 전에 심박수가 120bpm 이하로 떨어지고 호흡이 완전히 회복될 때까지 기다리세요.

4단계: 200m 타임 트라이얼 수행

푸시 스타트부터 최대 노력으로 200미터를 수영하세요. 정확하게 시간을 기록하세요. 100m당 페이스가 400m 페이스보다 빨라야 합니다.

5단계: CSS 계산

공식 사용: CSS 페이스/100m = (T400 - T200) / 2. 예시: (368초 - 150초) / 2 = 109초 = 1:49/100m. 또는 이 페이지의 계산기를 사용하여 개인화된 훈련 존과 함께 즉시 결과를 얻으세요.

실용적인 응용

1. 훈련 부하 지표 잠금 해제

CSS는 sTSS의 강도 계수 계산에서 분모입니다. 이것이 없으면 워크아웃 스트레스를 정량화하거나 체력/피로 추세를 추적할 수 없습니다.

2. 훈련 존 개인화

일반적인 페이스 차트는 개인 생리를 고려하지 않습니다. CSS 기반 존은 모든 수영 선수가 최적의 강도로 훈련하도록 보장합니다.

3. 체력 진행 상황 모니터링

6-8주마다 재테스트하세요. CSS 향상(빠른 페이스)은 성공적인 유산소 적응을 나타냅니다. 정체된 CSS는 훈련 조정이 필요함을 시사합니다.

4. 레이스 퍼포먼스 예측

CSS 페이스는 지속 가능한 30분 레이스 페이스에 근접합니다. 800m, 1500m, 오픈워터 종목의 현실적인 목표 설정에 활용하세요.

5. 임계 워크아웃 설계

클래식 CSS 세트: 8×100 @ CSS 페이스 (15초 휴식), 5×200 @ 101% CSS (20초 휴식), 3×400 @ 103% CSS (30초 휴식). 젖산 제거 능력 구축.

6. 테이퍼 전략 최적화

테이퍼 전후 CSS 추적. 성공적인 테이퍼는 CSS를 유지하거나 약간 향상시키면서 피로를 줄입니다 (증가한 TSB).

CSS 지식 활용하기

임계 수영 속도를 이해했으니, 훈련을 최적화하기 위한 다음 단계를 밟으세요:

  • 훈련 부하 점수(TSS) 계산을 통해 CSS를 기반으로 워크아웃 강도 정량화
  • 7개 훈련 존 탐색하여 특정 적응을 위한 워크아웃 구성 방법 학습
