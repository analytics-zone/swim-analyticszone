임계 수영 속도(CSS)와 임계 속도: 어느 것이 중요합니까?

수년 동안 "Threshold Pace"(T-Pace)가 표준이었습니다. 2026년에는 CSS(Critical Swim Speed)로의 전환으로 인해 운동선수가 세션 속도를 조절하고 경기 중 피로를 예측하는 방법이 재정의되었습니다.

🎯 빠른 답변

T-Pace(역치 페이스) 는 한 시간 동안 유지할 수 있는 속도에 대한 일반적인 추정치로, 넓은 훈련 영역을 설정하는 데 자주 사용됩니다. 이는 "실용적"이지만 덜 세분화된 측정항목입니다.

CSS(중요 수영 속도) 두 개의 서로 다른 거리(예: 400m와 200m)에서 수학적으로 파생된 "기능적" 임계값입니다. 이는 "벽"에 부딪히지 않고 유지할 수 있는 이론적 속도입니다. 지구력 수영 선수나 철인 3종 경기 선수라면 CSS가 훈련 프로그램에서 가장 중요한 숫자입니다.

2026년 전환: 노력에서 생리학으로

수영장에서 피로를 관리하는 것은 사이클링이나 달리기에 비해 매우 어렵습니다. 파워미터가 없으면 속도만이 강도를 나타내는 유일한 지표입니다. 많은 수영선수들이 여전히 전통적인 1500m "타임 트라이얼" 페이스에 의존하고 있지만, 현대 코칭은 2거리 CSS 모델.

주요 차이점은 정확성 대 인지된 노력. 한 번의 오랜 노력을 기반으로 한 T-Pace는 종종 대사 역치보다 정신적인 투지를 더 많이 포착합니다. "속도-지속 시간 곡선"의 두 가지 개별 지점에서 계산된 CSS는 엔진의 실제 유산소성 한계를 나타냅니다.

측정항목 비교

임계 속도(T-Pace)

"단순" 지표

T-Pace는 일반적으로 단일 1500m 또는 1000m 최대 노력에서 파생됩니다. 이는 지속적인 수영을 위한 "레드라인"을 나타냅니다.

  • 장점: 이해하기 간단합니다. 하나의 테스트, 하나의 숫자. 길고 연속적인 세트에 적합합니다.
  • 단점: 속도 오류에 매우 민감합니다. 피로로 인해 얼마나 빨리 퇴색되는지는 설명하지 않습니다. 동일한 1500m 페이스를 가진 모든 수영선수를 동일한 생리학적 프로필을 가진 것으로 간주합니다.

중요한 수영 속도(CSS)

"기능적" 지표

CSS는 두 가지 거리(일반적으로 400m와 200m의 최대 노력)를 기반으로 한 수학적 파생입니다.

  • 장점: 간격 편향을 제거합니다. 일정 간격으로 귀하의 행동을 더 잘 예측합니다. 객관적인 "유산소 천장"을 제공합니다.
  • 단점: 두 가지 별도의 테스트가 필요합니다. 약간 더 복잡한 수학(Swim Analytics에서 자동 계산됨) 지속적인 휴식을 취하면서 테스트를 수행해야 합니다.

델타가 중요한 이유

"델타"는 400m 페이스와 200m 페이스의 차이입니다. 두 명의 수영선수 모두 T-Pace가 1:40/100m일 수 있지만 CSS 프로필은 다음과 같습니다.

수영선수 A ("디젤")

200m에서 400m 사이의 작은 델타. 이 수영선수는 유산소 효율이 매우 높지만 질주 속도는 낮습니다. 그들의 CSS는 T-Pace에 매우 가깝습니다.

수영 선수 B ("단거리 선수")

큰 델타. 200m 속도는 훌륭하지만 400m에서는 속도가 크게 떨어집니다. CSS는 T-Pace가 제안하는 것보다 낮습니다. 일반적인 "T-Pace"로 훈련하려고 하면 무산소 시스템이 지속적으로 과도하게 익게 됩니다.

📈 CSS 공식

CSS = (D1 - D2) / (T1 - T2)

여기서 D1/D2는 두 거리이고 T1/T2는 시간(초)입니다. 이는 초기 무산소 급증의 소음 없이 지속 가능한 속도의 "기울기"를 보여줍니다.

CSS 테스트를 수행하는 방법(단계별)

CSS를 계산하려면 표준 풀에서 서로 다른 날(또는 그 사이에 전체 복구를 포함하여) 두 번의 최대 시험을 수행해야 합니다.

  1. 워밍업: 몸을 준비하기 위해 철저하고 표준화된 워밍업(일반적으로 400m~600m의 혼합 스트로크 및 빌드)을 수행하십시오.
  2. 시험 1(400m 최대 노력): 가능한 한 빨리 400m를 수영하세요. 정확한 시간을 기록하세요. 처음 100m 안에 지치지 않도록 속도를 조절하세요.
  3. 활성 복구: 100m에서 200m를 수영하면 젖산을 쉽게 배출할 수 있으며 5~10분간 휴식을 취합니다.
  4. 시험 2(200m 최대 노력): 절대 최대 노력으로 200m를 수영하세요. 정확한 시간을 기록하세요.
  5. 계산: 시간을 초로 변환하고 위의 공식을 적용하거나 Swim Analytics의 CSS 계산기 도구를 사용하여 100m 속도를 구하세요.

어느 것을 사용해야 합니까?

다음과 같은 경우에는 임계 속도(T-Pace)를 사용하십시오.

  • 이제 막 속도 기반 훈련을 시작하고 있습니다.
  • 길고 연속적인 수영(외부 3km 이상)만 수행합니다.
  • 여러 테스트 세션을 처리하고 싶지 않습니다.

다음과 같은 경우에는 중요 수영 속도(CSS)를 사용하세요.

  • 트라이애슬론 선수 및 마라톤 수영 선수: 경주 초기에 "촛불이 타는" 것을 방지하기 위해.
  • 인터벌 트레이닝: 실제로 작동하는 특정 CSS + 2s 또는 CSS - 1s 영역을 설정합니다.
  • 진행 상황 추적: CSS는 일반적인 1000m 테스트보다 유산소 개선에 더 민감합니다.

임계값 속도와 CSS FAQ

CSS는 임계 속도와 동일합니까?

아니요. 관련된 임계값 개념이지만 동일한 테스트는 아닙니다. 임계 속도는 일반적으로 한 번의 긴 수영에서 나오는 반면, CSS는 두 번의 노력으로 계산되며 인터벌 작업에 더 반복 가능한 기준을 제공합니다.

트라이애슬론과 오픈워터 페이스 중 어느 것이 더 좋나요?

CSS는 단기 속도에 과도한 보상을 주지 않고 지속 가능한 속도를 반영하기 때문에 일반적으로 더 나은 출발점이 됩니다. 임계 속도는 오랫동안 꾸준한 수영에 여전히 유용할 수 있지만 CSS는 일반적으로 속도 목표 및 영역 작업으로 변환하기가 더 쉽습니다.

임계 속도에서 CSS 기반 수영 구역으로 어떻게 이동합니까?

다음 400m 및 200m 테스트를 사용하여 CSS를 설정한 다음 해당 숫자를 기준으로 앵커 간격을 설정하세요. 이미 T-Pace로 훈련했다면 두 숫자를 비교하고 CSS 가이드와 수영 훈련 영역을 사용하여 반복 가능한 유산소 속도로 세트를 재구성하세요.
