Swim Analytics 개인정보 처리방침

최종 업데이트: 2025년 1월 10일 | 시행일: 2025년 1월 10일

소개

Swim Analytics("당사", "우리" 또는 "앱")는 귀하의 개인정보를 보호하기 위해 최선을 다하고 있습니다. 이 개인정보 처리방침은 당사의 모바일 애플리케이션(iOS 및 Android)이 귀하 기기의 건강 데이터에 접근하고, 이를 사용하며, 보호하는 방법을 설명합니다.

핵심 개인정보 보호 원칙: Swim Analytics는 서버 없는 로컬 아키텍처로 운영됩니다. Apple HealthKit(iOS) 또는 Health Connect(Android)에서 접근한 모든 건강 데이터는 전적으로 귀하의 물리적 기기에만 남으며, 외부 서버, 클라우드 서비스 또는 제3자에게 절대 전송되지 않습니다.

1. 건강 데이터 접근

Swim Analytics는 수영 운동 분석을 제공하기 위해 귀하 기기의 기본 건강 플랫폼과 통합됩니다.

1.1 iOS - Apple HealthKit 통합

iOS 기기에서 Swim Analytics는 Apple HealthKit과 통합하여 수영 운동 데이터에 접근합니다. 다음 항목에 대한 읽기 전용 접근 권한을 요청합니다:

  • 운동(Workouts): 시간 및 기간을 포함한 수영 운동
  • 거리: 총 거리 및 랩별 거리
  • 심박수: 운동 중 심박수 데이터
  • 활동 에너지: 수영 세션 중 소모된 칼로리
  • 스트로크 수: 분석을 위한 스트로크 데이터

Apple HealthKit 준수: Swim Analytics는 모든 Apple HealthKit 가이드라인을 준수합니다. 귀하의 건강 데이터는 iOS 기기 내에서 완전히 처리되며 절대 외부로 유출되지 않습니다. 당사는 HealthKit 데이터를 제3자, 광고 플랫폼 또는 데이터 브로커와 절대 공유하지 않습니다.

1.2 Android - Health Connect 통합

건강 데이터 유형 권한 목적
운동 세션 READ_EXERCISE Health Connect에서 수영 운동을 식별하고 가져오기 위해
거리 기록 READ_DISTANCE 총 수영 거리, 랩별 거리와 같은 주요 지표를 표시하고 페이스를 계산하기 위해
심박수 기록 READ_HEART_RATE 심박수 그래프를 표시하고, 운동 중 평균 및 최대 심박수를 계산하기 위해
속도 기록 READ_SPEED 수영 페이스, 페이스 존 및 스트로크 케이던스 분석을 계산하고 표시하기 위해
소모 칼로리 READ_TOTAL_CALORIES_BURNED 수영 세션 중 에너지 소비에 대한 전체적인 개요를 제공하기 위해

Android 권한: 이 권한들은 앱을 처음 실행할 때 요청됩니다. Android 설정 → 앱 → Health Connect → Swim Analytics에서 언제든지 이 권한을 철회할 수 있습니다.

1.3 건강 데이터 사용 방법

모든 건강 데이터는 다음 목적을 위해서만 독점적으로 사용됩니다:

  • 운동 표시: 상세 지표(거리, 시간, 페이스, 심박수)와 함께 수영 세션 표시
  • 성과 분석: 페이스 존, 스트로크 분석, CSS(임계 수영 속도) 및 sTSS(수영 훈련 스트레스 점수) 계산
  • 진행 상황 추적: 성과 추세, 개인 최고 기록 및 훈련 요약 표시
  • 데이터 내보내기: 개인적인 용도로 운동 데이터를 CSV 형식으로 내보낼 수 있도록 허용

1.4 데이터 저장

🔒 핵심 개인정보 보호 보증:

모든 건강 데이터는 귀하의 물리적 기기에만 남습니다.

  • iOS: 데이터는 iOS Core Data 및 UserDefaults를 사용하여 저장됩니다(기기 내부만)
  • Android: 데이터는 Android Room Database를 사용하여 저장됩니다(기기 내부 SQLite)
  • 외부 서버로 데이터가 업로드되지 않습니다
  • 인터넷을 통해 데이터가 전송되지 않습니다
  • 건강 데이터의 클라우드 동기화 또는 백업이 없습니다
  • 제3자가 귀하의 건강 데이터에 접근할 수 없습니다

데이터가 기기에서 나가는 유일한 경우는 귀하가 명시적으로 선택하여 운동을 CSV 형식으로 내보내고 해당 파일을 직접 공유하는 경우뿐입니다.

2. 필수 권한

2.1 iOS 권한

  • HealthKit 접근: 운동, 거리, 심박수, 활동 에너지 및 스트로크 수에 대한 읽기 접근
  • 사진 라이브러리(선택 사항): 운동 요약을 이미지로 저장하기로 선택한 경우에만

iOS 설정 → 개인정보 보호 및 보안 → 건강 → Swim Analytics에서 언제든지 HealthKit 권한을 관리할 수 있습니다.

2.2 Android 권한

  • android.permission.health.READ_EXERCISE
  • android.permission.health.READ_DISTANCE
  • android.permission.health.READ_HEART_RATE
  • android.permission.health.READ_SPEED
  • android.permission.health.READ_TOTAL_CALORIES_BURNED
  • 인터넷 접근 (INTERNET): 인앱 정적 콘텐츠를 표시하고 구독 관리(Google Play Billing)에 접근하는 데만 사용됩니다. 건강 데이터는 전송되지 않습니다.
  • 포그라운드 서비스 (FOREGROUND_SERVICE): 잠재적인 미래 백그라운드 동기화 기능을 위함(현재 구현되지 않음).

3. 수집하지 않는 데이터

Swim Analytics는 다음을 수집, 저장 또는 전송하지 않습니다:

  • ❌ 개인 식별 정보(이름, 이메일, 전화번호)
  • ❌ 기기 식별자(iOS의 IDFA, Android의 광고 ID)
  • ❌ 위치 데이터 또는 GPS 좌표
  • ❌ 사용 분석 또는 인앱 행동 추적
  • ❌ 외부 서버로의 오류 보고서 또는 진단 데이터
  • ❌ 제3자 SDK 또는 분석 서비스를 통한 어떠한 데이터

우리는 다음을 포함하여 제3자 추적 라이브러리를 전혀 사용하지 않습니다:

  • Google Analytics / Firebase Analytics 없음
  • Facebook SDK 없음
  • 광고 SDK 없음
  • 오류 보고 서비스(Crashlytics, Sentry 등) 없음

4. 인앱 구매 및 구독

Swim Analytics는 기기의 기본 결제 시스템을 통해 관리되는 선택적 인앱 구독을 제공합니다:

4.1 iOS - App Store 구독

iOS에서 구독을 구매할 때:

  • Apple이 App Store를 통해 모든 결제 처리를 담당합니다
  • 당사는 StoreKit을 통해 구독 상태(활성/비활성)만 받습니다
  • 당사는 귀하의 결제 정보(신용카드, 청구 주소)에 접근할 수 없습니다
  • 구독 데이터는 귀하의 기기에 로컬로 저장됩니다

구독 관리:

  • iOS 설정 → 사용자 이름 → 구독 → Swim Analytics
  • 또는 앱 내에서: 설정 → 구독 관리

4.2 Android - Google Play Billing

Android에서 구독을 구매할 때:

  • Google Play가 모든 결제 처리를 담당합니다
  • 당사는 Google Play Billing API를 통해 구독 상태(활성/비활성)만 받습니다
  • 당사는 귀하의 결제 정보(신용카드, 청구 주소)에 접근할 수 없습니다
  • 구독 데이터는 귀하의 기기에 로컬로 저장됩니다

구독 관리:

  • Google Play 스토어 → 계정 → 결제 및 정기 결제 → Swim Analytics
  • 또는 앱 내에서: 설정 → 구독 관리

5. 데이터 보존 및 삭제

5.1 데이터 보존

  • 건강 데이터는 귀하가 수동으로 삭제할 때까지 무기한 귀하의 기기에 저장됩니다
  • 운동 데이터는 과거 성과 추적 및 분석을 제공하기 위해 보존됩니다

5.2 데이터 삭제

귀하는 언제든지 데이터를 삭제할 수 있습니다:

방법 1: 개별 운동 삭제

  • 운동 상세 화면 열기
  • 삭제 버튼(휴지통 아이콘) 탭하기
  • 삭제 확인

방법 2: 모든 앱 데이터 지우기

  • iOS: 앱 삭제 및 재설치(모든 로컬 데이터가 제거됨)
  • Android: 설정 → 앱 → Swim Analytics → 저장공간 → 데이터 삭제

방법 3: 앱 제거

  • Swim Analytics를 제거하면 모든 로컬 데이터가 자동으로 제거됩니다

방법 4: 건강 권한 철회

  • iOS: 설정 → 개인정보 보호 및 보안 → 건강 → Swim Analytics → 모든 카테고리 끄기
  • Android: 설정 → 앱 → Health Connect → Swim Analytics → 모든 권한 철회

6. 데이터 보안

모든 데이터가 귀하의 기기에 남아 있더라도 당사는 데이터 보안을 중요하게 생각합니다:

6.1 보안 조치

  • iOS 보안: iOS Core Data를 사용하여 저장된 모든 데이터는 iOS 키체인과 기기 암호화로 보호됩니다. 기기가 잠겨 있을 때 데이터는 보호됩니다.
  • Android 보안: Room Database에 저장된 모든 데이터는 Android 내장 보안과 앱 샌드박스로 보호됩니다.
  • 네트워크 전송 없음: 건강 데이터가 기기를 떠나지 않으므로 전송 보안 위험이 제거됩니다
  • 앱 샌드박싱: iOS 및 Android 앱 샌드박스는 다른 앱이 Swim Analytics 데이터에 접근하는 것을 방지합니다
  • 안전한 저장: 기기 인증(비밀번호, Face ID, Touch ID, 지문, 얼굴 잠금 해제) 없이는 건강 데이터에 접근할 수 없습니다

6.2 귀하의 책임

데이터를 보호하기 위해:

  • 강력한 비밀번호/생체 인식으로 기기를 잠그십시오
  • 최신 보안 패치로 운영 체제를 최신 상태로 유지하십시오
  • iOS: 기기를 탈옥(Jailbreak)하지 마십시오
  • Android: 기기를 루팅(Root)하지 마십시오

7. 데이터 공유 및 제3자

Swim Analytics는 귀하의 건강 데이터를 어떠한 제3자와도 공유하지 않습니다.

7.1 데이터 공유 없음

  • 당사는 귀하의 데이터를 판매하지 않습니다
  • 당사는 귀하의 데이터를 광고주와 공유하지 않습니다
  • 당사는 분석 회사에 귀하의 데이터를 제공하지 않습니다
  • 당사는 소셜 미디어 플랫폼과 통합하지 않습니다

7.2 CSV 내보내기 (사용자 시작 전용)

데이터가 기기에서 나가는 유일한 방법은 귀하가 명시적으로 다음을 수행하는 경우입니다:

  1. 설정 → 원시 데이터 내보내기로 이동
  2. CSV 파일 생성
  3. 기기의 공유 메뉴(이메일, 클라우드 저장소, 메시지 앱)를 통해 CSV 파일 공유 선택

이는 전적으로 귀하의 통제하에 있습니다.

8. 아동 개인정보 보호

Swim Analytics는 13세 미만 아동의 데이터를 의도적으로 수집하지 않습니다. 앱은 연령 정보를 요청하지 않지만, 부모는 자녀의 건강 추적 앱 사용을 감독해야 합니다.

13세 미만 아동이 Swim Analytics를 사용했다고 판단되시면 연락해 주십시오. 기기에서 모든 로컬 데이터가 제거되도록 도와드리겠습니다.

9. 국제 데이터 전송

해당 없음. 모든 건강 데이터가 전적으로 귀하의 기기(iOS 또는 Android)에 남아 있고 서버로 전송되지 않으므로 국제 데이터 전송이 없습니다.

10. 귀하의 권리 (GDPR, CCPA 준수)

Swim Analytics는 서버에서 개인 데이터를 수집하지 않지만, 당사는 귀하의 데이터 개인정보 보호 권리를 존중합니다:

10.1 GDPR 권리 (유럽 사용자)

  • 접근 권한: 모든 데이터는 언제든지 앱 내에서 접근 가능합니다
  • 삭제 권한: 5.2절에 설명된 방법을 사용하여 데이터를 삭제하십시오
  • 이동 권한: 데이터를 CSV 형식으로 내보내십시오(설정 → 원시 데이터 내보내기)
  • 처리 제한 권한: 새로운 데이터에 대한 접근을 중지하려면 건강 권한을 철회하십시오

10.2 CCPA 권리 (캘리포니아 사용자)

  • 알 권리: 이 정책은 접근되는 모든 데이터와 사용 방법을 공개합니다
  • 삭제 권한: 5.2절에 설명된 방법을 사용하여 데이터를 삭제하십시오
  • 판매 거부 권한: 해당 없음(당사는 데이터를 판매하지 않습니다)

11. 개인정보 처리방침 변경

당사는 이 개인정보 처리방침을 수시로 업데이트할 수 있습니다. 변경 시:

  • 이 정책 상단의 "최종 업데이트" 날짜가 수정됩니다
  • 중요한 변경 사항은 앱 내에서 공지됩니다
  • 변경 후 앱을 계속 사용하는 것은 업데이트된 정책에 동의하는 것으로 간주됩니다

개인정보를 보호하는 방법에 대한 정보를 얻으려면 이 정책을 주기적으로 검토하는 것이 좋습니다.

12. 문의하기

이 개인정보 처리방침 또는 귀하의 데이터 개인정보 보호와 관련하여 질문, 우려 사항 또는 요청이 있는 경우:

  • 이메일: support@swimanalytics.app
  • 웹사이트: https://swimanalytics.app

응답 시간: 당사는 모든 개인정보 보호 문의에 대해 영업일 기준 7일 이내에 답변하는 것을 목표로 합니다.

13. 법적 준수

Swim Analytics는 다음을 준수합니다:

  • iOS: Apple App Store 심사 지침, Apple HealthKit 지침
  • Android: Google Play 개발자 프로그램 정책, Android Health Connect 지침
  • 일반 데이터 보호 규정 (GDPR)
  • 캘리포니아 소비자 개인정보 보호법 (CCPA)
  • 아동 온라인 개인정보 보호법 (COPPA)

요약

간단히 말해서:

  • ✅ 접근하는 데이터: Apple HealthKit(iOS) 또는 Health Connect(Android)의 수영 운동 데이터
  • ✅ 저장 위치: 귀하의 기기에만 저장(iOS Core Data 또는 Android Room Database)
