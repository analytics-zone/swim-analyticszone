수영에 특화된 이유

물의 저항은 지수적으로 증가합니다

수영은 공기보다 저항이 훨씬 크고, 속도가 올라갈수록 저항이 급증합니다. 그래서 수영 sTSS는 IF²가 아니라 IF³를 사용해 강도 비용을 정확히 반영합니다.

  • CSS 기반 페이스: 임계 페이스에 맞춘 존/목표 설정
  • SR/DPS/SWOLF: 스트로크 효율을 정량화하고 개선
  • 로컬 계산: 프라이버시 보호, 빠른 처리

주요 기능

Apple Watch 자동 동기화

Health에서 수영 워크아웃을 자동 불러와 페이스, SR, SWOLF, sTSS를 계산.

SR/DPS 트래킹

스트로크 레이트와 스트로크 거리를 함께 추적해 속도-효율 균형을 최적화.

페이스 존 오버레이

워크아웃 구간별 페이스를 CSS 존에 매핑해 강도 준수 여부 확인.

SWOLF 분석

시간 + 스트로크 수로 효율을 측정, 드릴 효과를 수치로 확인.

CTL/ATL/TSB

수영 전용 sTSS로 체력, 피로, 컨디션을 장기 추적.

100% 온디바이스

데이터가 서버를 거치지 않으며, 계정 생성 없이 바로 사용.

FAQ

어떤 워치가 필요한가요?

Apple Watch Series 2 이상이면 됩니다. Health와 동기화해 모든 워크아웃을 자동 불러옵니다.

데이터는 어디에 저장되나요?

100% 온디바이스 저장 및 처리. 서버 업로드나 계정이 없습니다.

CSS를 자주 업데이트해야 하나요?

베이스/빌드 단계는 6~8주마다, 피크 시즌은 4~6주마다를 권장합니다.

Garmin 워크아웃도 가져올 수 있나요?

Garmin 데이터를 Apple Health로 동기화하면 불러올 수 있습니다.
