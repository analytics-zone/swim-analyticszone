Kritische Schwimmgeschwindigkeit (CSS)

Der praktische Tempo-Benchmark, der Schwimmtrainingszonen, Schwellenwerte und Trainingsbelastungsanalysen verankert

Schnelle Antwort

Kritische Schwimmgeschwindigkeit (CSS) ist eine praxiserprobte Schätzung Ihrer nachhaltigen Schwimmgeschwindigkeit. Die meisten Schwimmer berechnen es aus a 400m + 200m Zeitfahren, und verwenden Sie dann das Ergebnis zum Festlegen Trainingszonen, Temposchwellenwerte und Leistung sTSS und Trainingsbelastung Analyse.

  • Beste Verwendung: Schwellentraining, wiederholbare Stimulation und Lastverfolgung
  • Typische Interpretation: ungefähr das schnellste Tempo, das Sie aerob für harte, kontrollierte Arbeit durchhalten können
  • Benutzen Sie es nicht blind: Schlechtes Tempo, unvollständige Erholung, unterschiedliche Beckenbedingungen oder Schlepphilfen können das Ergebnis verfälschen
  • Trittfrequenz erneut testen: alle 6-8 Wochen, oder nach einem sinnvollen Trainingsblock

Wichtige Erkenntnisse

  • Was CSS ist: eine praktische Schätzung der Schwellengeschwindigkeit für Schwimmer, basierend auf einfachen Feldtests
  • Warum es wichtig ist: CSS verwandelt das Schwimmtraining vom Rätselraten in wiederholbares Tempo und eine bessere Belastungskontrolle
  • Was es freischaltet:Zonen, sTSS, Leistungsmanagementund klarere Fortschrittskontrollen
  • Was es nicht tut: Ersetzen Sie die Technikanalyse, den Open-Water-Kontext oder Effizienzinterpretation
  • Bester nächster Schritt: Testen Sie CSS gründlich und verwenden Sie es dann, um Ihre Schwellen- und Aerobic-Arbeit zu strukturieren

Wenn Sie nur eine Schwimmmetrik als Tempoanker behalten, behalten Sie CSS bei. Es ist nicht perfekt und es handelt sich nicht um einen vollständigen physiologischen Labortest, aber es ist praktisch, wiederholbar und direkt nützlich. Diese Kombination ist der Grund, warum CSS das Rückgrat des datengesteuerten Schwimmtrainings bleibt.

Für die meisten Schwimmer gilt: Kritische Schwimmgeschwindigkeit ist die Brücke zwischen reinem Zeitfahren und alltäglichen Trainingsentscheidungen. Es hilft, fünf wichtige Fragen zu beantworten:

  • Wie schnell sollten Schwellenwerte eigentlich sein?
  • Welcher Tempobereich gehört jeweils dazu? Schwimmzone?
  • Wie soll ich die 100er, 200er und 400er Wiederholungen beschleunigen?
  • Wie quantifiziere ich den Schwimmstress? sTSS?
  • Geht meine Aerobic-Schwimmfitness in die richtige Richtung?

Welche CSS-Maßnahmen

CSS ist eine praktische Schätzung der harten aeroben Schwimmgeschwindigkeit, normalerweise abgeleitet aus einem 400-m- und 200-m-Zeitfahren. In der Trainersprache wird es oft als Ihr Schwellentempo für strukturiertes Pooltraining betrachtet: schnell genug, um von Bedeutung zu sein, kontrolliert genug, um es zu wiederholen, und stabil genug, um Zonen darum herum zu bilden.

AspektPraktische Antwort
Was CSS misstIhre vor Ort geschätzte Schwellen-Schwimmgeschwindigkeit für nachhaltiges Aerobic-Training

🎯 Das nützliche mentale Modell

Stellen Sie sich CSS als Ihr eigenes vor Betriebsgeschwindigkeits-Benchmark, keine magische Zahl. Es ist die Temporeferenz, die den Rest Ihrer Schwimmmetriken kohärent macht.

Warum CSS wichtig ist

CSS ist wichtig, weil es Physiologie mit der Ausführung verbindet. Viele Schwimmer wissen, dass sie einen „harten Satz“ oder eine „Tempo-Einheit“ absolviert haben, können aber nicht sagen, ob das Tempo richtig war. CSS behebt dieses Problem, indem es Ihnen einen Benchmark zur Verfügung stellt, den Sie Sitzung für Sitzung verwenden können.

🏊 Trainingszonen

CSS bietet Ihnen einen sauberen Anker für SchwimmzonenSo bleiben einfache Arbeiten einfach und Schwellenarbeiten zielgerichtet.

📊 Trainingsbelastung

Ohne ein brauchbares CSS, sTSS und Leistungsmanagement-Kennzahlen verlieren ihre Bedeutung.

⏱️ Wiederholbares Tempo

Mithilfe von CSS können Sie 100er, 200er und 400er mit mehr Konsistenz ablaufen, anstatt sich nur auf das Gefühl zu verlassen.

📈 Fortschrittskontrollen

Wenn sich CSS verbessert, erhöht sich Ihr Schwellenwerttempo. Das ist ein starkes Signal dafür, dass sich auch die Fitness beim aeroben Schwimmen verbessert.

⚠️ Kritische Abhängigkeit: Wenn Ihr CSS falsch ist, driften Ihre Zonen, sTSS und Performance-Management-Zahlen in die falsche Richtung. Eine schlechte Eingabe führt zu einer schlechten Analyse.

Wofür CSS gut ist und wofür nicht

CSS eignet sich hervorragend für: Schwellenwerte, aerobes Qualitätstraining, Zoneneinstellung und schwimmspezifische Belastungsverfolgung.

CSS reicht nicht aus für: Sprintprofilierung, Schlagtechnikdiagnose, Durchführung von Open-Water-Rennen unter wechselnden Bedingungen oder eigenständige Interpretation der Effizienz.

Verwenden Sie CSS für

  • Vorschriften zum Schwellentempo
  • Wiederholen Sie das Tempo 100/200/400
  • Personalisiert Zonenstruktur
  • sTSS und Lastmanagement
  • Verfolgen Sie den aeroben Fortschritt über eine Saison hinweg

Verwenden Sie CSS nicht allein für

  • Vergleich verschiedener Poollängen, als wären sie identisch
  • Vergleich von Schwimmern mit sehr unterschiedlichen Technikprofilen
  • Beurteilung der Schlagqualität ohne Schlaganfallmechanik
  • Beurteilung der Effizienz ohne Kontext SWOLF
  • Es wird davon ausgegangen, dass das Tempo im offenen Wasser genau dem CSS-Tempo im Pool entspricht

So testen Sie CSS in 4 praktischen Schritten

📋 Standardprotokoll

  1. Richtig aufwärmen

    Schwimmen Sie 300–800 m locker, integrieren Sie Übungen und fügen Sie vor dem Test ein paar kontrollierte Aufbauversuche hinzu.

  2. Schwimmen Sie ein 400-m-Zeitfahren

    Verwenden Sie einen Schubstart, keinen Tauchgang. Das Ziel sind Ihre schnellsten nachhaltigen 400 m, nicht die rücksichtslosen ersten 100 m.

  3. Erholen Sie sich vollständig

    Nehmen Sie sich 5-10 Minuten Zeit leichte oder vollständige Ruhe. Eine unvollständige Wiederherstellung ist eine der schnellsten Möglichkeiten, das Ergebnis zu ruinieren.

  4. Schwimmen Sie ein 200-m-Zeitfahren

    Benutzen Sie wieder einen Push-Start und notieren Sie die Zeit genau. Ihr 200-m-Tempo sollte deutlich schneller sein als Ihr 400-m-Tempo.

⚠️ Die häufigsten Testfehler

Schlechtes 400-m-Tempo

Problem: Du gehst zu hart raus und verblasst dann stark.

Ergebnis: Die 400-m-Stopps spiegeln nachhaltige Tempoqualität wider.

Fix: Streben Sie gleichmäßige oder leicht negative Aufteilungen an.

Unzureichende Erholung

Problem: Durch die Ermüdung wird der 200-m-Lauf künstlich langsam.

Ergebnis: CSS wird verzerrt und Ihre Trainingszonen verschieben sich.

Fix: Warten Sie, bis die Atmung kontrolliert ist und sich die Genesung vollständig anfühlt.

Gemischte Bedingungen

Problem: Unterschiedliche Beckenlängen, große Abstoßänderungen oder Schlepphilfen verändern die Testumgebung.

Ergebnis: Sie vergleichen Zahlen, die nicht wirklich vergleichbar sind.

Fix: Halten Sie die Bedingungen von Test zu Test so konstant wie möglich.

🔄 Häufigkeit erneuter Tests

Testen Sie CSS alle erneut 6-8 Wochen, oder nach einem sinnvollen Block, der sich auf Schwellenwert, Ausdauer oder Technikfestigung konzentriert. Es geht nicht um ständige Tests. Es geht darum, einen Benchmark konsistent genug zu verwenden, um Veränderungen zu erkennen.

CSS-Berechnungsformel

Formel

CSS (m/s) = (D₂ - D₁) / (T₂ - T₁)

Wo:

  • D₁ = 200 Meter
  • D₂ = 400 Meter
  • T₁ = Zeit für 200 m (in Sekunden)
  • T₂ = Zeit für 400 m (in Sekunden)

Vereinfacht für Tempo pro 100 m

CSS-Tempo/100 m (Sekunden) = (T₄₀₀ - T₂₀₀) / 2

Ausgearbeitetes Beispiel

Testergebnisse

  • 400m Zeit: 6:08 (368 Sekunden)
  • 200m Zeit: 2:30 (150 Sekunden)

Schritt 1: CSS-Geschwindigkeit berechnen

CSS-Tempo/100 m = (368 – 150) / 2
CSS-Tempo/100 m = 218 / 2
CSS-Tempo/100 m = 109 Sekunden = 1:49 pro 100 m

Schritt 2: Bei Bedarf auf Geschwindigkeit umstellen

CSS = 100/109
CSS = 0,917 m/s

Arbeitsbeispiel: Wie man dieses Ergebnis nutzt

Wenn Ihr CSS ist 1:49/100m, Ihre Schwellenwertwiederholungen sollten sich in der Nähe dieses Tempos ansammeln Zonentabelle sollte daraus gebaut werden, und Ihr sTSS Berechnungen sollte es als Nenner-Temporeferenz verwenden.

🎯 Praktische Zonennutzung

  • Zone 2: wo die meisten Schwimmer dauerhaftes Volumen aufbauen
  • Zone 3: nützlich für kontrollierte Qualitäts- und Übergangsarbeiten
  • Zone 4: wo CSS für Schwellenwertsätze direkt nützlich wird
  • Zone 5: Wo CSS aufhört, die ganze Geschichte zu sein, und Sprint-/Power-Eigenschaften wichtiger sind

So verwenden Sie CSS im echten Training

1️⃣ Temposchwellensätze

Verwenden Sie CSS, um mit dem Raten aufzuhören. Sätze wie 8x100, 5x200 oder 3x400 machen mehr Sinn, wenn Sie das Tempoziel kennen und es sauber halten können.

2️⃣ Pace mit sTSS verbinden

CSS gibt Ihrem Sinn sTSS. Schwere und einfache Sitzungen vermischen sich nicht mehr in derselben vagen Arbeitsbelastung.

3️⃣ Fortschritte besser verfolgen

Wenn CSS schneller wird, während Ihre Trainingseinheiten kontrolliert bleiben, verbessert sich wahrscheinlich Ihre Schwimmfitness an der Schwelle.

4️⃣ Bauen Sie bessere Tempogewohnheiten auf

CSS lehrt Sie, Wiederholungsübungen kontrolliert zu schwimmen, anstatt jeden harten Satz in einen Tempokollaps zu verwandeln.

Beispiele für CSS-orientierte Sitzungsideen

  • Schwellenwerterhaltung: 8x100 im CSS-Tempo mit kurzer Pause
  • Tempohaltbarkeit: 5x200 im CSS-Tempo mit konsistenten Aufteilungen
  • Aerobic-Unterstützung: 3x400, etwas langsamer als CSS-Tempo mit kontrollierter Strichzahl
  • Technik + Tempo: Abwechselnd 100 m bei CSS-Anstrengung mit 50 m einfacher technischer Zurücksetzung

Dies sind Beispiele, keine allgemeingültigen Rezepte. Der Punkt ist, dass CSS Ihnen eine stabile Tempo-Sprache bietet.

CSS, SWOLF und Effizienz

CSS sagt es Ihnen wie schnell Sie können an der Schwelle schwimmen. SWOLF und Effizienzkennzahlen Ihnen etwas anderes sagen: wie wirtschaftlich Sie sich durch das Wasser bewegen. Sie ergänzen sich, sind aber nicht austauschbar.

Vergleichen Sie SWOLF nicht blind

Ein niedrigerer SWOLF ist nicht immer „besser“ für alle Schwimmer, Trainingseinheiten, Schwimmzüge oder Pool-Setups. Benutzen Effizienzkennzahlen als Kontext neben CSS, Strichqualität und Wiederholbarkeit, nicht als eigenständiges Ranking-Tool.

Wissenschaftliche Validierung

Wakayoshi et al. und der praktische Fall für CSS

CSS wurde weit verbreitet, weil es Trainern und Schwimmern einen feldfreundlichen Benchmark bot, der sich gut mit schwellenbezogenen Leistungsmarkern vergleichen ließ, ohne dass wiederholte Laktattests im Labor erforderlich waren.

  • Praktisch: Wiederholbar durch einfache Pooltests
  • Nützlich: nah genug an der schwellenwertrelevanten Stimulation für die Programmierung
  • Umsetzbar: Einfaches erneutes Testen und Aktualisieren über eine Saison hinweg
  • Verbunden: Direkt nützlich für Trainingszonen, Tempo und Belastungsverfolgung

Häufig gestellte Fragen

Wie oft sollte ich CSS erneut testen?

Alle 6-8 Wochen ist ein guter Standardwert. Führen Sie den Test nur dann früher durch, wenn sich das Training stark verändert hat oder sich Ihre aktuellen Zonen eindeutig falsch anfühlen.

Ist CSS dasselbe wie die Laktatschwelle?

Nicht ganz. CSS ist eine Feldschätzung, die als praktischer Benchmark für die Schwellengeschwindigkeit verwendet wird. Es ist nützlich, weil es wiederholbar und umsetzbar ist, und nicht, weil es Labortests perfekt ersetzt.

Kann ich CSS für das Tempo im offenen Wasser verwenden?

Verwenden Sie es als erste Referenz, nicht als wörtliche Garantie. Sichtung, Windschatten, Kabbelwasser, Neoprenanzüge und Navigation verändern das Tempoverhalten im offenen Wasser.

Können Anfänger CSS verwenden?

Ja, aber das Ergebnis wird umso nützlicher, je besser sich Technik und Tempo verbessern. Sehr neue Schwimmer benötigen oft eine gleichmäßige Schlag- und Tempokonsistenz, bevor CSS sehr stabil wird.

Sollte ich mein CSS direkt mit dem eines anderen vergleichen?

Nur vorsichtig. Beckenlänge, Abstöße, Luftwiderstand, Tempofähigkeit und Technikqualität beeinflussen alle das Ergebnis. CSS ist als persönlicher Maßstab im Laufe der Zeit am wirkungsvollsten.
