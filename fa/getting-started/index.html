شروع کار با Swim Analytics

راهنمای کامل شما برای ردیابی عملکرد شنا، تست CSS و تجزیه و تحلیل بار تمرینی

به شنای داده محور خوش آمدید

Swim Analytics با استفاده از تمرینات شنای شما را به بینش های عملی تبدیل می کندسرعت بحرانی شنا (CSS)،امتیاز استرس تمرینی (sTSS)، ونمودار مدیریت عملکرد (PMC)معیارها این راهنما در 4 مرحله ساده شما را از اولین راه اندازی تا تجزیه و تحلیل بار آموزشی پیشرفته می برد.

شروع سریع (5 دقیقه)

1

دانلود و نصب کنید

Swim Analytics را از App Store دانلود کنید و اجازه دسترسی به Apple Health را بدهید. این برنامه تمرینات شنا را به صورت خودکار همگام‌سازی می‌کند—بدون نیاز به ثبت دستی.

دانلود برنامه →
2

تست CSS را انجام دهید

تایم تریل 400 متر و 200 متر را تکمیل کنید تا سرعت شنای حیاتی خود را تعیین کنید. این پایه و اساس همه معیارها است - بدون CSS، sTSS و مناطق آموزشی قابل محاسبه نیستند.

پروتکل تست CSS ↓
3

نتایج CSS را وارد کنید

400 و 200 متر خود را در برنامه وارد کنید. Swim Analytics CSS، مناطق سرعت را محاسبه می کند و تمام معیارها را برای فیزیولوژی شما شخصی می کند. با بهبود تناسب اندام، هر 6 تا 8 هفته یکبار به روز رسانی کنید.

4

تمرینات ردیابی را شروع کنید

با اپل واچ و اپلیکیشن سلامتی شنا کنید. Swim Analytics به طور خودکار تمرینات را وارد می کند، sTSS را محاسبه می کند، CTL/ATL/TSB را به روز می کند و پیشرفت را ردیابی می کند. بدون وارد کردن دستی داده ها مورد نیاز است.

پروتکل تست CSS را کامل کنید

📋 آنچه شما نیاز دارید

  • دسترسی به استخر:استخر 25 متری یا 50 متری (قابل قبول 25 متری)
  • زمان بندی:کرونومتر، ساعت سرعت یا اپل واچ
  • زمان گرم کردن:15-20 دقیقه قبل از آزمایش
  • بازیابی:5-10 دقیقه بین آزمون
  • تلاش:حداکثر سرعت پایدار (نه دوی سرعت همه جانبه)

⏱️ شرایط روز آزمون

  • استراحت کرد:بدون تمرین سخت 24-48 ساعت قبل
  • هیدراته شده:تغذیه خوب هیدراته شده و معمولی
  • دمای استخر:26 تا 28 درجه سانتی گراد (79 تا 82 درجه فارنهایت) ایده آل - آب بسیار سرد یا گرم می تواند بر عملکرد و تغییر نتایج تأثیر بگذارد
  • زمان روز:زمانی که به طور معمول بهترین تمرین را انجام می دهید
  • تجهیزات:همانند تمرین (عینک، کلاه، کت و شلوار)

تست گام به گام CSS

گرم کردن

15-20 دقیقه

400-800 متر شنای آسان، تمرینات، و افزایش تدریجی. 2-3×50 را با سرعت فزاینده (60٪، 75٪، 85٪ تلاش) اضافه کنید. 2-3 دقیقه قبل از آزمایش استراحت کنید.

آزمایشی 1

400 متر حداکثر تلاش

شروع فشاری(بدون شیرجه). 400 متر را با سریع ترین سرعتی که می توانید برای تمام مسافت شنا کنید. این یک دوی سرعت نیست: خود را برای کل مسافت سرعت دهید. رکورد زمان درmm:ssقالب (به عنوان مثال، 6:08).

نکته قدم زدن:حتی 100 متر اسپلیت را هدف بگیرید. 200 متر دوم باید ≤ 200 متر اول باشد (ایده آل تقسیم منفی).
بازیابی

5-10 دقیقه

فاز بحرانی:شنای آسان یا استراحت کامل. صبر کنید تا ضربان قلب به کمتر از 120 ضربه در دقیقه برسد و تنفس به طور کامل بهبود یابد. هدف: تنفس مکالمه و احساس آمادگی برای تلاش حداکثری دیگر. بازیابی ناکافی = CSS نادرست.

محاکمه 2

200 متر حداکثر تلاش

شروع فشاری(بدون شیرجه). حداکثر تلاش پایدار برای 200 متر. این باید در هر 100 متر سخت تر از 400 متر باشد. رکورد زمان درmm:ssقالب (به عنوان مثال، 2:30).

بررسی اعتبار سنجی:سرعت 200 متر / 100 متر باید 3-6 ثانیه سریعتر از سرعت 400 متر / 100 متر باشد. در غیر این صورت، ریکاوری ناکافی بود یا ضربان قلب خاموش بود.
خنک کردن

10-15 دقیقه

300-500 متر شنا آسان، کشش. زمان های خود را فوراً ثبت کنید - به حافظه اعتماد نکنید.

⚠️ اشتباهات رایج تست CSS

  • بیرون رفتن خیلی سریع در 400 متر:نتایج در blowup، CSS نادرست. از قدم زدن یکنواخت استفاده کنید.
  • بهبودی ناکافی بین آزمایش‌ها:خستگی 200 متر کند می شود و CSS را به طور مصنوعی سریع می کند → مناطق بیش از حد تمرین شده.
  • استفاده از شیرجه شروع می شود:مزیت 0.5-1.5 ثانیه را اضافه می کند، محاسبات را کج می کند. همیشه از دیوار فشار دهید
  • آزمایش در هنگام خستگی:بار تمرینی سنگین 24-48 ساعت قبل = نتایج افسرده. وقتی تازه است تست کنید.
  • بلافاصله ضبط نمی شود:حافظه غیر قابل اعتماد است. قبل از خنک شدن زمان ها را یادداشت کنید.

وارد کردن نتایج CSS در Swim Analytics

مرحله 1: تنظیمات CSS را باز کنید

در برنامه Swim Analytics، بهتنظیمات → سرعت بحرانی شنا. روی "Perform CSS Test" یا "Update CSS" ضربه بزنید.

مرحله 2: زمان های ورودی

زمان 400 متری خود را وارد کنید (به عنوان مثال،6:08) و زمان 200 متر (به عنوان مثال،2:30). از فرمت دقیق نشان داده شده استفاده کنید. روی "محاسبه" ضربه بزنید.

مرحله 3: نتایج را بررسی کنید

نمایش برنامه:

  • سرعت CSS:0.917 متر بر ثانیه
  • سرعت CSS:1:49/100 متر
  • مناطق تمرینی:7 منطقه شخصی سازی شده (منطقه 1-7)
  • خط پایه sTSS:اکنون برای همه تمرینات فعال است

مرحله 4: ذخیره و همگام سازی

روی «ذخیره CSS» ضربه بزنید. برنامه بلافاصله:

  • مناطق تمرین را دوباره محاسبه می کند
  • sTSS را برای 90 روز گذشته به‌طور عطف به‌روزرسانی می‌کند
  • محاسبات CTL/ATL/TSB را تنظیم می کند
  • تجزیه و تحلیل تمرین مبتنی بر منطقه را فعال می کند

💡 نکته حرفه ای: تست CSS تاریخی

اگر قبلاً CSS خود را از آزمایش‌های قبلی می‌دانید، می‌توانید آن زمان‌ها را مستقیماً وارد کنید. با این حال، برای دقیق ترین نتایج، هر 6-8 هفته یک آزمایش تازه انجام دهید. CSS شما باید با پیشرفت آموزش بهبود یابد (سریعتر شود).

درک معیارهای شما

سرعت بحرانی شنا (CSS)

چیست:سرعت آستانه هوازی شما - سریعترین سرعتی که می توانید تا 30 دقیقه بدون خستگی حفظ کنید.

به چه معناست:CSS = 1:49/100m به این معنی است که می‌توانید سرعت 1:49 را برای تلاش‌های آستانه پایدار نگه دارید.

نحوه استفاده:پایه برای تمام مناطق تمرین و محاسبه sTSS. به روز رسانی هر 6-8 هفته.

یادگیری CSS →

مناطق آموزشی

آنها چه هستند:7 محدوده شدت بر اساس CSS شما، از بازیابی (منطقه 1) تا سرعت (منطقه 7).

منظورشون چیه:هر منطقه سازگاری های فیزیولوژیکی خاصی را هدف قرار می دهد (پایه هوازی، آستانه، VO₂max).

نحوه استفاده:برای آموزش ساختاریافته از دستورات منطقه پیروی کنید. برنامه زمان در منطقه را برای هر تمرین نشان می دهد.

مناطق تمرین →

امتیاز استرس تمرین شنا (sTSS)

چیست:کمی استرس تمرین ترکیبی از شدت و مدت. 1 ساعت با سرعت CSS = 100 sTSS.

به چه معناست:sTSS 50 = بازیابی آسان، sTSS 100 = متوسط، sTSS 200+ = جلسه بسیار سخت.

نحوه استفاده:پیگیری روزانه/هفتگی sTSS برای مدیریت بار آموزشی. حداکثر 5 تا 10 برابر sTSS در هفته افزایش دهید.

راهنمای sTSS →

CTL / ATL / TSB

آنها چه هستند:

  • CTL:بار تمرینی مزمن (تناسب اندام) - میانگین 42 روزه sTSS
  • ATL:بار تمرینی حاد (خستگی) - میانگین 7 روزه sTSS
  • TSB:تعادل استرس آموزشی (فرم) = CTL - ATL

نحوه استفاده:TSB مثبت = تازه / مخروطی، منفی TSB = خسته. زمانی که TSB = 5+ تا 25+ است مسابقه دهید.

📊 اهداف هفته اول شما

پس از ورود به CSS و انجام 3-5 تمرین:

  • مقادیر sTSS را بررسی کنید:تأیید کنید که آنها با درک تلاش مطابقت دارند (آسان ~50، متوسط ~100، سخت ~150+)
  • توزیع منطقه بازبینی:آیا 60 تا 70 درصد در منطقه 2 (پایه هوازی) هزینه می کنید؟
  • ایجاد خط پایه CTL:میانگین sTSS هفته اول شما به پایه اولیه تناسب اندام تبدیل می شود
  • شناسایی الگوها:کدام تمرینات بالاترین sTSS را ایجاد می کنند؟ آیا به اندازه کافی در حال بهبودی هستید؟

سفر معمولی کاربر (8 هفته اول)

هفته 1-2: پایه را تعیین کنید

  • تست CSS را انجام دهید و نتایج را وارد کنید
  • 3-5 تمرین تمرینی معمولی را کامل کنید
  • مقادیر sTSS و توزیع ناحیه را رعایت کنید
  • ایجاد CTL اولیه (سطح تناسب اندام)
  • هدف:معیارها را درک کنید، هنوز تغییری ایجاد نشده است

هفته 3-4: اعمال مناطق

  • از مناطق CSS در برنامه ریزی تمرین استفاده کنید
  • منطقه 2 را عمدا برای ست های هوازی شنا کنید
  • ردیابی مجموع هفتگی sTSS (هدف برای ثبات)
  • مانیتور TSB (باید کمی منفی باشد = آموزش)
  • هدف:آموزش بر اساس مناطق، نه احساس

هفته 5-6: اضافه بار پیشرونده

  • هفتگی sTSS را 5 تا 10 درصد از سطح پایه افزایش دهید
  • هفته ای 1 جلسه آستانه (منطقه 4) اضافه کنید
  • CTL باید به تدریج افزایش یابد (بهبود تناسب اندام)
  • ATL ممکن است در هفته های سخت افزایش یابد (عادی)
  • هدف:پیشرفت تناسب اندام کنترل شده

هفته 7-8: دوباره آزمایش و تنظیم کنید

  • آزمایش دوم CSS را انجام دهید (باید سریعتر باشد)
  • به روز رسانی مناطق در برنامه (سرعت بهبود می یابد)
  • مقایسه CTL هفته 1 با هفته 8 (باید 10-20+ باشد)
  • پیشرفت مرور: آیا زمان در حال کاهش است؟ احساس راحتی می کنید؟
  • هدف:تایید اثربخشی آموزش

✅ شاخص های موفقیت

پس از 8 هفته تمرین ساختاری با Swim Analytics، باید ببینید:

  • بهبود CSS:1-3٪ سرعت CSS سریعتر (به عنوان مثال، 1:49 → 1:47)
  • افزایش CTL:+15-25 امتیاز (به عنوان مثال، 30 → 50 CTL)
  • sTSS سازگار:مجموع هفتگی در 10-15 درصد واریانس
  • قدم زدن بهتر:تقسیم‌های یکنواخت بیشتر، کالیبراسیون تلاش بهتر
  • بهبود بهبود یافته:چرخه TSB به طور قابل پیش بینی (10- تا 5+)

عیب یابی و سوالات متداول

به نظر می رسد sTSS من برای تلاش برای تمرین خیلی زیاد/پایین است

علت:CSS قدیمی یا نادرست است.

راه حل:CSS را مجدداً آزمایش کنید. اگر هنگام خستگی یا سرعت ضعیف تست کردید، CSS اشتباه خواهد بود. یک تست CSS مناسب برای تمام معیارهای پایین دستی حیاتی است.

برنامه "No CSS configured" را نشان می دهد

علت:تست CSS تکمیل نشده یا ذخیره نشده است.

راه حل:به Settings → Critical Swim Speed → Perform Test بروید. هر دو 400 متر و 200 متر را وارد کنید، سپس روی ذخیره ضربه بزنید.

تمرین‌ها از اپل واچ همگام‌سازی نمی‌شوند

علت:مجوزهای برنامه بهداشتی داده نشده است یا تمرین به عنوان "شنا" طبقه بندی نشده است.

راه حل:تنظیمات → حریم خصوصی → سلامتی → تجزیه و تحلیل شنا → اجازه خواندن برای تمرین را بررسی کنید. مطمئن شوید که نوع تمرین اپل واچ "شنا در استخر" یا "شنا در آب آزاد" است.

CTL با وجود آموزش مداوم افزایش نمی یابد

علت:فرکانس sTSS بسیار کم یا ناسازگار است.

راه حل:CTL میانگین وزنی نمایی 42 روزه است. به آرامی بالا می رود. sTSS هفتگی را 5-10% افزایش دهید و 4+ تمرین در هفته را برای رشد مداوم CTL حفظ کنید.

هر چند وقت یک بار باید CSS را دوباره آزمایش کنم؟

توصیه:هر 6-8 هفته در طول مراحل پایه / ساخت، هر 4-6 هفته در فصل اوج. پس از بیماری، آسیب، استراحت طولانی، یا زمانی که مناطق به طور مداوم بیش از حد آسان/سخت هستند، دوباره آزمایش کنید. از انجام تست در طول هفته های تمرین سنگین یا در 10 روز پس از مسابقه خودداری کنید.

آیا می توانم از Swim Analytics برای سکته های دیگر استفاده کنم؟

بله، با محدودیت:CSS معمولاً در سبک آزاد آزمایش می شود. برای تمرینات IM/کرال پشت/کرال سینه، sTSS بر اساس CSS آزاد محاسبه می شود. برای دقت بیشتر، آزمایش‌های CSS مختص سکته مغزی را در نظر بگیرید.
