سرعت بحرانی شنا (CSS)

بنیاد آموزش شنای داده محور

سرعت بحرانی شنا (CSS) چیست؟

سرعت بحرانی شنا (CSS)حداکثر سرعت تئوری شنا است که می توانید بدون خستگی حفظ کنید. این سرعت آستانه هوازی شما را نشان می دهد که معمولاً با 4 میلی مول در لیتر لاکتات خون مطابقت دارد و تقریباً 30 دقیقه پایدار است. CSS با استفاده از زمان آزمایشی 400 متر و 200 متر برای تعیین مناطق تمرین شخصی محاسبه می شود.

سرعت بحرانی شنا (CSS) نشان‌دهنده حداکثر سرعت تئوری شنا است که می‌توانید به طور مداوم بدون خستگی حفظ کنید. این آستانه هوازی شما در آب است - شدتی که در آن تولید لاکتات برابر با پاکسازی لاکتات است.

🎯 اهمیت فیزیولوژیکی

CSS دقیقاً با موارد زیر مطابقت دارد:

  • آستانه لاکتات 2 (LT2)- آستانه تهویه دوم
  • حداکثر حالت پایدار لاکتات (MLSS)- بالاترین سطح لاکتات پایدار
  • سرعت آستانه عملکردی (FTP)- شنا معادل دوچرخه سواری FTP
  • ~ 4 میلی مول در لیتر لاکتات خون- نشانگر سنتی OBLA

چرا CSS مهم است

CSS استمتریک پایهکه تمام تحلیل های پیشرفته بار آموزشی را باز می کند:

  • مناطق آموزشی:مناطق شدت را بر اساس فیزیولوژی شما شخصی می کند
  • محاسبه sTSS:تعیین کمیت دقیق استرس تمرینی را فعال می کند
  • CTL/ATL/TSB:برای معیارهای نمودار مدیریت عملکرد مورد نیاز است
  • پیگیری پیشرفت:اندازه گیری عینی بهبود تناسب اندام هوازی
⚠️ وابستگی بحرانی:بدون آزمون CSS معتبر، معیارهای بار آموزشی پیشرفته (sTSS، CTL، ATL، TSB) قابل محاسبه نیستند. یک CSS نادرست تمام تحلیل های آموزشی بعدی را خراب می کند.

نحوه محاسبه CSS در 4 مرحله آسان

📋 پروتکل استاندارد

  1. گرم کردن

    300-800 متر شنای آسان، تمرینات، و تمرینات پیشرونده برای آماده شدن برای حداکثر تلاش.

  2. 400 متر تایم تریل

    حداکثر تلاش پایدار از شروع فشار (بدون شیرجه). ثبت زمان به دوم هدف: سریعترین پایدار 400 متر.

  3. ریکاوری کامل

    5-10 دقیقهشنای آسان یا استراحت کامل این برای نتایج دقیق بسیار مهم است.

  4. 200 متر تایم تریل

    حداکثر تلاش از شروع فشار. زمان را دقیقا ثبت کنید این باید در هر 100 متر سریعتر از 400 متر باشد.

🔄 فرکانس آزمایش مجدد

هر بار CSS را مجدداً آزمایش کنید6-8 هفتهبه روز رسانی مناطق تمرین با بهبود تناسب اندام. با تطبیق دادن با تمرین، مناطق شما باید به تدریج سریعتر شوند.

فرمول محاسبه CSS

فرمول

CSS (m/s) = (D2 - D1) / (T2 - T1)

کجا:

  • D1 = 200 متر
  • D2 = 400 متر
  • T1 = زمان 200 متر (بر حسب ثانیه)
  • T2 = زمان 400 متر (بر حسب ثانیه)

برای سرعت در هر 100 متر ساده شده است

سرعت CSS/100 متر (ثانیه) = (T400 - T200) / 2

نمونه کار شده

نتایج آزمون:

  • زمان 400 متر:6:08(368 ثانیه)
  • زمان 200 متر:2:30(150 ثانیه)

مرحله 1: CSS را بر حسب m/s محاسبه کنید

CSS = (400 - 200) / (368 - 150)
CSS = 200 / 218
CSS =0.917 متر بر ثانیه

مرحله 2: تبدیل به سرعت در هر 100 متر

سرعت = 100 / 0.917
سرعت = 109 ثانیه
سرعت =1:49 در هر 100 متر

🎯 مزایای آموزش مبتنی بر منطقه

استفاده از مناطق مبتنی بر CSS، آموزش ذهنی "احساس" را به تمرینات عینی و قابل تکرار تبدیل می کند. هر منطقه سازگاری های فیزیولوژیکی خاصی را هدف قرار می دهد:

  • منطقه 2:ساخت موتور هوازی (60-70 درصد حجم هفتگی)
  • منطقه 3:افزایش راندمان سرعت مسابقه (15-20٪ حجم)
  • منطقه 4:فشار آستانه لاکتات بالاتر (10-15٪ حجم)
  • منطقه 5:توسعه سرعت و قدرت بالا (5-10٪ از حجم)

🌊 توسعه شناگران

<0.9 متر بر ثانیه

ساخت پایه و تکنیک هوازی. کمتر از 1-2 سال آموزش مداوم.

اعتبار سنجی علمی

Wakayoshi و همکاران (1992-1993) - تحقیقات بنیادی

مطالعات اولیه Kohji Wakayoshi در دانشگاه اوزاکا CSS را به عنوان یک جایگزین معتبر و عملی برای آزمایش لاکتات آزمایشگاهی معرفی کرد:

  • همبستگی قوی با VO2 در آستانه بی هوازی(r = 0.818)
  • همبستگی عالی با سرعت در OBLA(r = 0.949)
  • عملکرد 400 متر را پیش بینی می کند(r = 0.864)
  • مربوط به 4 میلی مول در لیتر لاکتات خون است- حداکثر حالت پایدار لاکتات
  • رابطه خطی بین فاصله و زمان(r² > 0.998)

مقالات کلیدی:

  1. Wakayoshi K، و همکاران. (1992). "تعیین و اعتبار سرعت بحرانی به عنوان شاخصی از عملکرد شنا در شناگر رقابتی."مجله اروپایی فیزیولوژی کاربردی، 64 (2)، 153-157.
  2. Wakayoshi K، و همکاران. (1992). "یک روش ساده برای تعیین سرعت بحرانی به عنوان آستانه خستگی شنا در شنای رقابتی."مجله بین المللی پزشکی ورزشی، 13(5)، 367-371.
  3. Wakayoshi K، و همکاران. (1993). "آیا سرعت بحرانی شنا نشان دهنده شدت تمرین در حداکثر حالت پایدار لاکتات است؟"مجله اروپایی فیزیولوژی کاربردی، 66 (1)، 90-95.

🔬 چرا CSS کار می کند

CSS نشان دهنده مرز بینحوزه های ورزشی سنگین و شدید. در زیر CSS، تولید لاکتات و پاکسازی متعادل باقی می‌مانند—شما می‌توانید برای مدت طولانی شنا کنید. بالاتر از CSS، لاکتات به تدریج تا پایان 20-40 دقیقه جمع می شود.

این باعث می‌شود CSS به شدت عالی برای:

  • تنظیم سرعت مسابقه پایدار برای مسابقات 800-1500 متر
  • تجویز تمرینات اینتروال آستانه
  • نظارت بر بهبود تناسب اندام هوازی
  • محاسبه بار تمرینی و نیازهای بازیابی

چگونه CSS خود را تست کنیم

مرحله 1: به درستی گرم کنید

300-800 متر شنای آسان، از جمله تمرینات و افزایش تدریجی را کامل کنید. این کار بدن شما را برای حداکثر تلاش آماده می کند و از آسیب جلوگیری می کند.

مرحله 2: تایم تریل 400 متر را انجام دهید

400 متر با حداکثر تلاش مداوم از یک استارت فشار (بدون شیرجه) شنا کنید. زمان خود را به دومی ثبت کنید. اسپلیت های یکنواخت یا تقسیم منفی را هدف بگیرید (200 متر دوم برابر یا سریعتر از 200 متر اول).

مرحله 3: به طور کامل بازیابی کنید

5-10 دقیقه با شنای آسان یا استراحت کامل استراحت کنید. این بازیابی بسیار مهم است. قبل از ادامه، صبر کنید تا ضربان قلب شما به زیر 120 ضربه در دقیقه برسد و تنفس به طور کامل بهبود یابد.

مرحله 4: تایم تریل 200 متر را انجام دهید

200 متر با حداکثر تلاش از یک شروع فشار شنا کنید. زمان خود را به طور دقیق ثبت کنید. این باید در هر 100 متر سریعتر از سرعت 400 متری شما باشد.

مرحله 5: CSS خود را محاسبه کنید

از فرمول استفاده کنید: CSS Pace/100m = (T400 - T200) / 2. مثال: (368s - 150s) / 2 = 109s = 1:49/100m. یا از ماشین حساب موجود در این صفحه برای دریافت نتایج فوری با مناطق تمرین شخصی استفاده کنید.

کاربردهای عملی

1️⃣ معیارهای بار آموزشی را باز کنید

CSS مخرج در محاسبه ضریب شدت برای sTSS است. بدون آن، نمی‌توانید استرس تمرینی را اندازه بگیرید یا روند تناسب اندام/خستگی را دنبال کنید.

2️⃣ مناطق آموزشی را شخصی کنید

نمودارهای سرعت عمومی فیزیولوژی فردی را در نظر نمی گیرند. مناطق مبتنی بر CSS تضمین می کند که هر شناگر با شدت بهینه ترن می شود.

3️⃣ نظارت بر پیشرفت تناسب اندام

هر 6-8 هفته یکبار آزمایش مجدد انجام دهید. بهبود CSS (سرعت سریعتر) نشان دهنده سازگاری هوازی موفق است. CSS راکد نشان می دهد که آموزش نیاز به تعدیل دارد.

4️⃣ پیش بینی عملکرد مسابقه

سرعت CSS تقریباً به سرعت مسابقه پایدار 30 دقیقه ای شما می رسد. از آن برای تعیین اهداف واقعی برای 800 متر، 1500 متر و رویدادهای آب های آزاد استفاده کنید.

5️⃣ تمرینات آستانه طراحی

مجموعه‌های کلاسیک CSS: 8×100 @ سرعت CSS (15 ثانیه استراحت)، 5×200 @ 101٪ CSS (استراحت 20 ثانیه)، 3×400 @ 103٪ CSS (30 ثانیه استراحت). ظرفیت پاکسازی لاکتات را ایجاد کنید.

6️⃣ استراتژی Taper را بهینه کنید

CSS را قبل و بعد از Taper ردیابی کنید. یک مخروطی موفق CSS را حفظ یا کمی بهبود می بخشد و در عین حال خستگی را کاهش می دهد (افزایش TSB).
