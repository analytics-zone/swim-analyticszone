クリティカルスイムスピード (CSS) と閾値ペース: どちらが重要ですか?

長年にわたり、「しきい値ペース」（T-Pace）がゴールドスタンダードでした。 2026 年、クリティカル スイム スピード (CSS) への移行により、アスリートがセッションのペースを調整し、レース当日の疲労を予測する方法が再定義されています。

🎯 簡単な答え

T-ペース (閾値ペース) は、1 時間維持できるペースの一般的な推定値であり、広範囲のトレーニング ゾーンを設定するためによく使用されます。これは「実用的」ですが、それほど粒度の低い指標です。

CSS (クリティカルスイムスピード) は、2 つの異なる距離 (例: 400m と 200m) から数学的に導出される「関数的」閾値です。これは、「壁」にぶつかることなく維持できる理論上のペースです。あなたが持久力スイマーやトライアスリートの場合、CSS はトレーニング プログラムで最も重要な数値です。

2026 年のシフト: 努力から生理へ

プールでの疲労管理は、サイクリングやランニングに比べて難しいことで知られています。パワーメーターがなければ、強度を表す唯一の指標はペースです。多くの水泳選手は依然として昔ながらの 1500 メートルの「タイムトライアル」ペースに依存していますが、現代のコーチングは 2 距離 CSS モデル.

主な違いは次のとおりです 精度と知覚される労力。 1 回の長い努力に基づいた T ペースは、代謝の閾値よりも精神的な根性を獲得することがよくあります。 CSS は、「ペース-持続時間曲線」上の 2 つの異なる点から計算され、エンジンの実際の有酸素運動限界を明らかにします。

メトリックの比較

閾値ペース (T-ペース)

「単純な」指標

T ペースは通常、1500 メートルまたは 1000 メートルの 1 回の最大限の努力から導き出されます。泳ぎ続けるための「レッドライン」を表します。

  • 長所: わかりやすい。 1 つのテスト、1 つの数字。長時間の連続セットに最適です。
  • 短所: ペーシングエラーに非常に敏感です。疲労による衰退の速さは考慮されていません。同じ 1500 メートルのペースで泳ぐすべての選手を同じ生理学的プロファイルを持つものとして扱います。

クリティカルスイムスピード (CSS)

「機能的」指標

CSS は、2 つの距離 (通常は 400 m と 200 m の最大努力) に基づいた数学的導出です。

  • 長所: ペーシングのバイアスを除去します。一定間隔での行動をより正確に予測します。客観的な「好気性の天井」を提供します。
  • 短所: 2 つの別々のテストが必要です。もう少し複雑な計算 (Swim Analytics で自動計算)。一貫した休息をとりながらテストを実行する必要があります。

デルタが重要な理由

「デルタ」は、400 メートルのペースと 200 メートルのペースの差です。 2 人の水泳選手は両方とも 1:40/100m の T ペースを持っているかもしれませんが、彼らの CSS プロファイルは次のようになります。

スイマーA (「ザ・ディーゼル」)

200mから400mの間にある小さなデルタ地帯。このスイマーは有酸素効率が非常に高いですが、スプリント速度は遅くなります。彼らの CSS は T-Pace に非常に近いです。

スイマー B (「ザ・スプリンター」)

大きなデルタ。 200mでは素晴らしいスピードですが、400mでは大幅に速度が落ちます。彼らの CSS は、T-Pace が示唆するものよりも低くなります。一般的な「T ペース」でトレーニングしようとすると、無酸素システムが常に過剰に調理されてしまいます。

📈 CSS の式

CSS = (D1 - D2) / (T1 - T2)

ここで、D1/D2 は 2 つの距離、T1/T2 は秒単位の時間です。これにより、最初の無酸素性急上昇のノイズなしで、持続可能なペースの「勾配」が明らかになります。

CSS テストを実行する方法 (ステップバイステップ)

CSS を計算するには、標準プールで異なる日に (または間に完全な回復を挟んで) 2 つの最大トライアルを実行する必要があります。

  1. 準備し始める： 徹底的な標準化されたウォームアップ（通常はストロークとビルドを組み合わせて400メートルから600メートル）を実行して、体の準備をします。
  2. トライアル 1 (400 メートルの最大努力): 400メートルをできるだけ速く泳ぎます。正確な時間を記録します。最初の 100 メートルで燃え尽きてしまわないように、自分のペースで走りましょう。
  3. アクティブリカバリ: 乳酸を洗い流すために 100 メートルから 200 メートル泳ぎ、その後 5 ～ 10 分間休憩します。
  4. トライアル 2 (200 メートルの最大努力): 絶対最大努力で200m泳ぎます。正確な時間を記録します。
  5. 計算します: タイムを秒に変換して上記の式を適用するか、Swim Analytics の CSS 計算ツールを使用して 100 メートルのペースを取得します。

どれを使用するべきですか?

次の場合は、しきい値ペース (T-Pace) を使用します。

  • あなたはペースベースのトレーニングを始めたばかりです。
  • 長時間の連続水泳のみを実行します (オープンウォーター 3km 以上)。
  • 複数のテスト セッションに対処する必要はありません。

次の場合は、クリティカル スイム スピード (CSS) を使用します。

  • トライアスリートとマラソンスイマー: レースの早すぎる「ろうそくの火が消える」ことを避けるため。
  • インターバルトレーニング: 実際に機能する特定の CSS + 2s ゾーンまたは CSS - 1s ゾーンを設定します。
  • 進捗状況の追跡: CSS は、一般的な 1000 メートルのテストよりも有酸素性の向上に敏感です。

しきい値ペースと CSS に関するよくある質問

CSS はしきい値ペースと同じですか?

いいえ。これらは関連するしきい値の概念ですが、同じテストではありません。通常、閾値ペースは 1 回の長い泳ぎから得られますが、CSS は 2 回の努力から計算され、インターバルワークのより再現性の高いアンカーとなります。

トライアスロンとオープンウォーターペーシングにはどちらが適していますか?

CSS は、短期的なスピードに過度の報酬を与えることなく、持続可能なペースを反映するため、通常、CSS がより良い出発点となります。閾値ペースは長く安定して泳ぐ場合には依然として役立ちますが、通常は CSS の方がペース目標やゾーンワークに変換するのが簡単です。

しきい値ペースから CSS ベースのスイム ゾーンに移行するにはどうすればよいですか?

次の 400 メートルと 200 メートルのテストを使用して CSS を設定し、アンカー間隔でその数値を調整します。すでに T ペースでトレーニングしている場合は、両方の数値を比較し、CSS ガイドと水泳トレーニング ゾーンを使用して、再現可能な有酸素ペースを中心にセットを再構築してください。

プールゾーンを推測するのはやめましょう。

Swim Analytics は、CSS を実用的なプール ゾーン、インターバル ペーシング、および時間の経過に伴う有酸素性の上限の明確なビューに変えるのに役立ちます。
