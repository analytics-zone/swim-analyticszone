クリティカルスイムスピード (CSS)

水泳トレーニングゾーン、閾値セット、トレーニング負荷分析を固定する実用的なペースベンチマーク

簡単な回答

クリティカルスイムスピード (CSS) は、フィールドでテストされた、持続可能な水泳ペースの閾値の推定値です。ほとんどの水泳選手は次から計算します。 400m+200mタイムトライアル、その結果を使用して設定します トレーニングゾーン、ペース閾値セット、およびパワー sTSS そして トレーニング負荷 分析。

  • 最適な用途: 閾値トレーニング、反復可能なペーシング、負荷追跡
  • 典型的な解釈: ハードで制御された作業で有酸素運動を維持できるほぼ最速のペース
  • 盲目的に使用しないでください。 悪いペーシング、不完全な回復、異なるプールの状態、ドラッグエイドにより結果が歪む可能性があります。
  • 再テストの頻度: 6～8週間ごと、または有意義なトレーニングブロックの後に

重要なポイント

  • CSS とは: 簡単なフィールドテストから構築された、水泳選手のための実用的な閾値ペース推定値
  • なぜそれが重要なのか: CSS は水泳トレーニングを当て推量から反復可能なペーシングとより優れた負荷制御に変えます
  • それによってロックが解除されるもの:ゾーン、 sTSS、 パフォーマンス管理、より明確な進行状況チェック
  • できないこと: テクニック分析、オープンウォーターコンテキスト、または 効率の解釈
  • 最良の次のステップ: CSS を十分にテストし、それを使用してしきい値と有酸素運動を構成します。

ペーシング アンカーとして水泳メトリクスを 1 つだけ保持する場合は、CSS を保持します。これは完璧ではなく、完全な生理学的検査ではありませんが、実用的で再現性があり、直接役立ちます。この組み合わせが、CSS がデータ駆動型の水泳トレーニングのバックボーンであり続ける理由です。

ほとんどの水泳選手にとって、 クリティカルスイムスピード 生のタイムトライアルと日々のトレーニングの意思決定の間の架け橋となります。次の 5 つの重要な質問に答えるのに役立ちます。

  • しきい値セットは実際どのくらいの速さで設定すべきでしょうか?
  • それぞれにどのようなペース範囲が属するか 水泳ゾーン?
  • 100 秒、200 秒、400 秒を繰り返すペースはどうすればよいですか?
  • 水泳のストレスを数値化するにはどうすればよいですか sTSS?
  • 私の有酸素運動のフィットネスは正しい方向に進んでいますか?

CSS が測定するもの

CSS はハード有酸素水泳のペースの実際的な推定値です、通常は400メートルと200メートルのタイムトライアルから導き出されます。コーチングの言葉では、構造化されたプールトレーニングの閾値ペースとして扱われることがよくあります。つまり、重要になるほど速く、繰り返すのに十分に制御され、ゾーンを構築するのに十分に安定しているということです。

アスペクト実際的な答え
CSS が測定するもの持続的な有酸素運動のための現場で推定された水泳ペースの閾値
最適な使用方法トレーニングゾーン、閾値セット、ペース目標、および トレーニング負荷
よくある間違い1 つの悪いテストを永続的な真実として扱うか、プールの長さ、休憩、ペーシングの質を考慮せずに CSS を使用する
関連する指標トレーニングゾーン、 sTSS、 CTL/ATL/TSB、そして SWOLF / 効率

🎯 役立つメンタルモデル

CSS を自分のものとして考えてください 稼働ペースのベンチマーク、魔法の数字ではありません。残りの水泳指標を一貫性のあるものにするのは、ペースの基準です。

CSS が重要な理由

CSS が重要なのは、生理学と実行を結び付けるためです。多くの水泳選手は、自分が「ハードセット」や「テンポセッション」を行ったことは知っていますが、そのペースが適切だったのかどうかはわかりません。 CSS は、セッションごとに使用できるベンチマークを提供することで、この問題を解決します。

🏊 トレーニングゾーン

CSS はクリーンなアンカーを提供します 水泳ゾーン、そのため、簡単な作業は簡単なままであり、しきい値の作業はターゲットを絞ったままになります。

📊 トレーニング負荷

使えるCSSがないと、 sTSS そして パフォーマンス管理指標 意味を失います。

⏱️ 再現可能なペーシング

CSS を使用すると、感触だけに頼るのではなく、より一貫性を持って 100 秒、200 秒、400 秒のペースを保つことができます。

📈 進捗状況のチェック

CSS が向上すると、しきい値のペースが速くなります。これは、有酸素水泳のフィットネスも向上していることを示す強力な兆候です。

⚠️ 重大な依存関係: CSS が間違っていると、ゾーン、sTSS、およびパフォーマンス管理の数値が間違った方向に流れてしまいます。不適切な入力は不適切な分析を生み出します。

CSS のメリットとそうでないもの

CSS は次の場合に優れています。 閾値セット、有酸素性の質の高いワーク、ゾーン設定、水泳特有の負荷追跡。

CSS だけでは次のような場合には十分ではありません。 スプリントプロファイリング、ストロークテクニック診断、さまざまな条件下でのオープンウォーターレースの実行、またはそれ自体の効率の解釈。

CSS を使用する目的

  • 閾値ペースの処方箋
  • 100/200/400 ペーシングを繰り返す
  • パーソナライズされた ゾーン構造
  • sTSS そして 負荷管理
  • 季節を通じた有酸素運動の進捗状況の追跡

CSS を単独で使用しないでください

  • 異なるプールの長さを同一であるかのように比較する
  • まったく異なる技術プロファイルを持つ水泳選手を比較する
  • ストロークの質を判断せずに ストロークの仕組み
  • 周囲の状況を考慮せずに効率性を判断する スウルフ
  • オープンウォーターのペースがプールの CSS ペースと正確に一致すると仮定する

4 つの実践的なステップで CSS をテストする方法

📋 標準プロトコル

  1. しっかりウォーミングアップ

    300 ～ 800 メートルを簡単に泳ぎ、ドリルを組み込んで、テスト前にいくつかの制御された体づくりの努力を追加します。

  2. 400メートルのタイムトライアルを泳ぐ

    ダイブではなくプッシュスタートを使用してください。目標は、無謀な最初の 100 メートルではなく、持続可能な最速の 400 メートルです。

  3. 完全に回復する

    5～10分程度かかります 簡単な休息または完全な休息。不完全なリカバリは、結果を台無しにする最も早い方法の 1 つです。

  4. 200メートルのタイムトライアルを泳ぐ

    もう一度プッシュスタートを使用して時間を正確に記録します。 200 メートルのペースは 400 メートルのペースよりも有意に速いはずです。

⚠️ 最も一般的なテストエラー

400mのペース配分が悪い

問題: 頑張りすぎて、ひどくフェードアウトしてしまう。

結果: 400メートルの停止は持続可能なペースの質を反映しています。

修正: 均等またはわずかにマイナスのスプリットを目指します。

回復が不十分

問題: 疲労により200メートルが人為的に遅くなります。

結果: CSS が歪んでしまい、トレーニング ゾーンがずれてしまいます。

修正: 呼吸がコントロールされ、完全に回復したと感じるまで待ちます。

混合条件

問題: プールの長さの違い、プッシュオフの大きな変化、またはドラッグ補助により、テスト環境が変化します。

結果: 実際には比較できない数値を比較します。

修正: テストごとに条件を可能な限り一定に保ちます。

🔄 再テストの頻度

CSS を次の間隔で再テストする 6～8週間、または閾値、持久力、またはテクニックの統合に焦点を当てた有意義なブロックの後。重要なのは継続的なテストではありません。重要なのは、変化を確認できる程度に 1 つのベンチマークを一貫して使用することです。

CSS計算式

計算式

CSS (m/s) = (D₂ - D₁) / (T₂ - T₁)

場所:

  • D₁ = 200 メートル
  • D₂ = 400 メートル
  • T₁ = 200m の時間 (秒単位)
  • T₂ = 400m の時間 (秒)

100mあたりのペースの簡略化

CSS ペース/100m (秒) = (T₄₀₀ - T₂₀₀) / 2

作業例

テスト結果

  • 400メートルタイム： 6:08 (368秒)
  • 200メートルタイム： 2:30 (150秒)

ステップ 1: CSS ペースを計算する

CSS ペース/100m = (368 - 150) / 2
CSS ペース/100m = 218 / 2
CSS ペース/100m = 109 秒 = 100 メートルあたり 1:49

ステップ 2: 必要に応じて速度に変換する

CSS = 100 / 109
CSS = 0.917m/秒

実用的な例: その結果の使用方法

CSS が 1:49/100m、しきい値の繰り返しがそのペース近くに集中するはずです。 ゾーンテーブル そこから構築する必要があります。 sTSS の計算 それを分母ペースの基準として使用する必要があります。

実際のトレーニングで CSS を使用する方法

1️⃣ ペース閾値セット

CSS を使用して推測をやめてください。 8x100、5x200、または 3x400 などのセットは、ペース目標がわかっていて、それをきれいに維持できる場合に、より意味があります。

2️⃣ ペースをsTSSに接続します

CSS は、 sTSS。難しいセッションと簡単なセッションが、同じ漠然としたワークロードに溶け込むのをやめます。

3️⃣ 進捗状況をより良く追跡する

セッションを制御しながら CSS が高速化すれば、水泳フィットネスの閾値が向上している可能性があります。

4️⃣ より良いペース配分の習慣を築く

CSS は、すべてのハードセットをペースの崩壊に変えるのではなく、コントロールしながら反復作業を泳ぐことを教えます。

CSS 指向のセッションのアイデアのサンプル

  • 閾値の維持: CSS ペースで 8x100、短い休憩あり
  • ペース耐久性: 一貫した分割による CSS ペースあたり 5x200
  • 有酸素サポート: ストローク数を制御した場合の 3x400 は CSS ペースよりわずかに遅い
  • テクニック + ペース: 100 メートルの CSS 作業と 50 メートルの簡単なテクニカル リセットを交互に実行

これらは一例であり、普遍的な処方箋ではありません。重要なのは、CSS によって安定したペーシング言語が提供されるということです。

CSS、SWOLF、および効率

CSS が教えてくれる どのくらい速いか 閾値で泳ぐことができます。 SWOLF と効率の指標 何か違うことを教えてください。水中をどれだけ経済的に移動しているかです。これらは相互に補完しますが、互換性はありません。

SWOLFを盲目的に比較しないでください

SWOLF が低いほど、スイマー、セッション、ストローク、またはプールのセットアップが常に「優れている」とは限りません。使用する 効率性の指標 スタンドアロンのランキング ツールとしてではなく、CSS、ストロークの品質、再現性と並行したコンテキストとして。

科学的検証

Wakayoshi 他と CSS の実践例

CSS は、実験室での乳酸検査を繰り返し行う必要がなく、閾値関連のパフォーマンス マーカーとよく一致するフィールドフレンドリーなベンチマークをコーチや水泳選手に提供したため、広く使用されるようになりました。

  • 実用的: 単純なプールテストから再現可能
  • 役に立つ： プログラミングの閾値に関連したペーシングに十分近い
  • 実用的なもの: シーズンを通して再テストと更新が簡単
  • 接続済み: トレーニングゾーン、ペーシング、負荷追跡に直接役立ちます

よくある質問

CSS はどのくらいの頻度で再テストする必要がありますか?

デフォルトでは 6 ～ 8 週間ごとが適切です。トレーニングが大幅に変更された場合、または現在のゾーンが明らかに間違っていると感じた場合にのみ、すぐに再テストしてください。

CSS は乳酸閾値と同じですか?

正確には違います。 CSS は、実際のしきい値ペースのベンチマークとして使用されるフィールド推定値です。これが役立つのは、臨床検査を完全に置き換えるためではなく、再現可能で実用的なためです。

オープンウォーターペーシングに CSS を使用できますか?

これは文字通りの保証ではなく、出発点の参考として使用してください。目視、ドラフト、チョップ、ウェットスーツ、ナビゲーションはすべて、オープンウォーターでのペースの挙動を変えます。

初心者でもCSSは使えるのか？

はい、しかし、テクニックとペースが向上するにつれて、結果はより有用になります。初心者のスイマーは、CSS が高度に安定する前にストロークとペーシングの一貫性を必要とすることがよくあります。

自分の CSS を他の人の CSS と直接比較する必要がありますか?

ただ慎重に。プールの長さ、プッシュオフ、ドラッグ、ペーススキル、テクニックの質はすべて結果に影響します。 CSS は、時間の経過とともに個人的なベンチマークとして最も強力になります。
