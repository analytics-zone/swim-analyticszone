Критическая скорость плавания (CSS)

Практический эталон темпа, который определяет зоны тренировок по плаванию, наборы порогов и анализ тренировочной нагрузки.

Быстрый ответ

Критическая скорость плавания (CSS) — это проверенная на практике оценка вашего устойчивого порогового темпа плавания. Большинство пловцов рассчитывают его по 400 м + 200 м, гонка на время, затем используйте результат для установки тренировочные зоны, наборы пороговых значений темпа и мощность sTSS и тренировочная нагрузка анализ.

  • Лучшее использование: пороговая тренировка, повторяемая стимуляция и отслеживание нагрузки
  • Типичная интерпретация: примерно самый быстрый темп, который вы можете поддерживать в аэробном режиме для тяжелой, контролируемой работы
  • Не используйте его вслепую: плохой темп, неполное восстановление, различные условия в бассейне или вспомогательные средства сопротивления могут исказить результат
  • Повторное тестирование частоты: каждые 6-8 недель или после значимого тренировочного блока

Ключевые выводы

  • Что такое CSS: практическая оценка порогового темпа для пловцов, полученная на основе простых полевых испытаний
  • Почему это важно: CSS превращает тренировку по плаванию из догадок в повторяемую скорость и лучший контроль нагрузки
  • Что он открывает:зоны, sTSS, управление производительностьюи более четкие проверки прогресса
  • Чего он не делает: заменить технический анализ, контекст открытой воды или интерпретация эффективности
  • Лучший следующий шаг: тщательно протестируйте CSS, а затем используйте его для структурирования своего порога и аэробной работы.

Если в качестве привязки темпа вы используете только один показатель плавания, сохраните CSS. Он не идеален и не является полноценным физиологическим лабораторным тестом, но он практичен, воспроизводим и очень полезен. Именно эта комбинация является причиной того, что CSS остается основой тренировок по плаванию, основанных на данных.

Для большинства пловцов Критическая скорость плавания это мост между обычными гонками на время и повседневными тренировочными решениями. Это помогает ответить на пять важных вопросов:

  • Насколько быстрыми на самом деле должны быть установки порогов?
  • Какой диапазон темпов принадлежит каждому зона для купания?
  • Как мне следует повторять 100, 200 и 400 секунд?
  • Как количественно оценить стресс от плавания с помощью sTSS?
  • Движется ли моя аэробная тренировка по плаванию в правильном направлении?

Что измеряет CSS

CSS — это практическая оценка темпа тяжелого аэробного плавания., обычно полученный на основе результатов гонок на время на 400 и 200 метров. На тренерском языке его часто рассматривают как пороговый темп для структурированных тренировок в бассейне: достаточно быстрый, чтобы иметь значение, достаточно контролируемый, чтобы повторять, и достаточно стабильный, чтобы выстраивать вокруг него зоны.

🎯 Полезная ментальная модель

Думайте о CSS как о своем эталон темпа работы, а не магическое число. Именно эталон темпа обеспечивает согласованность остальных показателей вашего плавания.

Почему CSS имеет значение

CSS важен, потому что он связывает физиологию с исполнением. Многие пловцы знают, что они выполнили «тяжелый подход» или «темповую тренировку», но не могут сказать, был ли выбран правильный темп. CSS решает эту проблему, предоставляя вам эталон, который вы можете использовать сеанс за сеансом.

🏊 Тренировочные зоны

CSS дает вам чистый якорь для зоны для купания, поэтому легкая работа остается легкой, а пороговая работа остается целенаправленной.

📊 Тренировочная нагрузка

Без полезного CSS, sTSS и показатели управления производительностью теряют свой смысл.

⏱️ Повторяемый темп

CSS помогает вам шагать по 100, 200 и 400 с большей последовательностью, а не полагаться только на ощущения.

📈 Проверка прогресса

Когда CSS улучшается, ваш пороговый темп увеличивается. Это сильный сигнал о том, что аэробная физическая форма плавания также улучшается.

⚠️ Критическая зависимость: Если ваш CSS неправильный, ваши зоны, sTSS и показатели управления производительностью будут смещаться в неправильном направлении. Плохие входные данные приводят к плохому анализу.

Для чего CSS хорош, а для чего нет

CSS отлично подходит для: пороговые значения, аэробная качественная работа, настройка зон и отслеживание нагрузки, специфичной для плавания.

CSS недостаточно для: профилирование спринта, диагностика техники гребка, проведение гонок на открытой воде в различных условиях или интерпретация эффективности сама по себе.

Используйте CSS для

  • Рекомендации по пороговому темпу
  • Повторите темп 100/200/400.
  • Персонализированный структура зоны
  • sTSS и управление нагрузкой
  • Отслеживание аэробного прогресса в течение сезона

Не используйте только CSS для

  • Сравнение разных длин бассейнов, как если бы они были идентичными
  • Сравнение пловцов с очень разными профилями техники
  • Оценка качества гребка без механика удара
  • Оценка эффективности без контекста СВОЛФ
  • Предполагая, что темп на открытой воде будет точно соответствовать темпу CSS в бассейне.

Как протестировать CSS за 4 практических шага

📋 Стандартный протокол

  1. Разогревайтесь правильно

    Легко проплывите 300–800 м, включите упражнения и добавьте несколько контролируемых упражнений перед тестом.

  2. Проплыть 400 метров в гонке на время

    Используйте толчок, а не пикирование. Цель – ваш самый быстрый устойчивый забег на 400 метров, а не безрассудные первые 100 метров.

  3. Полностью восстановиться

    Уделите 5-10 минут легкий или полный отдых. Неполное восстановление – один из самых быстрых способов испортить результат.

  4. Проплыть дистанцию 200 метров на время

    Снова используйте кнопку запуска и точно запишите время. Ваш темп на 200 м должен быть значительно быстрее, чем на 400 м.

⚠️ Наиболее распространенные ошибки тестирования

Плохой темп на 400 м.

Проблема: Вы выходите слишком сильно, а затем плохо исчезаете.

Результат: Бег на 400 метров перестает отражать устойчивое качество темпа.

Исправление: Стремитесь к равномерному или слегка отрицательному расколу.

Недостаточное восстановление

Проблема: Усталость делает бег на 200 метров искусственно медленным.

Результат: CSS искажается, и ваши тренировочные зоны смещаются.

Исправление: Подождите, пока дыхание не станет контролируемым и выздоровление не станет полным.

Смешанные условия

Проблема: Различная длина бассейна, большие изменения отталкивания или средства перетаскивания меняют среду тестирования.

Результат: Вы сравниваете цифры, которые на самом деле несопоставимы.

Исправление: Сохраняйте условия как можно более постоянными от испытания к испытанию.

🔄 Частота повторного тестирования

Повторно тестируйте CSS каждые 6-8 недельили после значимого блока, посвященного порогу, выносливости или закреплению техники. Дело не в постоянном тестировании. Суть в том, чтобы использовать один эталонный показатель достаточно последовательно, чтобы увидеть изменения.

Формула расчета CSS

Формула

CSS (м/с) = (D₂ - D₁) / (T₂ - T₁)

Где:

  • D₁ = 200 метров
  • Д₂ = 400 метров
  • T₁ = Время на 200 м (в секундах)
  • T₂ = Время на 400 м (в секундах)

Упрощено для темпа на 100 м.

CSS Темп/100 м (секунды) = (T₄₀₀ - T₂₀₀) / 2

Рабочий пример

Результаты испытаний

  • Время на 400 м: 6:08 (368 секунд)
  • Время на 200 м: 2:30 (150 секунд)

Шаг 1. Рассчитайте темп CSS

CSS Темп/100 м = (368 - 150) / 2
CSS Темп/100м = 218/2
CSS Темп/100 м = 109 секунд = 1:49 на 100 м.

Шаг 2. При необходимости преобразуйте в скорость.

CSS = 100/109
CSS = 0,917 м/с

Рабочий пример: как использовать этот результат

Если ваш CSS 1:49/100м, ваши пороговые повторы должны группироваться примерно в этом темпе, ваш таблица зон должно быть построено на его основе, и ваш sTSS расчеты следует использовать его в качестве ориентира темпа знаменателя.

Как использовать CSS в реальном обучении

1️⃣ Наборы порогов темпа

Используйте CSS, чтобы перестать гадать. Такие подходы, как 8x100, 5x200 или 3x400, имеют больше смысла, когда вы знаете целевой темп и можете четко его удерживать.

2️⃣ Подключите Pace к sTSS

CSS придает смысл вашему sTSS. Тяжелые занятия и легкие занятия перестают сливаться в одну и ту же неопределенную рабочую нагрузку.

3️⃣ Лучше отслеживайте прогресс

Если CSS становится быстрее, а ваши тренировки остаются под контролем, ваша пороговая подготовленность к плаванию, вероятно, улучшится.

4. Выработайте привычку лучше ориентироваться

CSS учит вас плавно повторять работу, контролируя ее, вместо того, чтобы превращать каждый тяжелый сет в темповый коллапс.

Примеры идей сеансов, ориентированных на CSS

  • Поддержание порога: 8х100 в темпе CSS с коротким отдыхом
  • Темп долговечности: 5x200 в темпе CSS с последовательным разделением
  • Аэробная поддержка: 3х400 немного медленнее, чем темп CSS, с контролируемым количеством гребков
  • Техника + темп: чередуйте 100 м с усилием CSS с легким техническим сбросом на 50 м.

Это примеры, а не универсальные рецепты. Дело в том, что CSS дает вам стабильный язык темпа.

CSS, SWOLF и эффективность

CSS говорит вам как быстро вы можете плавать у порога. SWOLF и показатели эффективности скажу вам другое: насколько экономно вы передвигаетесь по воде. Они дополняют друг друга, но не взаимозаменяемы.

Не сравнивайте SWOLF вслепую

Более низкий SWOLF не всегда «лучше» для пловцов, сессий, гребков или настроек бассейна. Использование показатели эффективности как контекст наряду с CSS, качеством штрихов и повторяемостью, а не как отдельный инструмент ранжирования.

Часто задаваемые вопросы

Как часто мне следует повторно тестировать CSS?

Каждые 6-8 недель — хороший вариант по умолчанию. Повторите тестирование раньше, только если тренировка сильно изменилась или ваши текущие зоны явно кажутся неправильными.

CSS — это то же самое, что порог лактата?

Не совсем. CSS — это полевая оценка, используемая в качестве практического ориентира порогового темпа. Это полезно, потому что оно повторяемо и осуществимо, а не потому, что оно идеально заменяет лабораторное тестирование.

Могу ли я использовать CSS для плавания по открытой воде?

Используйте его как отправную точку, а не как буквальную гарантию. Прицеливание, драфтинг, рубка, гидрокостюмы и навигация — все это меняет поведение темпа на открытой воде.

Могут ли новички использовать CSS?

Да, но результат становится более полезным по мере улучшения техники и темпа. Новичкам-пловцам часто требуется постоянство гребка и темпа, прежде чем CSS станет очень стабильным.

Должен ли я напрямую сравнивать свой CSS с чьим-то другим?

Только осторожно. Длина бассейна, отталкивания, сопротивление, умение шагать и качество техники — все это влияет на результат. CSS является наиболее мощным личным ориентиром с течением времени.

Используйте CSS в качестве центра кластера показателей плавания
