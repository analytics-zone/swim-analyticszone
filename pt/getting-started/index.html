Começar com Swim Analytics

O teu guia completo para monitorizar o desempenho de natação, testes CSS e análise de carga de treino

Bem-vindo à Natação Orientada por Dados

Swim Analytics transforma os teus treinos de natação em insights acionáveis usando métricas de Velocidade Crítica de Natação (CSS), Training Stress Score (sTSS) e Gráfico de Gestão de Desempenho (PMC). Este guia vai levar-te da configuração inicial até à análise avançada de carga de treino em 4 passos simples.

Início Rápido (5 Minutos)

1

Descarregar e Instalar

Descarrega Swim Analytics da App Store e concede permissão para aceder ao Apple Health. A app sincroniza treinos de natação automaticamente—sem necessidade de registo manual.

Descarregar App →
2

Realizar Teste CSS

Completa um teste contrarrelógio de 400m e 200m para estabelecer a tua Velocidade Crítica de Natação. Esta é a base de todas as métricas—sem CSS, sTSS e zonas de treino não podem ser calculados.

Protocolo de Teste CSS ↓
3

Inserir Resultados CSS

Insere os teus tempos de 400m e 200m na app. Swim Analytics calcula CSS, zonas de ritmo e personaliza todas as métricas à tua fisiologia. Atualiza a cada 6-8 semanas conforme a condição física melhora.

4

Começar a Monitorizar Treinos

Nada com Apple Watch e app Saúde. Swim Analytics importa automaticamente treinos, calcula sTSS, atualiza CTL/ATL/TSB e monitoriza progresso. Sem necessidade de entrada manual de dados.

Protocolo Completo de Teste CSS

📋 O Que Precisas

  • Acesso a piscina: Piscina de 25m ou 50m (25yd aceitável)
  • Cronometragem: Cronómetro, relógio de parede, ou Apple Watch
  • Tempo de aquecimento: 15-20 minutos antes do teste
  • Recuperação: 5-10 minutos entre testes
  • Esforço: Ritmo máximo sustentável (não sprint total)

⏱️ Condições do Dia de Teste

  • Descansado: Sem treino pesado 24-48 horas antes
  • Hidratado: Bem hidratado, alimentação normal
  • Temperatura piscina: 26-28°C ideal (evitar muito fria/quente)
  • Hora do dia: Quando normalmente treinas melhor
  • Equipamento: Igual ao treino (óculos, touca, fato de banho)

Teste CSS Passo a Passo

Aquecimento

15-20 minutos

400-800m nado suave, educativos e progressivos. Inclui 2-3×50 em ritmo crescente (60%, 75%, 85% esforço). Descansa 2-3 minutos antes do teste.

Teste 1

400m Esforço Máximo

Partida da parede (sem mergulho). Nada 400m ao ritmo mais rápido que consegues sustentar para toda a distância. Isto NÃO é um sprint—doseias o teu esforço. Regista o tempo em formato mm:ss (ex: 6:08).

Dica de ritmo: Procura parciais de 100m uniformes. Segundos 200m devem ser ≤ primeiros 200m (negativo ideal).
Recuperação

5-10 minutos

FASE CRÍTICA: Nado suave ou descanso completo. Aguarda até a frequência cardíaca baixar de 120 bpm e respiração totalmente recuperada. Recuperação insuficiente = CSS impreciso.

Teste 2

200m Esforço Máximo

Partida da parede (sem mergulho). Esforço máximo sustentável para 200m. Isto deve sentir-se mais difícil por 100m do que os 400m. Regista o tempo em formato mm:ss (ex: 2:30).

Verificação de validação: Ritmo 200m/100m deve ser 3-6 segundos mais rápido que ritmo 400m/100m. Se não, a recuperação foi insuficiente ou a dosagem estava incorreta.
Desaquecimento

10-15 minutos

300-500m nado suave, alongamentos. Regista os teus tempos imediatamente—não confies na memória.

⚠️ Erros Comuns no Teste CSS

  • Partir muito rápido nos 400m: Resulta em colapso, CSS impreciso. Usa ritmo uniforme.
  • Recuperação insuficiente entre testes: Fadiga atrasa 200m, tornando CSS artificialmente rápido → zonas de sobretreinamento.
  • Usar partidas com mergulho: Adiciona vantagem de 0,5-1,5s, distorcendo cálculos. Sempre parte da parede.
  • Testar quando fatigado: Carga de treino pesada 24-48h antes = resultados deprimidos. Testa quando fresco.
  • Não registar imediatamente: Memória não é confiável. Escreve os tempos antes do desaquecimento.

Inserir Resultados CSS em Swim Analytics

Passo 1: Abrir Definições CSS

Na app Swim Analytics, vai a Definições → Velocidade Crítica de Natação. Toca "Realizar Teste CSS" ou "Atualizar CSS".

Passo 2: Inserir Tempos

Insere o teu tempo de 400m (ex: 6:08) e tempo de 200m (ex: 2:30). Usa o formato exato mostrado. Toca "Calcular".

Passo 3: Rever Resultados

A app exibe:

  • Velocidade CSS: 0,917 m/s
  • Ritmo CSS: 1:49/100m
  • Zonas de treino: 7 zonas personalizadas (Zona 1-7)
  • Linha base sTSS: Agora ativada para todos os treinos

Passo 4: Guardar e Sincronizar

Toca "Guardar CSS". A app imediatamente:

  • Recalcula zonas de treino
  • Atualiza retroativamente sTSS dos últimos 90 dias
  • Ajusta cálculos CTL/ATL/TSB
  • Ativa análise de treino baseada em zonas

💡 Dica Pro: Testes CSS Históricos

Se já conheces a tua CSS de testes anteriores, podes inserir esses tempos diretamente. No entanto, para resultados mais precisos, realiza um teste novo a cada 6-8 semanas. A tua CSS deve melhorar (ficar mais rápida) conforme o treino progride.

Compreender as Tuas Métricas

Velocidade Crítica de Natação (CSS)

O que é: O teu ritmo de limiar aeróbico—a velocidade mais rápida que podes sustentar por ~30 minutos sem exaustão.

O que significa: CSS = 1:49/100m significa que podes manter ritmo 1:49 para esforços de limiar sustentados.

Como usar: Base para todas as zonas de treino e cálculo de sTSS. Atualiza a cada 6-8 semanas.

Aprender CSS →

Zonas de Treino

O que são: 7 intervalos de intensidade baseados na tua CSS, de recuperação (Zona 1) a sprint (Zona 7).

O que significam: Cada zona visa adaptações fisiológicas específicas (base aeróbica, limiar, VO₂máx).

Como usar: Segue prescrições de zona para treino estruturado. A app mostra tempo em zona para cada treino.

Zonas de Treino →

Pontuação de Stress de Treino de Natação (sTSS)

O que é: Stress de treino quantificado combinando intensidade e duração. 1 hora em ritmo CSS = 100 sTSS.

O que significa: sTSS 50 = recuperação fácil, sTSS 100 = moderado, sTSS 200+ = sessão muito difícil.

Como usar: Monitoriza sTSS diário/semanal para gerir carga de treino. Meta de aumento de 5-10 sTSS por semana no máximo.

Guia sTSS →

CTL / ATL / TSB

O que são:

  • CTL: Chronic Training Load (condição) - média de 42 dias de sTSS
  • ATL: Acute Training Load (fadiga) - média de 7 dias de sTSS
  • TSB: Training Stress Balance (forma) = CTL - ATL

Como usar: TSB positivo = fresco/polido, TSB negativo = fatigado. Compete quando TSB = +5 a +25.

📊 Metas da Tua Primeira Semana

Após inserir CSS e completar 3-5 treinos:

  • Verificar valores sTSS: Confirma que coincidem com perceção de esforço (fácil ~50, moderado ~100, difícil ~150+)
  • Rever distribuição de zona: Estás a passar 60-70% na Zona 2 (base aeróbica)?
  • Estabelecer CTL base: Média sTSS da primeira semana torna-se linha base de condição inicial
  • Identificar padrões: Que treinos geram maior sTSS? Estás a recuperar adequadamente?

Jornada Típica do Utilizador (Primeiras 8 Semanas)

Semana 1-2: Estabelecer Linha Base

  • Realizar teste CSS e inserir resultados
  • Completar 3-5 treinos normais
  • Observar valores sTSS e distribuição de zona
  • Estabelecer CTL inicial (nível de condição)
  • Meta: Compreender métricas, sem mudanças ainda

Semana 3-4: Aplicar Zonas

  • Usar zonas CSS no planeamento de treinos
  • Intencionalmente nadar Zona 2 para séries aeróbicas
  • Monitorizar totais sTSS semanais (procurar consistência)
  • Monitorizar TSB (deve ser ligeiramente negativo = a treinar)
  • Meta: Treinar por zonas, não sensação

Semana 5-6: Sobrecarga Progressiva

  • Aumentar sTSS semanal 5-10% da linha base
  • Adicionar 1 sessão de limiar (Zona 4) por semana
  • CTL deve subir gradualmente (condição a melhorar)
  • ATL pode ter pico em semanas difíceis (normal)
  • Meta: Progressão controlada da condição

Semana 7-8: Retestar e Ajustar

  • Realizar segundo teste CSS (deve ser mais rápido)
  • Atualizar zonas na app (ritmo melhora)
  • Comparar CTL Semana 1 vs Semana 8 (deve ser +10-20)
  • Rever progresso: Tempos a baixar? Sentes mais fácil?
  • Meta: Validar eficácia do treino

✅ Indicadores de Sucesso

Após 8 semanas de treino estruturado com Swim Analytics, deves ver:

  • Melhoria CSS: 1-3% ritmo CSS mais rápido (ex: 1:49 → 1:47)
  • Aumento CTL: +15-25 pontos (ex: 30 → 50 CTL)
  • sTSS consistente: Totais semanais dentro de 10-15% variância
  • Melhor dosagem: Parciais mais uniformes, melhor calibração de esforço
  • Recuperação melhorada: TSB cicla previsivelmente (-10 a +5)

Resolução de Problemas e Perguntas Frequentes

O meu sTSS parece muito alto/baixo para o esforço do treino

Causa: CSS está desatualizada ou imprecisa.

Solução: Retesta CSS. Se testaste quando fatigado ou doseaste mal, a CSS estará errada. Um teste CSS adequado é crítico para todas as métricas subsequentes.

A app mostra "Nenhuma CSS configurada"

Causa: Teste CSS não completado ou não guardado.

Solução: Vai a Definições → Velocidade Crítica de Natação → Realizar Teste. Insere ambos os tempos de 400m e 200m, depois toca Guardar.

Treinos não sincronizam do Apple Watch

Causa: Permissões da app Saúde não concedidas ou treino não categorizado como "Natação".

Solução: Verifica Definições → Privacidade → Saúde → Swim Analytics → Permitir Leitura para Treinos. Assegura que o tipo de treino Apple Watch é "Natação em Piscina" ou "Natação em Águas Abertas".

CTL não aumenta apesar de treino consistente

Causa: Totais sTSS muito baixos ou frequência inconsistente.

Solução: CTL é média ponderada exponencialmente de 42 dias. Sobe lentamente. Aumenta sTSS semanal 5-10%, e mantém 4+ treinos/semana para crescimento CTL consistente.

Com que frequência devo retestar CSS?

Recomendação: A cada 6-8 semanas durante fases base/construção. Retesta após doença, lesão, pausa longa, ou quando as zonas sentem consistentemente muito fáceis/difíceis.

Posso usar Swim Analytics para outros estilos?

Sim, com limitações: CSS é tipicamente testada em crol. Para treinos medley/costas/bruços, sTSS é calculado baseado em CSS de crol. Considera realizar testes CSS específicos por estilo para maior precisão.
